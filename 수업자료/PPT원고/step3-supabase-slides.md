## 슬라이드 2: 친구도 접속할 수 있으려면?

**핵심 텍스트:**
- padlet.db는 내 컴퓨터에만 있다

**시각 자료 제안:** 왼쪽: 내 컴퓨터(padlet.db) + 나 혼자. 오른쪽: 구름(클라우드 DB) + 여러 명이 접속하는 모습. 가운데에 화살표 "업그레이드!"

**선생님 멘트:** "내 컴퓨터에 있는 파일은 나만 쓸 수 있죠. 그런데 인터넷 어딘가에 DB를 두면 어떨까요? 누구든 인터넷만 되면 접속할 수 있겠죠! 이게 바로 클라우드 데이터베이스입니다."

---
## 슬라이드 3: Supabase란?

**핵심 텍스트:**
- Supabase = 클라우드 PostgreSQL을 쉽게 만들어주는 서비스
- 브라우저에서 클릭 몇 번으로 DB 생성
- 웹 대시보드에서 표로 데이터 확인 (Notion처럼!)
- 무료 (프로젝트 2개까지)

**시각 자료 제안:** Supabase 로고 + 웹 대시보드 스크린샷 위치 표시 (표 형태 데이터 뷰). "Notion처럼 눈으로 확인!" 말풍선

**선생님 멘트:** "Supabase는 클라우드 데이터베이스를 아주 쉽게 만들어주는 서비스입니다. Step 1에서 Notion 대시보드로 데이터를 확인했던 것 기억나죠? Supabase도 비슷해요. 웹에서 표로 데이터를 볼 수 있습니다."

---
## 슬라이드 4: 3단계 비교 다이어그램

**핵심 텍스트:**
- Step 1: 브라우저 -> 서버 -> Notion (남의 서비스)
- Step 2: 브라우저 -> 서버 -> padlet.db (내 컴퓨터)
- Step 3: 브라우저 -> 서버 -> Supabase (클라우드)
- 비유: 남의 가게 장부 / 내 책상 장부 / 학교 서버실 장부

**시각 자료 제안:** 3행 구조도. 브라우저와 서버는 모두 같은 아이콘, DB 부분만 다른 아이콘과 색상. 각 행 옆에 비유 텍스트

**선생님 멘트:** "3단계를 한눈에 비교해 봅시다. 브라우저와 서버는 매번 같고, DB만 바뀌었죠? 장부를 남의 가게에 뒀다가, 내 책상에 가져왔다가, 이번엔 학교 서버실에 두는 것과 같습니다."

## 슬라이드 5: Supabase 설정 절차

**핵심 텍스트:**
- 1. supabase.com 접속 + GitHub 로그인 (1분)
- 2. New Project 만들기 (2분)
- 3. SQL Editor에서 테이블 생성 (1분)
- 4. Settings에서 URL + anon key 복사 (1분)
- 5. .env 파일에 붙여넣기

**시각 자료 제안:** 5단계 번호 매겨진 흐름도. 각 단계에 스크린샷 위치 표시. 총 소요 시간 "약 5분" 강조

**선생님 멘트:** "Supabase 설정은 5분이면 됩니다. 화면을 따라 하세요. 직접 PostgreSQL을 설치하면 40분 넘게 걸리는데, Supabase 덕분에 5분 만에 끝납니다."

## 슬라이드 6: 핵심 교육 포인트 — db.js만 바꾸면 끝! (3번째!)

**핵심 텍스트:**
- Step 1: db.js = Notion API
- Step 2: db.js = SQLite (better-sqlite3)
- Step 3: db.js = Supabase (@supabase/supabase-js)
- server.js 변경 없음! 화면 변경 없음!
- 3번이나 저장소를 바꿨는데, 한 글자도 안 바뀜 = 모듈화의 힘

**시각 자료 제안:** db.js를 중심에 놓고 3개의 화살표가 Notion/SQLite/Supabase로 뻗어나가는 다이어그램. server.js와 public/에 "3번 다 안 바뀜!" 도장

**선생님 멘트:** "이번이 세 번째입니다. 또 db.js만 바꿨더니 저장소가 바뀌었어요. server.js는 세 번 다 한 글자도 안 바뀌었습니다. 이것이 모듈화의 힘이에요. 실제 회사에서도 이렇게 설계합니다."

## 슬라이드 8: 3단계 전체 비교표

**핵심 텍스트:**
- | 항목 | Notion | SQLite | Supabase |
- | DB 위치 | 클라우드 | 내 컴퓨터 | 클라우드 |
- | 동시 접속 | 가능 | 불가(1명) | 가능(여러 명) |
- | 인터넷 없이 | 불가 | 가능 | 불가 |
- | API 키 | 필요(복잡) | 불필요 | 필요(간단) |
- | 실제 서비스 느낌 | 중간 | 낮음 | 가장 높음! |

**시각 자료 제안:** 3열 비교표. Supabase 열을 약간 크게 또는 테두리 강조. "가장 높음!" 항목에 별 아이콘

**선생님 멘트:** "3단계를 비교하면 각각 장단점이 있죠. Supabase가 실제 서비스에 가장 가까운 형태입니다. 여러 명이 동시에 접속할 수 있고, 데이터도 클라우드에 안전하게 저장됩니다."

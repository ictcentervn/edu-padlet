# Step 4: GitHub — 코드를 안전하게 저장하고 공유하기 — PPT 텍스트 원고

> 총 12장 | 선생님이 Canva/Google Slides/PowerPoint에서 슬라이드를 만들 때 참고할 원고입니다.

---
## 슬라이드 1: 코드를 잃어버리지 않으려면?

**핵심 텍스트:**
- USB를 잃어버린 적 있나요?
- 컴퓨터가 꺼져서 파일이 날아간 적은?
- "최종", "진짜최종", "진짜진짜최종" 파일이 생긴 적은?
- 코드도 마찬가지! GitHub이 해결합니다

**시각 자료 제안:** 파일 이름이 "보고서_최종.doc", "보고서_진짜최종.doc", "보고서_진짜진짜최종_v3.doc" 으로 늘어나는 일러스트 + 멘붕 이모지

**선생님 멘트:** "레포트를 USB에 담아 본 적 있죠? 잃어버리면? 컴퓨터가 꺼지면? 그리고 '최종', '진짜최종' 파일을 만들어 본 적 있죠? 코드도 마찬가지입니다. 오늘 배울 GitHub은 이 모든 문제를 해결해 줍니다."

---
## 슬라이드 2: Git = 게임 세이브 포인트

**핵심 텍스트:**
- Git = 코드 변경 기록을 저장하는 프로그램
- 비유: 게임 세이브 포인트
- 보스전 전에 세이브 -> 지면 되돌리기
- 코드를 망치면 -> 이전 커밋으로 되돌리기

**시각 자료 제안:** 게임 세이브 슬롯 UI 모양 다이어그램. "세이브 1: 첫 버전" -> "세이브 2: 기능 추가" -> "세이브 3: 버그 수정". 세이브 3에서 세이브 2로 되돌리는 화살표

**선생님 멘트:** "게임을 하다가 중요한 장면 전에 세이브하죠? 보스전에서 지면 세이브 포인트로 돌아가잖아요. Git도 같습니다. 코드에 세이브 포인트를 만들어 두면, 뭔가 잘못됐을 때 이전 상태로 돌아갈 수 있어요."

---
## 슬라이드 3: GitHub = 코드 구글 드라이브

**핵심 텍스트:**
- GitHub = Git 기록을 인터넷에 저장하는 서비스
- 비유: 코드 전용 구글 드라이브
- 어디서든 인터넷만 있으면 다운로드 가능
- 친구에게 링크 하나로 공유 가능
- 전 세계 개발자가 사용!

**시각 자료 제안:** 구글 드라이브 아이콘과 GitHub 아이콘 나란히 배치. 공통점: "인터넷 저장", "공유 가능", "어디서든 접근". GitHub만의 장점: "변경 기록", "되돌리기", "협업"

**선생님 멘트:** "구글 드라이브에 과제 폴더를 올려본 적 있죠? GitHub도 같은 원리입니다. 단, 코드 전용이라 더 강력해요. 누가 언제 무엇을 바꿨는지 다 기록됩니다."

---
## 슬라이드 4: 핵심 3단어 — add, commit, push

**핵심 텍스트:**
- git add = "이 파일을 세이브 목록에 넣겠다"
- git commit = "세이브 버튼 누르기" (메모 포함)
- git push = "인터넷(GitHub)에 업로드"
- 이 3단계만 기억하면 됩니다!

**시각 자료 제안:** 3칸 가로 흐름도. [add: 파일 선택] -> [commit: 세이브!] -> [push: 업로드!]. 각 칸에 직관적 아이콘 (체크박스/저장 아이콘/업로드 화살표)

**선생님 멘트:** "Git 명령어는 많지만, 오늘 기억할 건 3개뿐입니다. add, commit, push. 파일을 고른다, 세이브한다, 올린다. 이 세 단계만 기억하세요!"

---
## 슬라이드 5: .gitignore — 올리면 안 되는 것

**핵심 텍스트:**
- .gitignore = "이 파일은 GitHub에 올리지 마" 목록
- 올리면 안 되는 것:
  - .env (비밀 키가 들어 있음!)
  - node_modules/ (용량이 너무 큼)
  - padlet.db (개인 데이터)
- .gitignore에 적어두면 자동으로 무시

**시각 자료 제안:** 파일 목록에서 .env, node_modules/, padlet.db에 빨간 X 표시. 나머지 파일에 초록 체크. ".gitignore가 걸러줍니다" 라벨

**선생님 멘트:** "GitHub에 올리면 안 되는 파일이 있어요. 특히 .env 파일에는 비밀 키가 들어 있죠. 이걸 인터넷에 올리면 누구나 볼 수 있어서 위험합니다. .gitignore 파일에 적어두면 자동으로 제외됩니다."

---
## 슬라이드 6: 실습 — 첫 push!

**핵심 텍스트:**
- [ ] GitHub에 새 저장소(Repository) 만들기
- [ ] 터미널에서 git init
- [ ] git add . (파일 선택)
- [ ] git commit -m "첫 번째 커밋" (세이브)
- [ ] git push (업로드!)

**시각 자료 제안:** 체크리스트 형태. 각 단계 옆에 터미널 명령어를 코드 박스로 표시

**선생님 멘트:** "이제 직접 해 봅시다! GitHub에 저장소를 만들고, 우리 코드를 올려볼 거예요. 화면을 따라 하세요. 처음이라 긴장될 수 있는데, 한 번만 해 보면 금방 익숙해집니다."

## 슬라이드 7: GitHub에서 확인

**핵심 텍스트:**
- github.com에서 내 저장소 페이지 열기
- 코드가 올라가 있다!
- 파일 목록, 폴더 구조 확인
- 커밋 기록 확인 (언제, 무엇을 바꿨는지)

**시각 자료 제안:** GitHub 저장소 페이지 스크린샷 위치 표시. 파일 목록 + 커밋 히스토리 부분 강조

**선생님 멘트:** "GitHub 웹사이트를 열어 보세요. 여러분의 코드가 올라가 있죠? 파일도 보이고, 언제 커밋했는지 기록도 남아 있습니다. 이제 컴퓨터가 고장나도 코드가 안전합니다!"

## 슬라이드 8: 친구 코드 clone

**핵심 텍스트:**
- clone = GitHub에서 코드를 내 컴퓨터로 복사
- git clone [저장소 URL]
- 친구의 GitHub 주소만 알면 코드를 받을 수 있다
- 비유: 구글 드라이브에서 폴더 다운받기

**시각 자료 제안:** [내 컴퓨터] <--(clone)-- [GitHub 저장소] <--(push)-- [친구 컴퓨터]. 양방향 화살표 다이어그램

**선생님 멘트:** "이번엔 반대로 해 봅시다. 옆 친구의 GitHub 주소를 받아서 코드를 다운받아 보세요. git clone 명령어 하나면 됩니다."

---
## 슬라이드 9: Fork란?

**핵심 텍스트:**
- Fork = 남의 저장소를 내 계정에 복사
- clone과의 차이:
  - clone = 내 컴퓨터에 복사
  - fork = 내 GitHub 계정에 복사
- Fork하면 원본에 영향 없이 자유롭게 수정 가능
- 오픈소스 기여의 시작!

**시각 자료 제안:** 원본 저장소(친구) --(Fork)--> 내 저장소(나). 내 저장소에서 수정하는 모습. "원본은 안전!" 라벨

**선생님 멘트:** "Fork는 남의 저장소를 내 GitHub 계정에 통째로 복사하는 것입니다. 원본에 영향을 주지 않고 자유롭게 수정할 수 있어요. 실제 오픈소스 프로젝트에 기여할 때도 이 방법을 씁니다."

## 슬라이드 10: GitHub = 개발자 포트폴리오

**핵심 텍스트:**
- GitHub 프로필 = 개발자의 이력서
- 대입 자기소개서에 "제 GitHub 보세요"
- IT 회사 취업 시 GitHub 주소 필수
- 지금 만든 프로젝트가 첫 포트폴리오!

**시각 자료 제안:** GitHub 프로필 페이지 스크린샷 위치 표시 (잔디밭 그래프 강조). "여러분의 첫 포트폴리오!" 배지

**선생님 멘트:** "GitHub 프로필은 개발자의 이력서와 같습니다. 여러분이 오늘 올린 프로젝트가 바로 첫 번째 포트폴리오예요. 대입이나 취업 때 '제 GitHub을 보세요'라고 말할 수 있는 겁니다."

## 슬라이드 11: 오늘 배운 것 정리

**핵심 텍스트:**
- Git = 코드의 타임머신 (세이브 포인트)
- GitHub = 코드의 구글 드라이브 (인터넷 저장)
- 핵심 3단어: add -> commit -> push
- .gitignore로 비밀 파일 보호

**시각 자료 제안:** 4개 키워드 카드. Git(타임머신)/GitHub(구글 드라이브)/add-commit-push(3단계 화살표)/.gitignore(방패)

**선생님 멘트:** "오늘 핵심을 정리합니다. Git은 세이브 시스템, GitHub은 인터넷 저장소. 명령어는 add, commit, push 세 개만 기억하세요. 그리고 비밀 파일은 .gitignore로 보호합니다."

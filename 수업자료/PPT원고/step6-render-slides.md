## 슬라이드 1: 24시간 돌아가는 서비스를 만들려면?

**핵심 텍스트:**
- Codespaces는 끄면 앱이 멈춘다
- 실제 서비스(인스타, 유튜브)는 24시간 돌아간다
- 코드를 수정할 때마다 서버에 직접 올릴까?
- 아니요! push만 하면 자동 배포!

**시각 자료 제안:** Codespaces(끄면 멈춤, 정지 아이콘) vs 실제 서비스(24시간, 시계 아이콘) 비교. "어떻게?" 물음표

**선생님 멘트:** "Codespaces에서 서버를 실행해 봤죠. 하지만 Codespace를 끄면 앱이 멈춥니다. 인스타그램이나 유튜브가 그러면 안 되겠죠? 오늘은 24시간 돌아가는 진짜 서비스를 만들어 봅니다."

---
## 슬라이드 2: CI/CD란? — 출판사 비유

**핵심 텍스트:**
- CI/CD = 코드를 push하면 자동으로 배포
- 비유: 자동 출판 시스템
  - 작가가 원고 수정 -> 클라우드에 업로드
  - 자동 검수 -> 자동 인쇄 -> 새 판 출간
- 우리 버전: 코드 수정 -> git push -> 자동 배포!

**시각 자료 제안:** 2행 비교. 수동: 원고 수정 -> 전화 -> 편집자 -> 인쇄소 -> 출간 (느림). 자동(CI/CD): 원고 수정 -> 업로드 -> 자동 검수+인쇄 -> 출간 (빠름). 아래에 "git push -> Render가 자동 배포" 흐름

**선생님 멘트:** "CI/CD는 어려운 말처럼 들리지만 간단합니다. 코드를 push하면, 나머지는 전부 자동이에요. 마치 원고를 올리면 출판사가 알아서 책을 찍어주는 것과 같습니다."

---
## 슬라이드 3: Codespaces vs Render 비교

**핵심 텍스트:**
- | 항목 | Codespaces | Render |
- | 역할 | 연습장 (코딩+테스트) | 출판사 (실제 배포) |
- | 코딩 방식 | 브라우저에서 직접 | 로컬 VS Code에서 코딩 후 push |
- | 배포 방식 | 수동 (서버 직접 실행) | 자동 (push하면 배포) |
- | 비활동 시 | 5분 후 슬립 | 15분 후 슬립 |
- | 적합한 용도 | 교육, 프로토타입 | 실무, 실제 서비스 |

**시각 자료 제안:** 좌우 비교 카드. Codespaces: 연습장 아이콘 / Render: 출판사 아이콘. 핵심 차이에 강조색

**선생님 멘트:** "Codespaces는 연습장이고, Render는 출판사입니다. 연습장에서 초안을 쓰고, 출판사에서 책을 만들어 세상에 공개하는 거죠. 오늘은 출판사에 원고를 보내는 과정을 배웁니다."

## 슬라이드 5: 환경변수 설정 (Environment 메뉴)

**핵심 텍스트:**
- .env 파일은 GitHub에 올리지 않았다 (.gitignore)
- 서버에서 환경변수를 어떻게 알까?
- Render 대시보드 -> Environment 탭에서 직접 입력!
- SUPABASE_URL, SUPABASE_ANON_KEY 등
- 코드에 비밀 키를 넣지 않으니 안전!

**시각 자료 제안:** Render Environment 설정 화면 스크린샷 위치 표시. Key/Value 입력 표. ".env 파일과 같은 역할이지만 클라우드에서 관리" 설명 라벨

**선생님 멘트:** ".env 파일은 GitHub에 안 올렸죠? 그러면 Render 서버는 API 키를 어떻게 알까요? Render 대시보드에서 직접 입력해 줍니다. 이렇게 하면 비밀 키가 코드에 들어가지 않아서 안전합니다."

## 슬라이드 8: 자동 배포 체험 — push하면 끝!

**핵심 텍스트:**
- 코드를 수정한다 (예: 제목을 "우리 반 게시판"으로 변경)
- git add + git commit + git push
- Render가 자동으로 감지 -> 재배포!
- 1~2분 후 URL에 반영됨

**시각 자료 제안:** 4단계 흐름도. [코드 수정] -> [git push] -> [Render 자동 감지] -> [URL 업데이트!]. Render 단계에 톱니바퀴 아이콘(자동)

**선생님 멘트:** "이제 진짜 CI/CD를 체험해 봅시다. 코드를 조금 수정하고 push해 보세요. Render가 자동으로 감지해서 재배포합니다. 1~2분 후에 URL을 새로고침하면 변경 사항이 반영되어 있을 거예요."

## 슬라이드 9: 6단계 전체 여정 회고

**핵심 텍스트:**
- Step 1: DB 개념 (Notion) — "데이터를 어디에 저장하지?"
- Step 2: SQL 입문 (SQLite) — "DB를 직접 다뤄보자"
- Step 3: 클라우드 DB (Supabase) — "인터넷에서 DB를 쓰자"
- Step 4: 코드 관리 (GitHub) — "코드를 안전하게 관리하자"
- Step 5: 클라우드 코딩 (Codespaces) — "브라우저로 코딩하자"
- Step 6: 실무 배포 (Render) — "진짜 서비스를 만들자"

**시각 자료 제안:** 6단계 세로 타임라인. 각 단계에 아이콘 + 핵심 키워드. 전부 체크 표시. Step 1에서 Step 6으로 이어지는 성장 그래프

**선생님 멘트:** "6단계를 모두 완료했습니다! 처음에 Notion으로 시작해서, SQLite, Supabase로 DB를 배우고, GitHub으로 코드를 관리하고, Codespaces로 브라우저에서 코딩하고, 마지막으로 Render로 진짜 서비스를 배포했습니다. 대단해요!"

## 슬라이드 2: 이번엔 내 컴퓨터에 저장!

**핵심 텍스트:**
- Notion = 인터넷에 있는 장부 (인터넷 필요)
- SQLite = 내 컴퓨터에 있는 장부 (인터넷 불필요!)
- 비유: 구글 시트 vs 엑셀 파일

**시각 자료 제안:** 좌우 비교 다이어그램. 왼쪽: 구름 아이콘 + "Notion (클라우드)" / 오른쪽: 노트북 아이콘 + "padlet.db (내 컴퓨터)". 가운데에 VS 표시

**선생님 멘트:** "Notion은 인터넷 어딘가에 있는 장부였죠. 그런데 인터넷이 안 되면 어떻게 할까요? 오늘은 내 컴퓨터 안에 직접 장부를 만들어 봅니다. 파일 딱 하나로요!"

---
## 슬라이드 3: SQLite란?

**핵심 텍스트:**
- SQLite = 파일 하나가 곧 데이터베이스
- padlet.db 파일 하나에 모든 데이터 저장
- 설치 복잡한 과정 없음
- 인터넷 불필요, API 키 불필요

**시각 자료 제안:** 파일 아이콘(padlet.db) 하나를 크게 배치. 그 안에 표 형태 데이터가 들어 있는 모습. "이게 전부!" 라벨

**선생님 멘트:** "SQLite는 정말 간단합니다. padlet.db라는 파일 하나가 곧 데이터베이스예요. 이 파일 하나에 모든 카드 데이터가 들어갑니다."

---
## 슬라이드 4: SQL이란? — 4가지 명령어

**핵심 텍스트:**
- SQL = 데이터베이스에게 말하는 언어
- 비유: 도서관 사서에게 부탁하기
  - CREATE TABLE → "새 장부 만들어 주세요"
  - SELECT → "목록 보여 주세요"
  - INSERT → "이 기록 추가해 주세요"
  - DELETE → "이 기록 지워 주세요"

**시각 자료 제안:** 4칸짜리 격자. 각 칸에 SQL 명령어 + 사서 비유 + 아이콘 (노트/돋보기/연필/지우개)

**선생님 멘트:** "SQL은 데이터베이스에게 말하는 언어입니다. 도서관에서 사서에게 '이 책 찾아 주세요'라고 말하는 것처럼, SQL로 '이 데이터 보여줘', '이거 추가해줘' 라고 말합니다. 이 4가지만 알면 대부분의 작업이 가능해요."

---
## 슬라이드 5: Notion vs SQLite 비교표

**핵심 텍스트:**
- | 항목 | Notion | SQLite |
- | 저장 위치 | 클라우드 | 내 컴퓨터 (padlet.db) |
- | 인터넷 | 필요 | 불필요 |
- | API 키 | 필요 (복잡) | 불필요 (쉽다!) |
- | 속도 | 인터넷 속도에 따라 | 매우 빠름 |
- | 확인 방법 | Notion 웹사이트 | VS Code에서 바로 열기 |

**시각 자료 제안:** 깔끔한 비교표. 각 행에 아이콘 배치. SQLite 장점 항목에 초록 체크 강조

**선생님 멘트:** "Notion과 SQLite를 비교해 봅시다. SQLite는 인터넷도 필요 없고, API 키도 필요 없고, 속도도 빠릅니다. 대신 내 컴퓨터에만 있으니 친구가 접근하기는 어렵겠죠."

---
## 슬라이드 6: 핵심 교육 포인트 — db.js만 바꾸면 끝!

**핵심 텍스트:**
- server.js 안 바뀜!
- 화면(public/) 안 바뀜!
- 바뀐 것은 db.js 하나뿐!
- 이것이 좋은 설계(모듈화)의 힘

**시각 자료 제안:** 3개 파일 아이콘 나열. server.js와 public/에 "그대로" 도장. db.js에만 "변경!" 빨간 표시. 아래에 "저장 도구만 바꿨을 뿐, 나머지는 그대로" 문구

**선생님 멘트:** "여기가 오늘의 가장 중요한 포인트입니다. server.js도 안 바뀌고, 화면도 안 바뀌었어요. 바뀐 건 db.js 파일 하나뿐입니다. 데이터를 가져오는 도구만 바꿨을 뿐, 나머지는 그대로예요. 이게 좋은 설계입니다."

## 슬라이드 10: Step 1 vs Step 2 비교

**핵심 텍스트:**
- Step 1: 브라우저 -> 서버 -> Notion (클라우드)
- Step 2: 브라우저 -> 서버 -> padlet.db (내 컴퓨터)

- db.js만 바꾸면 저장소가 바뀐다!

**시각 자료 제안:** 좌우 구조도 비교. 브라우저와 서버 부분은 동일 색상, DB 부분만 다른 색상으로 강조

**선생님 멘트:** "두 앱을 나란히 놓으면 화면은 똑같아요. 하지만 데이터가 저장되는 곳이 다릅니다. 이것이 좋은 설계의 장점이에요. 저장소를 바꿔도 나머지 코드에 영향을 주지 않습니다."

## 슬라이드 11: 오늘 배운 것 정리

**핵심 텍스트:**
- SQLite = 파일 하나가 곧 데이터베이스
- SQL = DB에게 말하는 언어 (4가지 명령어)
- db.js만 바꾸면 저장소가 바뀐다 (좋은 설계)

**시각 자료 제안:** 4개 키워드 카드. 각각 아이콘 (파일/말풍선/퍼즐 조각/와이파이 끊김)

**선생님 멘트:** "오늘 핵심은 세 가지입니다. 첫째, SQLite는 파일 하나가 데이터베이스. 둘째, SQL은 4가지 명령어만 알면 됩니다. 셋째, db.js만 바꾸면 저장소가 바뀌는 좋은 설계를 배웠어요."

# 자주 막히는 문제 해결법 — Step 5: Glitch

> 문제가 생기면 아래에서 증상을 찾아보세요.
> 해결이 안 되면 Glitch 하단 **"Logs"** 탭을 열어서 에러 메시지를 확인하세요.

---

## 1. "Error: Could not find module 'express'" (모듈을 찾을 수 없음)

### 증상
Logs 탭에 빨간 글씨로 `Cannot find module 'express'` 또는 `Cannot find module '@supabase/supabase-js'` 등이 보임.

### 원인
`package.json`에 필요한 패키지가 없거나, npm install이 완료되지 않았습니다.

### 해결법
1. 왼쪽에서 `package.json` 클릭
2. `dependencies`에 필요한 패키지가 있는지 확인:
   ```json
   "dependencies": {
     "express": "^4.18.2",
     "@supabase/supabase-js": "^2.39.0",
     "dotenv": "^16.3.1"
   }
   ```
3. 없으면 추가하고 저장 → Glitch가 자동으로 npm install 실행
4. 하단 **"Terminal"** 버튼을 클릭해서 직접 입력할 수도 있습니다:
   ```
   npm install
   ```

## 2. 프로젝트가 깨어나지 않음 (슬립 모드)

### 증상
URL에 접속했는데 로딩이 오래 걸리거나 타임아웃이 발생함.

### 원인
Glitch 무료 플랜에서는 5분 동안 접속이 없으면 프로젝트가 잠듭니다 (슬립 모드).

### 해결법
1. **30초 정도 기다리세요.** 누군가 접속하면 자동으로 깨어납니다.
2. 그래도 안 되면 Glitch 편집기에서 프로젝트를 열어보세요. 편집기를 열면 프로젝트가 깨어납니다.
3. Logs 탭에 "Server started" 또는 "서버 시작" 메시지가 보이면 깨어난 것입니다.

> 💡 교사 메모: 수업 시작 5분 전에 학생들의 프로젝트 URL에 한 번씩 접속해두면 수업 시작 시 슬립 문제를 피할 수 있습니다.

## 3. .env 환경변수가 적용 안 됨

### 증상
앱은 열리지만 카드 목록이 안 나오거나, "Supabase error" 메시지가 보임.

### 원인
.env 변수가 잘못 입력되었습니다.

### 해결법

**Glitch에서 .env를 설정하는 올바른 방법:**
1. 왼쪽 파일 목록에서 **".env"** 클릭
2. Glitch 전용 편집기가 열립니다 (일반 텍스트 편집기와 다릅니다!)
3. 변수 이름과 값을 정확히 입력

**자주 하는 실수:**

| 잘못된 입력 | 올바른 입력 |
|------------|-----------|
| `SUPABASE_URL = https://...` (공백) | `SUPABASE_URL=https://...` |
| `SUPABASE_URL="https://..."` (따옴표) | `SUPABASE_URL=https://...` |
| `supabase_url=...` (소문자) | `SUPABASE_URL=...` (대문자) |

수정 후 Glitch가 자동으로 서버를 재시작합니다. Logs 탭에서 에러가 사라졌는지 확인하세요.

## 4. GitHub Import 실패

### 증상
"Import from GitHub"을 했는데 에러가 발생하거나 프로젝트가 비어 있음.

### 원인
GitHub 저장소가 private(비공개)이거나 URL이 잘못되었습니다.

### 해결법
1. **저장소가 public인지 확인:** GitHub.com에서 저장소 페이지에 접속 → Settings → General → "Change repository visibility"에서 Public 확인
2. **URL 형식 확인:**
   ```
   올바른: https://github.com/내이름/edu-padlet
   잘못된: https://github.com/내이름/edu-padlet.git
   잘못된: github.com/내이름/edu-padlet (https:// 빠짐)
   ```
3. **그래도 안 되면:** 방법 B(직접 파일 올리기)를 사용하세요.

## 5. 포트 에러 — "Error: listen EADDRINUSE" 또는 앱이 안 열림

### 증상
Logs에 포트 관련 에러가 보이거나, Preview에서 "There was an error" 메시지가 나옴.

### 원인
Glitch에서는 고정 포트(3000)를 사용할 수 없습니다. `process.env.PORT`를 사용해야 합니다.

### 해결법
`server.js`에서 포트 설정 부분을 찾아 수정:

```javascript
// 수정 전
const PORT = 3000;

// 수정 후
const PORT = process.env.PORT || 3000;
```

`app.listen` 부분도 확인:
```javascript
app.listen(PORT, () => {
  console.log(`서버 시작: http://localhost:${PORT}`);
});
```

## 6. 미리보기가 빈 화면 (흰 화면)

### 증상
Preview를 열었는데 아무것도 안 보임.

### 원인
여러 가지 가능성이 있습니다.

### 해결법 (순서대로 확인)

1. **Logs 탭 확인:** 빨간 에러 메시지가 있는지 확인. 에러가 있다면 해당 에러를 먼저 해결.
2. **서버가 시작되었는지 확인:** Logs에 "서버 시작" 또는 "listening on port" 메시지가 보여야 합니다.
3. **public 폴더 확인:** `public/index.html` 파일이 있는지 확인. 파일 이름에 오타가 없는지 확인.
4. **Express 정적 파일 설정 확인:** `server.js`에 아래 줄이 있는지 확인:
   ```javascript
   app.use(express.static('public'));
   ```
5. **Preview URL 확인:** `Preview in a new window`로 열었는지 확인. 주소가 `https://프로젝트명.glitch.me`인지 확인.

## 7. 협업 중 코드 충돌

### 증상
두 명이 동시에 같은 파일을 수정했는데 한 사람의 변경이 사라짐.

### 원인
Glitch는 동시 편집을 지원하지만, **같은 줄을 동시에 수정하면** 나중에 저장한 내용이 우선 적용됩니다.

### 해결법
1. **역할 분담:** 파일을 나눠서 작업하세요.
   - A: `public/index.html` 담당
   - B: `public/style.css` 담당
2. **같은 파일을 수정해야 한다면:** 수정할 부분을 미리 정하고, 다른 줄을 수정하세요.
3. **변경이 사라졌다면:** Ctrl+Z로 되돌리거나, Glitch 상단의 "Rewind" 기능으로 이전 상태를 복원할 수 있습니다.

> 💡 교사 메모: 협업 실습 시 "한 명은 HTML, 한 명은 CSS"처럼 파일 단위로 역할을 나누면 충돌을 방지할 수 있습니다.

## 8. 무료 사용량 제한 (월 1,000시간)

### 증상
프로젝트가 갑자기 안 열리거나 "Project has been suspended" 메시지가 보임.

### 원인
Glitch 무료 플랜은 월 1,000시간까지 사용할 수 있습니다.

### 해결법
1. 보통 교육용으로는 충분합니다. 한 프로젝트가 하루 24시간 돌아도 월 720시간입니다.
2. 슬립 모드 덕분에 실제 사용 시간은 훨씬 적습니다.
3. 만약 제한에 걸렸다면:
   - 사용하지 않는 프로젝트를 삭제하세요
   - 다음 달 1일에 사용량이 초기화됩니다

## 9. package.json의 start 스크립트 문제

### 증상
Logs에 "No start script found" 또는 서버가 시작되지 않음.

### 원인
`package.json`에 `start` 스크립트가 없거나 잘못되어 있습니다.

### 해결법
`package.json`을 열고 `scripts` 부분을 확인:

```json
{
  "scripts": {
    "start": "node server.js"
  }
}
```

`server.js`가 아닌 다른 파일이 메인 파일이라면 그에 맞게 수정하세요.

## 10. Glitch 편집기가 느리거나 먹통

### 증상
코드를 입력해도 반응이 느리거나, 편집기가 멈춤.

### 해결법
1. **브라우저 새로고침** (F5 또는 Ctrl+R)
2. 열려 있는 **다른 탭을 줄이세요.** 브라우저 메모리 부족일 수 있습니다.
3. **크롬 시크릿 모드**에서 Glitch를 열어보세요. 브라우저 확장 프로그램이 충돌할 수 있습니다.
4. 그래도 안 되면 **다른 브라우저**(Edge, Firefox)를 사용해보세요.

## 문제 해결 기본 순서

어떤 문제든 이 순서로 확인하면 대부분 해결됩니다:

```
① Logs 탭 열기 → 에러 메시지 확인
     │
     ▼
② 에러 메시지를 읽고 위 목록에서 찾기
     │
     ▼
③ 못 찾겠으면 → 에러 메시지를 Claude.ai에 붙여넣기:
   "Glitch에서 이런 에러가 나왔어. 어떻게 해결해?"
     │
     ▼
④ 그래도 안 되면 → 선생님 호출!
```

# Glitch — 브라우저에서 코딩하고 바로 공유하기

> **대상:** 중학교 1학년 ~ 고등학교 3학년 (수준별 표시 포함)

---

## 오늘 배울 내용

> 브라우저 하나만으로 우리 패들렛 앱을 **인터넷에 올리고**, 친구에게 링크를 보내고, **함께 코드를 수정**합니다.

---

## 생각해보기

학교 컴퓨터실에 갔는데 Node.js가 설치되어 있지 않습니다.

- 선생님께 설치해달라고 하면 "관리자 권한이 없어서 안 돼요"라는 답이 돌아옵니다.
- 집에 컴퓨터가 없는 친구는 어떻게 하죠?
- 크롬북만 있는 학교는요?

**설치 안 해도 코딩할 수 있는 방법이 있다면?**

오늘 배울 **Glitch**가 바로 그 답입니다!

---

## 핵심 개념 1: Glitch — 브라우저 안의 코딩 환경

```
┌─────────────────────────────────────────────────┐
│  Glitch (glitch.com)                            │
│                                                 │
│  ┌──────────┐ ┌─────────────────────────────┐  │
│  │ 파일 목록 │ │                             │  │
│  │          │ │   코드 편집기                 │  │
│  │ server.js│ │                             │  │
│  │ db.js    │ │   const express =           │  │
│  │ public/  │ │     require('express');      │  │
│  │ .env     │ │   const app = express();     │  │
│  │          │ │   ...                       │  │
│  └──────────┘ └─────────────────────────────┘  │
│                                                 │
│  [Logs]  [Terminal]  [Preview ▶ 내앱.glitch.me] │
└─────────────────────────────────────────────────┘
```

### 비유로 이해하기: Google Docs

Google Docs를 써본 적 있나요?

- Word를 설치하지 않아도 문서를 작성할 수 있죠.
- 링크를 공유하면 친구도 같이 편집할 수 있고요.
- 저장 버튼을 안 눌러도 자동으로 저장됩니다.

**Glitch는 코딩판 Google Docs**입니다.

| Google Docs | Glitch |
|-------------|--------|
| Word 설치 필요 없음 | Node.js 설치 필요 없음 |
| 브라우저에서 문서 작성 | 브라우저에서 코드 작성 |
| 링크로 공유 | 링크로 공유 |
| 동시 편집 가능 | 동시 편집 가능 |
| 자동 저장 | 자동 저장 + 자동 실행 |

## 핵심 개념 2: localhost vs Glitch

지금까지 우리는 이렇게 했습니다:

```
지금까지:   내 컴퓨터에서 node server.js 실행
            → localhost:3000으로 접속
            → 나만 볼 수 있음
            → 컴퓨터를 끄면 서버도 꺼짐

Glitch:     브라우저에서 코드 작성
            → 프로젝트명.glitch.me 주소가 자동 생성
            → 누구나 볼 수 있음!
            → 내 컴퓨터를 꺼도 서버가 살아 있음
```

이것이 바로 **배포(Deploy)** 입니다.

배포란 "내 앱을 인터넷에 올려서 누구나 쓸 수 있게 만드는 것"입니다.

## 핵심 개념 3: Glitch 화면 구성

```
┌─────────────────────────────────────────────────────────┐
│  🐟 프로젝트명 ▾       [Share]  [Preview ▾]  [Settings]  │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│  파일     │           코드 편집 영역                      │
│  목록     │                                              │
│          │   여기서 코드를 수정하면                        │
│ server.js│   자동으로 서버가 재시작됩니다!                  │
│ db.js    │                                              │
│ public/  │                                              │
│ .env     │                                              │
│          │                                              │
├──────────┴──────────────────────────────────────────────┤
│  [Logs]  서버 로그가 여기에 표시됩니다                     │
└─────────────────────────────────────────────────────────┘
```

| 영역 | 역할 |
|------|------|
| **왼쪽: 파일 목록** | 프로젝트의 모든 파일을 보여줍니다 |
| **가운데: 코드 편집기** | 코드를 수정하는 곳. 수정하면 자동 저장 + 자동 재시작 |
| **상단: Preview 버튼** | 내 앱을 미리보기. 고유 URL이 부여됩니다 |
| **하단: Logs** | 서버가 출력하는 메시지를 보여줍니다 (에러 확인용) |
| **.env 파일** | 비밀 키를 저장하는 곳 (다른 사람에게 자동으로 숨겨짐!) |

## Glitch 무료 플랜 알아두기

- 무료로 사용 가능합니다.
- **5분 동안 아무도 접속하지 않으면** 프로젝트가 잠듭니다 (슬립 모드).
- 누군가 접속하면 30초 안에 다시 깨어납니다.
- 월 1,000시간까지 사용 가능 — 수업용으로 충분합니다.

---

## 이번 단계에서 바뀐 것

| 항목 | 변경 내용 |
|------|---------|
| server.js | 포트 설정 1줄만 수정 (`process.env.PORT`) |
| db.js | 변경 없음 |
| public/ | 변경 없음 |
| .env | Glitch 전용 편집기에서 입력 (파일 내용 동일) |

**코드는 거의 그대로입니다.** 바뀌는 것은 "어디서 실행하느냐"뿐입니다.

> 비유: 같은 레시피(코드)를 집 주방(localhost)에서 만들다가, 식당 주방(Glitch)으로 옮기는 것과 같습니다. 레시피는 안 바뀌고, 주방만 바뀝니다.

# Step 5: Glitch 수업 지도안

**대상:** 중학교 1학년 ~ 고등학교 3학년 (전체)
**시간:** 2차시 x 45분
**준비물:** 노트북 또는 데스크톱 (크롬북도 가능), 인터넷, 크롬 브라우저
**계정:** Glitch 계정 (이메일 또는 GitHub), Supabase 계정 (Step 3에서 생성)

> 💡 교사 메모: 이 단계의 최대 장점은 **학교 컴퓨터에 아무것도 설치하지 않아도 된다**는 것입니다. Node.js, VS Code, Git 모두 불필요합니다. 브라우저만 있으면 됩니다. 크롬북 환경의 학교에서도 진행 가능합니다.

---

## 사전 준비 (교사)

- [ ] glitch.com 접속이 학교 네트워크에서 차단되지 않는지 확인
- [ ] 교사용 Glitch 프로젝트를 미리 하나 만들어 시연 준비
- [ ] 학생들의 Supabase URL + ANON KEY 목록 확인 (Step 3에서 받은 것)
- [ ] (선택) Step 4를 완료한 학생과 안 한 학생 파악 → Import 방법 분기

## 1차시 — Glitch 소개 + 배포 (45분)

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~5분 | 도입: 질문 던지기 | "학교 컴퓨터에 Node.js를 설치할 수 없다면?" / "만든 앱을 친구에게 보여주려면?" | 동기유발 |
| 5~10분 | Glitch 소개 | Google Docs 비유로 설명. localhost vs Glitch 비교 다이어그램 보여주기 | 00_개념교안.md 활용 |
| 10~15분 | 교사 시연 | Glitch에서 프로젝트 import → .env 설정 → Preview → URL 공유까지 전체 과정을 보여줌 | 화면 공유 필수 |
| 15~35분 | 학생 실습 | 학생 각자 Glitch 가입 → 프로젝트 import (방법 A 또는 B) → .env 설정 → 포트 수정 → Preview 확인 | 03_학생실습지침서.md Step A~D |
| 35~40분 | URL 공유 체험 | 옆 친구에게 내 Glitch URL 보내기 → 친구 앱에 카드 추가해보기 | 🟢 성공 경험! |
| 40~45분 | 정리 | 오늘 한 일 3줄 정리: (1) Glitch 가입 (2) 앱 배포 (3) URL 공유 | 미완성 학생은 2차시에 이어서 |

> 💡 교사 메모:
> - 가장 많이 막히는 곳: .env 입력 시 따옴표를 넣는 실수 (따옴표 없이 값만 입력)
> - GitHub import가 실패하면 방법 B(직접 올리기)로 안내
> - 포트 수정을 빠뜨리는 학생이 많음 → 반드시 확인

## 2차시 — 실시간 협업 + 커스터마이징 (45분)

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~5분 | 복습 | "지난 시간에 만든 앱 URL을 말해보세요" → 프로젝트 다시 열기 | 슬립 상태면 30초 기다리기 |
| 5~10분 | 협업 소개 | "Google Docs처럼 여러 명이 동시에 코드를 편집할 수 있어요" | 교사가 시연: 초대 링크 생성 |
| 10~25분 | 2인 1조 협업 실습 | 짝꿍에게 편집 초대 → 한 명은 HTML 수정, 한 명은 CSS 수정 → 실시간으로 서로의 커서가 보이는 것 확인 | 03_학생실습지침서.md Step F |
| 25~35분 | 코드 수정 → 즉시 반영 체험 | style.css에서 배경색 변경 → 저장 즉시 반영 확인 → 카드 색상 변경 → 폰트 변경 | 🟡 기본 ~ 🔴 심화 |
| 35~40분 | 심화: 디자인 커스터마이징 | 바이브코딩 프롬프트를 활용해서 나만의 디자인 만들기 (Claude.ai에게 CSS 물어보기) | 04_바이브코딩프롬프트집.md 활용 |
| 40~45분 | 발표 + 정리 | 2~3팀 결과 URL 공유. "브라우저만으로 코딩, 배포, 협업을 했다!" 정리 | 전체 공유 |

> 💡 교사 메모:
> - 협업 시 같은 줄을 동시에 수정하면 나중에 저장한 내용이 우선 적용됩니다. 다른 부분을 나눠서 수정하도록 안내하세요.
> - 시간이 부족하면 심화(디자인 커스터마이징)는 과제로 전환 가능합니다.
> - 빠른 학생에게는 바이브코딩 프롬프트집의 🔴 심화 과제를 안내하세요.

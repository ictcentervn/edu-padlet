# Step 5: Glitch 실습 지침서

> **대상:** 중학교 1학년 ~ 고등학교 3학년
> 순서대로 따라하면 반드시 동작합니다!

---

## 시작 전 체크리스트

- [ ] 인터넷에 연결되어 있다
- [ ] 크롬(Chrome) 브라우저가 있다
- [ ] Step 3에서 받은 Supabase URL과 ANON KEY를 메모해두었다

**이번 Step에서는 Node.js, VS Code, Git 설치가 필요 없습니다!**

---

## Step A: Glitch 가입하기

### A-1. Glitch 접속

1. 브라우저 주소창에 입력:
   ```
   glitch.com
   ```
2. 오른쪽 상단 **"Sign in"** 클릭

### A-2. 로그인 방법 선택

**GitHub 계정이 있는 경우 (Step 4를 한 경우):**
- "Sign in with GitHub" 클릭
- GitHub 로그인 → 권한 허용

**GitHub 계정이 없는 경우:**
- "Sign in with Email" 클릭
- 이메일 주소 입력 → 이메일로 온 인증 코드 입력

> ✅ 대시보드 화면이 보이면 성공!

---

## Step B: 프로젝트 가져오기

### 방법 1: GitHub에서 Import (Step 4를 완료한 경우) — 🟢

1. **"New project"** 클릭
2. **"Import from GitHub"** 선택
3. GitHub 저장소 URL 입력:
   ```
   https://github.com/내이름/edu-padlet
   ```
4. **"OK"** 클릭
5. 잠시 기다립니다 (파일 복사 + npm install 자동 실행)

> ✅ 왼쪽에 server.js, db.js, public/ 폴더가 보이면 성공!

### 방법 2: 직접 파일 올리기 (Step 4를 안 한 경우) — 🟡

1. **"New project"** → **"glitch-hello-node"** 선택
2. 왼쪽 파일 목록에서 **"New File"** 클릭
3. 아래 파일들을 하나씩 만들고, 내 컴퓨터의 코드를 복사-붙여넣기:
   - `db.js`
   - `public/index.html` (폴더가 자동 생성됨)
   - `public/style.css`
   - `public/app.js`
4. `server.js`를 클릭해서 내용을 내 코드로 교체
5. `package.json`을 클릭해서 dependencies 확인:
   ```json
   {
     "name": "edu-padlet",
     "version": "1.0.0",
     "scripts": {
       "start": "node server.js"
     },
     "dependencies": {
       "express": "^4.18.2",
       "@supabase/supabase-js": "^2.39.0",
       "dotenv": "^16.3.1"
     }
   }
   ```

> ✅ Glitch가 자동으로 npm install을 실행합니다. Logs 탭에서 확인하세요.

## Step C: 환경변수 설정 (.env)

1. 왼쪽 파일 목록에서 **".env"** 클릭
2. Glitch 전용 환경변수 편집기가 열립니다
3. 아래 내용을 입력:

```
SUPABASE_URL=여기에_수파베이스_URL_붙여넣기
SUPABASE_ANON_KEY=여기에_수파베이스_키_붙여넣기
```

**주의사항:**
- 따옴표(`"` 또는 `'`)를 넣지 마세요!
- `=` 앞뒤에 공백을 넣지 마세요!
- 줄바꿈으로 변수를 구분합니다

**올바른 예:**
```
SUPABASE_URL=https://abcdefgh.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOi...
```

**잘못된 예:**
```
SUPABASE_URL = "https://abcdefgh.supabase.co"   ← 공백, 따옴표 잘못됨
```

> ✅ .env 설정 후 Glitch가 자동으로 서버를 재시작합니다

## Step D: 포트 수정 + 앱 확인

### D-1. 포트 설정 수정

왼쪽에서 `server.js`를 클릭합니다.

아래 부분을 찾으세요:
```javascript
const PORT = 3000;
```

이것을 아래처럼 수정합니다:
```javascript
const PORT = process.env.PORT || 3000;
```

**이것이 Step 5에서 유일하게 바꾸는 코드 1줄입니다!**

### D-2. 미리보기 열기

1. 상단 **"Preview"** 버튼 클릭
2. **"Preview in a new window"** 선택
3. 새 탭에서 앱이 열립니다!

주소를 확인하세요:
```
https://프로젝트명.glitch.me
```

### D-3. 동작 확인

- [ ] 앱 화면이 보인다
- [ ] 카드 추가 버튼을 누르면 카드가 생긴다
- [ ] 페이지를 새로고침해도 카드가 남아 있다 (Supabase에 저장됨)

> ✅ 여기까지 되면 배포 완료! 축하합니다!
>
> 🆘 화면이 안 보이나요? → [05_트러블슈팅.md] 참조

## Step E: 친구에게 URL 공유하기

1. 주소창의 URL을 복사합니다:
   ```
   https://프로젝트명.glitch.me
   ```
2. 카카오톡이나 문자로 친구에게 보냅니다
3. 친구가 링크를 클릭하면 내 앱이 열립니다!
4. 친구가 카드를 추가하면...
5. **내 화면에서도 카드가 보입니다!** (같은 Supabase DB를 사용하니까)

> ✅ 친구가 추가한 카드가 내 화면에서 보이면 성공!

## Step F: 협업 실습 — 친구와 함께 코드 수정하기

### F-1. 편집자 초대

1. 왼쪽 상단 **프로젝트 이름** 클릭
2. **"Invite Others to Edit"** 선택
3. 초대 링크가 복사됩니다
4. 이 링크를 짝꿍에게 보내세요

### F-2. 동시 편집 체험

짝꿍이 초대 링크를 클릭하면:
- 같은 프로젝트의 코드가 보입니다
- 짝꿍의 커서가 내 화면에 보입니다 (Google Docs처럼!)

**실습:** 역할을 나눠서 동시에 수정해보세요.

| 역할 | 할 일 |
|------|------|
| 나 | `public/index.html`에서 제목 수정 |
| 짝꿍 | `public/style.css`에서 배경색 변경 |

수정 후 Preview를 새로고침하면 **두 사람의 변경 내용이 모두 반영**됩니다!

> ✅ 두 명이 동시에 코드를 수정하고, 결과가 바로 보이면 성공!

## Step G: 심화 — 디자인 커스터마이징 (🔴)

**Glitch에서는 코드를 수정하면 즉시 반영됩니다.** 이 장점을 활용해서 디자인을 바꿔보세요!

### G-1. 배경색 바꾸기

`public/style.css`에서:
```css
body {
  background-color: #f0f0f0;  /* 이 값을 바꿔보세요 */
}
```

추천 색상:
- `#fff5e6` — 따뜻한 크림색
- `#e6f3ff` — 시원한 하늘색
- `#f0ffe6` — 산뜻한 연두색
- `#ffe6f0` — 부드러운 분홍색

### G-2. 제목 바꾸기

`public/index.html`에서 "우리 반 패들렛" 부분을 찾아서 원하는 이름으로 변경:
- "3학년 2반 아이디어 보드"
- "코딩반 프로젝트 벽"
- "수학 질문 게시판"

### G-3. 더 도전하고 싶다면?


AI가 준 CSS 코드를 Glitch에 붙여넣으면 **바로 결과**를 확인할 수 있어요!

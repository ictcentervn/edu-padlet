# 도움말 — 막혔을 때 여기를 봐요

> **대상:** 중학교 1학년 ~ 고등학교 2학년
> 에러가 나면 당황하지 마세요. 아래에서 같은 증상을 찾아보세요!

---

## 1. node: command not found

### 증상
터미널에 `node server.js`를 입력하면:
```
zsh: command not found: node
```
또는
```
'node'은(는) 내부 또는 외부 명령...으로 인식되지 않습니다.
```

### 원인
Node.js가 설치되어 있지 않습니다.

### 해결법
1. https://nodejs.org 에 접속합니다
2. **LTS** 버전(왼쪽 초록색 버튼)을 클릭해서 다운로드합니다
3. 다운로드된 파일을 실행해서 설치합니다 (모든 설정 기본값 OK)
4. **터미널을 닫고 다시 엽니다** (중요!)
5. `node --version`을 입력해서 버전이 나오면 성공

> ⚠️ 설치 후 반드시 터미널을 **새로 열어야** 합니다. 기존 터미널에서는 인식 안 됩니다.

## 2. Cannot find module 'express'

### 증상
`node server.js`를 실행하면:
```
Error: Cannot find module 'express'
```

### 원인
`npm install`을 아직 실행하지 않았거나, 다른 폴더에서 실행했습니다.

### 해결법
1. 터미널에서 현재 위치를 확인합니다:
   ```bash
   ls
   ```
   `package.json` 파일이 보여야 합니다.

2. 보이지 않으면 올바른 폴더로 이동합니다:
   ```bash
   cd step1-notion
   ```

3. npm install을 실행합니다:
   ```bash
   npm install
   ```

4. 다시 서버를 실행합니다:
   ```bash
   node server.js
   ```

## 3. EADDRINUSE: address already in use :::3000

### 증상
`node server.js`를 실행하면:
```
Error: listen EADDRINUSE: address already in use :::3000
```

### 원인
이미 다른 프로그램(또는 이전에 실행한 서버)이 3000번 포트를 사용하고 있습니다.

### 해결법

**방법 1: 이전 서버 종료**
- 다른 터미널 탭에서 서버가 실행 중이라면 → 그 터미널에서 `Ctrl + C` 누르기

**방법 2: 강제 종료 (Mac)**
```bash
lsof -i :3000
kill -9 [PID번호]
```

**방법 2: 강제 종료 (Windows)**
```bash
netstat -ano | findstr :3000
taskkill /PID [PID번호] /F
```

그 후 다시 `node server.js` 실행

## 4. .env 파일이 없다는 에러

### 증상
서버 실행 시 아래 메시지가 나옵니다:
```
❌ 환경변수가 설정되지 않았습니다!
```

### 원인
`.env` 파일이 없거나, 파일 안의 값이 비어 있습니다.

### 해결법

1. `.env.example` 파일을 `.env`로 복사합니다:

   **Mac:**
   ```bash
   cp .env.example .env
   ```

   **Windows:**
   ```bash
   copy .env.example .env
   ```

2. `.env` 파일을 열어서 실제 값을 입력합니다:
   ```
   NOTION_API_KEY=secret_실제키값
   NOTION_DATABASE_ID=실제DB아이디
   ```

3. 파일을 저장(Ctrl+S)하고 서버를 다시 실행합니다.

> ⚠️ `.env` 파일 이름이 `.env.txt`가 되지 않았는지 확인하세요! Windows에서 자주 발생합니다.

### .env 파일이 폴더에서 안 보여요!

`.env`처럼 점(`.`)으로 시작하는 파일은 **숨김파일**이라 일반 폴더(Finder/탐색기)에서 보이지 않습니다.

**확인 방법:**
- **Mac Finder:** `Cmd + Shift + .` (점) 키를 누르면 숨김파일 표시됨
- **Windows 탐색기:** 상단 "보기" → "숨겨진 항목" 체크
- **VS Code:** 왼쪽 파일 목록에서는 **항상 보입니다** (숨김파일도 표시)
- **터미널:** `ls -a` 명령어로 확인 가능 (`ls`만 치면 안 보임)

> 💡 **팁:** `.env` 파일은 VS Code에서 편집하는 게 가장 편합니다. Finder/탐색기에서 찾을 필요 없어요!

## 5. API 키가 `secret_`이 아니라 `ntn_`으로 시작해요

### 증상
Notion에서 복사한 API 키가 `secret_`이 아니라 `ntn_`으로 시작합니다.

### 원인
Notion이 2024년 이후 API 키 형식을 변경했습니다.
- **이전:** `secret_AbCdEf...`
- **현재:** `ntn_44461653...`

### 해결법
**정상입니다!** `ntn_`으로 시작해도 그대로 `.env`에 넣으면 됩니다:
```
NOTION_API_KEY=ntn_44461653370b1Fage...
```

## 6. DB ID에 페이지 이름이 붙어 있어요

### 증상
URL에서 DB ID를 복사했는데 이런 형태입니다:
```
Paddle-339a5a1efd7a808fbba7fcdfe314865c
```

### 원인
Notion URL에 페이지 이름이 포함되어 있습니다:
```
https://notion.so/내이름/Paddle-339a5a1efd7a808f...?v=...
                        ^^^^^^^
                        이 부분은 페이지 이름!
```

### 해결법
**마지막 32자리만 복사하세요.** 하이픈(`-`) 뒤의 영숫자 부분:
```
❌ Paddle-339a5a1efd7a808fbba7fcdfe314865c  (페이지 이름 포함)
✅ 339a5a1efd7a808fbba7fcdfe314865c          (32자리 ID만)
```

> 💡 **쉬운 방법:** URL에서 `?v=` 바로 앞의 32글자를 세어서 복사하세요. 영어 소문자 + 숫자로만 이루어져 있습니다.

## 7. DB ID 뒤에 `?v=...`가 붙어 있어요

### 증상
`.env`에 넣은 DB ID가 이렇게 생겼습니다:
```
NOTION_DATABASE_ID=339a5a1efd7a80e6...?v=339a5a1efd7a80f1...
```

### 원인
URL 전체를 복사하면 `?v=` 이후의 **뷰 ID**까지 포함됩니다.

### 해결법
`?v=` **앞부분 32자리만** 남기고 나머지를 삭제하세요:
```
❌ 339a5a1efd7a80e6ada5c934e427b7d1?v=339a5a1efd7a80f1af9a000c45462bae
✅ 339a5a1efd7a80e6ada5c934e427b7d1
```

## 8. "Provided ID is a page, not a database" 에러

### 증상
서버 실행 시:
```
Provided ID ... is a page, not a database
```

### 원인
Notion에서 "인라인 데이터베이스"를 만들면 URL에 보이는 ID는 **페이지 ID**입니다. 데이터베이스 ID가 아니에요.

### 해결법
데이터베이스를 **Full page**로 열어야 합니다:
1. Notion에서 DB 표 위에 마우스 올리기
2. 표 왼쪽 상단 **"⤢ Open as full page"** 클릭
3. URL이 바뀜 → 새 URL에서 `?v=` 앞 32자리 복사

또는 처음부터 **Full page database**로 만들기:
1. "+ Add a page" → 본문에서 `/database` 입력
2. **"Database - Full page"** 선택 (inline이 아닌!)

## 9. Notion "object_not_found" 에러

### 증상
서버 실행 또는 카드 추가 시:
```
APIResponseError: Could not find database with ID: ...
```
또는 응답에 `object_not_found` 포함

### 원인
Integration이 데이터베이스에 연결되지 않았습니다.

### 해결법
1. Notion에서 **패들렛 DB** 페이지를 엽니다
2. 우측 상단 **"..."** 클릭
3. **"Connections"** 클릭
4. **"우리반 패들렛"** Integration을 찾아서 선택
5. **"Confirm"** 클릭

```
[패들렛 DB]  ←연결→  [우리반 패들렛 Integration]
이 연결이 반드시 되어 있어야 합니다!
```

> 💡 이 단계를 빠뜨리는 학생이 가장 많습니다. 꼭 확인하세요!

## 6. Notion "unauthorized" 에러

### 증상
```
APIResponseError: API token is invalid.
```
또는 `unauthorized` 에러

### 원인
`.env` 파일의 `NOTION_API_KEY` 값이 잘못되었습니다.

### 해결법
1. https://www.notion.so/my-integrations 에 접속합니다
2. 만들어둔 Integration을 클릭합니다
3. **API 키를 다시 복사**합니다 (`secret_`로 시작하는 문자열 전체)
4. `.env` 파일을 열어서 정확하게 붙여넣습니다

확인 사항:
- `=` 앞뒤에 공백이 없는지
- 따옴표(`"`, `'`)가 들어가지 않았는지
- 키 앞뒤에 불필요한 공백이 없는지
- `secret_`부분이 포함되어 있는지

## 7. 화면이 흰 화면 (아무것도 안 보임)

### 증상
http://localhost:3000 에 접속했는데 아무것도 안 보입니다. 하얀 화면만 나옵니다.

### 원인
서버가 실행 중이지 않거나, 주소를 잘못 입력했을 수 있습니다.

### 해결법

**1단계: 서버가 실행 중인지 확인**
- 터미널에 "서버가 시작되었습니다!" 메시지가 있는지 확인
- 없으면 `node server.js` 실행

**2단계: 주소 확인**
- 주소창에 **정확하게** `http://localhost:3000` 입력
- `https`가 아니라 `http` (s 없음!)
- `localhost` 철자 확인 (localhos**t**)

**3단계: public 폴더 확인**
- `public/index.html` 파일이 존재하는지 확인
- 없으면 프로젝트 파일이 불완전한 것. 선생님께 다시 받으세요.

## 8. 카드가 추가되는데 화면에 안 보임

### 증상
카드 추가 버튼을 눌렀는데 화면에 새 카드가 나타나지 않습니다. 하지만 Notion DB에는 데이터가 들어가 있습니다.

### 원인
프론트엔드에서 카드 목록을 다시 불러오지 않고 있습니다.

### 해결법

**방법 1: 브라우저 새로고침**
- `F5` 키 또는 `Ctrl + R` (Mac: `Cmd + R`)

**방법 2: 강제 새로고침**
- `Ctrl + Shift + R` (Mac: `Cmd + Shift + R`) — 캐시 무시하고 새로고침

**방법 3: 개발자 도구 확인 (🟡 고등학생)**
- `F12` 눌러서 개발자 도구 열기
- **Console** 탭에서 빨간색 에러 메시지가 있는지 확인

## 9. 브라우저 콘솔에 CORS 에러

### 증상
브라우저 개발자 도구(F12) Console에:
```
Access to fetch at 'http://localhost:3000/api/cards' from origin '...'
has been blocked by CORS policy
```

### 원인
서버에서 CORS 설정이 되어 있지 않거나, 다른 주소에서 접속하고 있습니다.

### 해결법

**방법 1: 같은 주소에서 접속**
- 반드시 `http://localhost:3000`에서 앱을 사용하세요
- 별도의 HTML 파일을 따로 열어서 접속하면 CORS 에러가 납니다

**방법 2: cors 패키지 확인 (🟡 고등학생)**
- `server.js`에 아래 코드가 있는지 확인:
  ```javascript
  const cors = require('cors');
  app.use(cors());
  ```
- 없다면 추가하고 `npm install cors` 실행

## 10. DB ID를 잘못 복사함

### 증상
서버는 실행되지만 Notion 관련 에러가 계속 발생합니다.

### 원인
`.env` 파일의 `NOTION_DATABASE_ID`가 올바르지 않습니다.

### 해결법: DB ID 다시 찾기

1. Notion에서 **패들렛 DB** 페이지를 엽니다
2. 브라우저 주소창을 확인합니다:

```
https://www.notion.so/myworkspace/abc123def456abc123def456abc12345?v=xyz...
                                  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                  이 부분이 DB ID (32자리)
```

3. `?v=` **앞에 있는** 32자리만 복사합니다
4. `.env` 파일에 붙여넣습니다:
   ```
   NOTION_DATABASE_ID=abc123def456abc123def456abc12345
   ```

주의사항:
- 하이픈(`-`)은 포함하지 않습니다 (Notion이 URL에 하이픈을 넣는 경우 제거)
- `?v=` 뒤의 내용은 복사하지 않습니다
- 워크스페이스 이름 부분도 포함하지 않습니다

## 11. npm install에서 에러 (ERESOLVE, permission 등)

### 증상
`npm install`을 실행하면 빨간색 에러가 많이 나옵니다.

### 해결법

**ERESOLVE 에러:**
```bash
npm install --legacy-peer-deps
```

**Permission 에러 (Mac):**
```bash
sudo npm install
```
(비밀번호 입력 필요)

**그래도 안 되면:**
1. `node_modules` 폴더를 삭제합니다:
   ```bash
   rm -rf node_modules
   ```
2. `package-lock.json` 파일도 삭제합니다:
   ```bash
   rm package-lock.json
   ```
3. 다시 설치합니다:
   ```bash
   npm install
   ```

## 12. Windows에서 터미널 명령어가 다르게 동작

### 증상
Mac용 명령어가 Windows에서 안 됩니다.

### 해결법: 명령어 대응표

| 동작 | Mac/Linux | Windows (cmd) | Windows (PowerShell) |
|------|-----------|--------------|---------------------|
| 파일 복사 | `cp a b` | `copy a b` | `Copy-Item a b` |
| 폴더 내용 보기 | `ls` | `dir` | `ls` 또는 `dir` |
| 파일 삭제 | `rm file` | `del file` | `Remove-Item file` |
| 폴더 삭제 | `rm -rf folder` | `rmdir /s folder` | `Remove-Item -Recurse folder` |

> 💡 VS Code 터미널에서 **Git Bash**를 사용하면 Mac 명령어를 그대로 쓸 수 있습니다.
> VS Code 터미널 오른쪽 드롭다운 → "Git Bash" 선택

# Step 1 수업 지도안 — Notion으로 배우는 웹 앱 만들기

> **대상:** 중학교 1~3학년 / 고등학교 1~2학년
> **시간:** 3차시 x 45분 (총 135분)
> **준비물:** 노트북(충전 완료), 인터넷 연결, Notion 계정, VS Code 설치

---

## 준비물 목록

| 항목 | 필수 여부 | 비고 |
|------|---------|------|
| 노트북 (Windows/Mac) | 필수 | 충전 완료 상태 |
| 인터넷 연결 (Wi-Fi) | 필수 | 학교 네트워크 확인 |
| Notion 계정 | 필수 | 무료 플랜 OK |
| VS Code | 필수 | 사전 설치 권장 |
| Node.js (v18 이상) | 필수 | 사전 설치 권장 |
| Chrome 또는 Edge 브라우저 | 필수 | |
| 프로젝트 파일 (USB/공유폴더) | 필수 | 교사가 미리 준비 |

> 💡 **교사 메모:** Node.js와 VS Code는 수업 전에 설치하는 것이 좋습니다. 설치에만 10~15분이 소요될 수 있습니다. IT 담당 선생님께 미리 협조를 구하세요.

---

## 차시별 계획

---

### 1차시 — 개념 이해 + Notion DB 구조 만들기 (45분)

**학습 목표:** 데이터베이스, 서버, API의 개념을 이해하고 Notion DB를 직접 만든다.

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~5분 | **도입:** "카카오톡 메시지는 어디에 저장될까?" 질문 | 질문 던지고 학생 답변 유도 | 00_개념교안.md 활용 |
| 5~15분 | **개념 설명:** DB, 서버, API 비유 설명 | 칠판/화면에 다이어그램 표시 | 도서관 장부, 식당 주방, 주문서 비유 |
| 15~20분 | **우리 앱 구조 설명:** 전체 데이터 흐름 | 구조 다이어그램 보여주기 | 브라우저 → 서버 → Notion DB |
| 20~25분 | **완성 화면 시연:** 교사가 완성된 앱 시연 | 카드 추가/수정/삭제 보여주기 | 학생 동기부여! |
| 25~40분 | **실습:** Notion 계정 만들기 + DB 구조 만들기 | 돌아다니며 개별 지원 | 01_API키설정가이드.md 4단계까지 |
| 40~45분 | **정리:** 오늘 배운 3가지 키워드 | 학생에게 발표시키기 | DB, 서버, API |

> 💡 **교사 메모:**
> - 🟢 중학생: 비유 설명에 시간을 더 쓰세요. "도서관 장부"에서 행/열 개념을 충분히 이해시킵니다.
> - 🟡 고등학생: 비유 후 실제 코드(server.js)를 간단히 보여주면 좋습니다.
> - 시연할 때 Notion DB 화면도 같이 보여주면 "데이터가 진짜 저장되네!" 반응이 좋습니다.

---

### 2차시 — API 키 발급 + 서버 코드 이해 (45분)

**학습 목표:** Notion API 키를 발급하고 서버 코드의 역할을 이해한다.

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~3분 | **복습:** 지난 시간 핵심 키워드 3개 | 학생에게 물어보기 | DB, 서버, API |
| 3~15분 | **실습:** Integration 만들기 + API 키 복사 | 교사 화면 공유하며 같이 진행 | 01_API키설정가이드.md 참조 |
| 15~25분 | **실습:** Integration-DB 연결 + DB ID 찾기 | 개별 지원 (5단계가 핵심!) | 가장 많이 막히는 부분 |
| 25~30분 | **코드 설명:** server.js 구조 훑어보기 | 화면에 코드 띄우고 설명 | 전체를 다 설명하지 않아도 OK |
| 30~40분 | **실습:** .env 파일 만들기 + npm install | 터미널 명령어 같이 입력 | 03_학생실습지침서.md D단계 |
| 40~45분 | **확인:** 서버 실행 테스트 (node server.js) | "서버 시작!" 메시지 확인 | 안 되는 학생 개별 지원 |

> 💡 **교사 메모:**
> - Integration-DB 연결(5단계)을 빠뜨리는 학생이 **매우 많습니다.** 전체 화면으로 한 번 더 보여주세요.
> - `npm install`을 입력했는데 에러가 나면 → 폴더 위치가 맞는지 확인 (`ls`로 package.json 있는지 확인)
> - 🟢 중학생: 코드 설명은 "이 부분이 카드를 가져오는 코드, 이 부분이 카드를 만드는 코드" 정도로 간단히.
> - 🟡 고등학생: `async/await`, `req/res` 패턴을 간단히 짚어주면 좋습니다.
> - 시간이 부족하면 코드 설명을 줄이고 실습에 집중하세요.

---

### 3차시 — 프론트엔드 연결 + 완성 앱 시연 (45분)

**학습 목표:** 전체 앱을 실행하고, 브라우저에서 Notion DB까지의 데이터 흐름을 확인한다.

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~3분 | **복습:** API 키가 하는 역할 | 학생 답변 유도 | "회원카드" 비유 |
| 3~10분 | **실습:** 서버 실행 + 브라우저 접속 | 함께 진행, 화면 공유 | http://localhost:3000 |
| 10~25분 | **실습:** 카드 추가/수정/삭제 해보기 | 돌아다니며 지원 | Notion DB에서 데이터 확인! |
| 25~35분 | **활동:** "우리 반 소개 보드 만들기" | 자유 활동 시간 | 각자 카드 3개 이상 만들기 |
| 35~40분 | **발표:** 재미있는 카드 소개 | 2~3명 발표 | 분위기 전환 |
| 40~45분 | **정리:** 전체 흐름 되짚기 + 학습 목표 체크 | 체크리스트 확인 | 00_개념교안.md 목표 체크리스트 |

> 💡 **교사 메모:**
> - "우리 반 소개 보드" 활동은 자유롭게 진행합니다. 자기소개, 좋아하는 것, 오늘 점심 등 아무 주제나 OK.
> - 🟢 중학생: 카드 추가/삭제만 해도 충분합니다. 성공 경험이 중요합니다.
> - 🟡 고등학생: 브라우저 개발자 도구(F12) → Network 탭에서 API 요청이 오가는 걸 보여주면 좋습니다.
> - 서버가 안 되는 학생은 05_트러블슈팅.md를 참고하게 하세요.

## 심화 활동 아이디어 (고등학생용)

빠르게 끝낸 학생에게 안내할 수 있는 추가 활동입니다.

### 🟡 중급 심화
1. **카드 색상 추가하기:** Color 옵션에 `orange`, `purple` 추가 후 프론트엔드에서 선택 가능하게 수정
2. **브라우저 개발자 도구 탐험:** F12 → Network 탭에서 API 요청/응답 확인하기
3. **db.js 코드 읽기:** `notionToCard` 함수가 하는 일 이해해보기

### 🔴 고급 심화
1. **카드 검색 기능 추가:** 제목에 특정 단어가 포함된 카드만 필터링
2. **바이브코딩으로 UI 개선:** 04_바이브코딩프롬프트집.md의 수정 프롬프트 활용
3. **Notion API 문서 읽기:** https://developers.notion.com/ 방문해서 다른 API 기능 탐색
4. **카드 정렬 기능:** 생성일순, 색상별 정렬 구현

> 💡 **교사 메모:**
> - 심화 활동은 **선택 사항**입니다. 기본 실습을 완료한 학생에게만 안내하세요.
> - 고등학생 그룹에서는 2~3차시를 합쳐서 심화 활동 시간을 확보할 수도 있습니다.
> - 자주 막히는 부분: `.env` 파일 오타, Integration-DB 연결 누락, `npm install` 누락. 이 세 가지만 사전에 강조하면 80%의 문제를 예방할 수 있습니다.

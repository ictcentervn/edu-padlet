# 데이터베이스란 무엇인가요? — Notion으로 배우는 DB 기초

> **대상:** 중학교 1학년 ~ 고등학교 2학년 (수준별 표시 포함)

---

## 오늘 배울 내용

> Notion 데이터베이스를 연결해서, 브라우저에서 카드를 붙이고 떼는 **우리 반 패들렛** 앱을 만듭니다.

---

## 생각해보기

카카오톡으로 친구에게 메시지를 보냈습니다. 핸드폰을 껐다가 다시 켜도 메시지가 남아 있죠?

그렇다면 이 메시지는 **어디에** 저장되어 있을까요?

- 내 핸드폰 안? 카카오톡 회사 서버?
- 인스타그램에 올린 사진은 어디에 있을까요?
- 유튜브 댓글은 어디에 저장될까요?

정답: 전부 **서버 안에 있는 데이터베이스**에 저장되어 있습니다.

오늘 우리도 데이터를 저장하는 앱을 직접 만들어 봅니다!

---

## 핵심 개념 1: 데이터베이스 (Database)

```
┌──────────────────────────────────────────┐
│         데이터베이스 (DB)                  │
│                                          │
│   번호 │ 제목       │ 내용     │ 색상    │
│   ─────┼───────────┼─────────┼────────│
│     1  │ 오늘 할일  │ 수학숙제 │ 노랑    │
│     2  │ 점심 메뉴  │ 비빔밥   │ 파랑    │
│     3  │ 동아리     │ 코딩반   │ 초록    │
│                                          │
└──────────────────────────────────────────┘
```

### 비유로 이해하기: 도서관 대출 장부

도서관에 가면 대출 장부가 있습니다.

| 학생 이름 | 빌린 책    | 빌린 날짜   | 반납 여부 |
|----------|----------|-----------|---------|
| 김민수    | 해리포터   | 2026-04-01 | X       |
| 이서연    | 수학의 정석 | 2026-04-03 | O       |

이 장부가 바로 **데이터베이스**입니다.

- 한 줄(행) = 데이터 하나 (레코드)
- 한 칸(열) = 데이터의 종류 (컬럼)

우리 앱에서는 **카드 하나하나**가 데이터베이스의 한 줄입니다.

---

## 핵심 개념 2: 서버 (Server)

```
      [손님 = 브라우저]
            │
            │  "비빔밥 주세요" (요청)
            ▼
   ┌─────────────────┐
   │   주방 = 서버     │
   │                   │
   │  주문 확인         │
   │  재료 꺼내기       │
   │  요리하기          │
   │  접시에 담기       │
   └─────────────────┘
            │
            │  "비빔밥 나왔습니다" (응답)
            ▼
      [손님 = 브라우저]
```

### 비유로 이해하기: 식당 주방

- 손님(브라우저)이 주문을 합니다.
- 주방(서버)이 주문을 받아서 요리를 만듭니다.
- 요리가 완성되면 손님에게 전달합니다.

우리 앱에서 **Express 서버**(server.js)가 바로 이 주방 역할입니다.

- "카드 목록 보여줘" 요청이 오면 → DB에서 카드를 꺼내 전달합니다.
- "새 카드 만들어줘" 요청이 오면 → DB에 카드를 저장합니다.

---

## 핵심 개념 3: API (Application Programming Interface)

```
   ┌─────────┐     주문서(API)      ┌──────────┐
   │  손님    │ ──────────────────▶ │  주방     │
   │ (브라우저)│                     │ (서버)    │
   │         │ ◀────────────────── │          │
   └─────────┘     완성된 요리       └──────────┘

   주문서 항목 (= API 엔드포인트):
   ┌─────────────────────────────────────┐
   │  GET  /api/cards     → 카드 목록    │
   │  POST /api/cards     → 카드 만들기  │
   │  PATCH /api/cards/1  → 카드 수정    │
   │  DELETE /api/cards/1 → 카드 삭제    │
   └─────────────────────────────────────┘
```

### 비유로 이해하기: 식당 주문서

식당에서 아무 말이나 하면 안 되죠. 메뉴판에 있는 음식만 주문할 수 있습니다.

API도 마찬가지입니다.

- 정해진 형식으로 요청해야 합니다.
- 정해진 형식으로 응답이 돌아옵니다.
- 이 "형식"을 API라고 부릅니다.

---

## 우리 앱 전체 구조

```
  브라우저(HTML/CSS/JS)
        │
        │ HTTP 요청 (fetch)
        │ 예: GET /api/cards
        ▼
  Express 서버 (server.js)
        │
        │ Notion API 호출
        │ 예: notion.databases.query(...)
        ▼
  Notion 데이터베이스 (클라우드)
        │
        │ 데이터 응답 (JSON)
        ▼
  Express 서버 (server.js)
        │
        │ HTTP 응답 (JSON)
        ▼
  브라우저 → 화면에 카드 표시!
```

요약하면:

```
브라우저 → (HTTP 요청) → Express 서버 → (Notion API) → Notion DB
```

브라우저가 서버에게 요청하고, 서버가 Notion에게 다시 요청해서 데이터를 가져옵니다.

## 오늘 만들 것

완성하면 이런 화면이 됩니다:

```
┌──────────────────────────────────────────────────┐
│            우리 반 패들렛                           │
│  ┌──────────────────────────────────────────┐    │
│  │  [+ 새 카드 만들기]                        │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ 오늘 할일  │  │ 점심 메뉴 │  │ 동아리    │      │
│  │          │  │          │  │          │      │
│  │ 수학숙제  │  │ 비빔밥   │  │ 코딩반    │      │
│  │    노랑   │  │   파랑   │  │   초록    │      │
│  │  [수정]   │  │  [수정]  │  │  [수정]   │      │
│  │  [삭제]   │  │  [삭제]  │  │  [삭제]   │      │
│  └──────────┘  └──────────┘  └──────────┘      │
└──────────────────────────────────────────────────┘
```

- 브라우저에서 카드를 추가하면 → Notion DB에 저장됩니다.
- Notion에서 데이터를 확인할 수 있습니다.
- 카드 수정, 삭제도 가능합니다.

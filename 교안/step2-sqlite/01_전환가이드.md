## 왜 server.js는 안 바꿔도 될까요?

### 비유: 주방 도구가 바뀌어도 주문서는 같다

```
[Step 1]                              [Step 2]
손님 → 주문서 → 주방(가스레인지)       손님 → 주문서 → 주방(인덕션)
        ↑ 같음!                                ↑ 같음!
```

식당에서 가스레인지를 인덕션으로 바꿔도, **주문서 양식**은 바뀌지 않죠?

우리 앱도 마찬가지입니다:
- **server.js** = 주문서 (어떤 요청이 오면 어떻게 처리할지)
- **db.js** = 주방 도구 (데이터를 실제로 저장하고 꺼내는 방법)

db.js가 바뀌어도, **같은 함수 이름**(`getCards`, `createCard`, `deleteCard`)을 유지하기 때문에 server.js는 수정할 필요가 없습니다.

```javascript
// server.js는 이렇게 db.js의 함수를 호출합니다
const db = require('./db');        // ← db.js 파일만 바꾸면 됨!

app.get('/api/cards', async (req, res) => {
    const cards = await db.getCards();   // ← 이 함수 이름은 안 바뀜
    res.json(cards);
});
```

이것이 바로 **모듈화**의 장점입니다. 부품 하나만 교체하면 전체가 바뀝니다.

## Notion vs SQLite 코드 나란히 비교

### 카드 목록 가져오기 (Before / After)

**Step 1: Notion (db.js)**
```javascript
// Notion API로 데이터 가져오기
getCards: async (boardId) => {
    const response = await notion.databases.query({
        database_id: NOTION_DATABASE_ID,
        filter: { property: 'BoardId', rich_text: { equals: boardId } }
    });
    return response.results.map(notionToCard);
}
```

**Step 2: SQLite (db.js)**
```javascript
// SQL로 데이터 가져오기
getCards: async (boardId) => {
    const stmt = db.prepare(
        'SELECT * FROM cards WHERE board_id = ? ORDER BY created_at DESC'
    );
    return stmt.all(boardId);
}
```

> 어떤 차이가 보이나요?
> - Notion: API를 통해 인터넷으로 요청 → 응답을 변환(`notionToCard`)해야 함
> - SQLite: SQL 한 줄로 바로 데이터를 가져옴 → 변환 불필요!

### 카드 추가하기 (Before / After)

**Step 1: Notion (db.js)**
```javascript
createCard: async ({ title, content, color, boardId }) => {
    const response = await notion.pages.create({
        parent: { database_id: NOTION_DATABASE_ID },
        properties: {
            Title:   { title: [{ text: { content: title } }] },
            Content: { rich_text: [{ text: { content: content } }] },
            Color:   { select: { name: color } },
            BoardId: { rich_text: [{ text: { content: boardId } }] }
        }
    });
    return notionToCard(response);
}
```

**Step 2: SQLite (db.js)**
```javascript
createCard: async ({ title, content, color, boardId }) => {
    const id = Date.now().toString();
    db.prepare(
        'INSERT INTO cards (id, title, content, color, board_id) VALUES (?, ?, ?, ?, ?)'
    ).run(id, title, content, color, boardId);
    return { id, title, content, color, boardId };
}
```

> 확연히 줄어들었죠!
> - Notion: 복잡한 JSON 구조를 만들어서 API로 보내야 함
> - SQLite: SQL INSERT 한 문장으로 끝!

### 4단계: 새 db.js 작성하기

VS Code에서 `step2-sqlite/db.js` 파일을 만들고 아래 코드를 붙여넣습니다:

```javascript
// SQLite db.js — Step 1의 Notion db.js를 이것으로 교체!
const Database = require('better-sqlite3');
const path = require('path');

// padlet.db 파일을 프로젝트 폴더에 생성 (없으면 자동 생성!)
const db = new Database(path.join(__dirname, 'padlet.db'));

module.exports = {
  // 테이블이 없으면 만들기 (서버 시작 시 한 번 실행)
  initialize: async () => {
    db.exec(`
      CREATE TABLE IF NOT EXISTS cards (
        id         TEXT PRIMARY KEY,
        title      TEXT NOT NULL,
        content    TEXT DEFAULT '',
        color      TEXT DEFAULT 'yellow',
        board_id   TEXT DEFAULT 'default',
        created_at TEXT DEFAULT (datetime('now'))
      );
      CREATE TABLE IF NOT EXISTS boards (
        id   TEXT PRIMARY KEY,
        name TEXT NOT NULL
      );
      INSERT OR IGNORE INTO boards (id, name) VALUES ('default', '우리 반 보드');
    `);
    console.log('✅ SQLite DB 초기화 완료 (padlet.db)');
  },

  // 카드 목록 조회
  getCards: async (boardId = 'default') => {
    const stmt = boardId === 'all'
      ? db.prepare('SELECT * FROM cards ORDER BY created_at DESC')
      : db.prepare('SELECT * FROM cards WHERE board_id = ? ORDER BY created_at DESC');
    return boardId === 'all' ? stmt.all() : stmt.all(boardId);
  },

  // 카드 생성
  createCard: async ({ title, content = '', color = 'yellow', boardId = 'default' }) => {
    const id = Date.now().toString();
    db.prepare(
      'INSERT INTO cards (id, title, content, color, board_id) VALUES (?, ?, ?, ?, ?)'
    ).run(id, title, content, color, boardId);
    return { id, title, content, color, boardId };
  },

  // 카드 수정
  updateCard: async (id, { title, content, color }) => {
    const updates = [];
    const values = [];
    if (title !== undefined)   { updates.push('title = ?');   values.push(title); }
    if (content !== undefined) { updates.push('content = ?'); values.push(content); }
    if (color !== undefined)   { updates.push('color = ?');   values.push(color); }
    values.push(id);
    db.prepare(`UPDATE cards SET ${updates.join(', ')} WHERE id = ?`).run(...values);
    return db.prepare('SELECT * FROM cards WHERE id = ?').get(id);
  },

  // 카드 삭제
  deleteCard: async (id) => {
    db.prepare('DELETE FROM cards WHERE id = ?').run(id);
    return { success: true };
  },

  // 보드 목록
  getBoards: async () => db.prepare('SELECT * FROM boards').all(),
  createBoard: async ({ name }) => {
    const id = name.toLowerCase().replace(/\s/g, '-');
    db.prepare('INSERT OR IGNORE INTO boards (id, name) VALUES (?, ?)').run(id, name);
    return { id, name };
  },
};
```

## Step 1 데이터는 어디 갔나요?

SQLite는 **완전히 새로운 데이터베이스**입니다. Step 1에서 Notion에 저장했던 카드는 여전히 Notion에 남아 있지만, Step 2의 padlet.db에는 들어 있지 않습니다.

처음 실행하면 빈 보드가 나타나는 것이 정상입니다!

> 💡 **교육 포인트:** 이것은 "데이터 마이그레이션(migration)"이라는 개념과 관련됩니다. 데이터베이스를 바꾸면 기존 데이터를 옮기는 작업이 필요합니다. 실제 서비스에서는 이 과정이 매우 중요합니다.

## 정리: Step 1과 Step 2의 핵심 차이

| 비교 항목 | Step 1 (Notion) | Step 2 (SQLite) |
|----------|----------------|----------------|
| 설치 난이도 | 어려움 (API 키, Integration, DB 연결) | **쉬움** (npm install이면 끝) |
| .env 파일 | 필수 (API 키 + DB ID) | 거의 불필요 |
| 인터넷 | 필수 | **불필요** |
| 데이터 확인 | Notion 웹사이트 접속 | VS Code에서 바로 확인 |
| 데이터 저장 방식 | Notion API (JSON 형식) | **SQL** (표준 데이터베이스 언어) |
| 바꾼 파일 | - | **db.js 하나!** |

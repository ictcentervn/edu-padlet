### A-4. db.js 파일 만들기

VS Code에서 `step2-sqlite` 폴더를 열고, **db.js** 파일을 새로 만듭니다.

아래 코드를 **그대로** 붙여넣으세요:

```javascript
// SQLite db.js — 데이터를 내 컴퓨터 파일(padlet.db)에 저장합니다
const Database = require('better-sqlite3');
const path = require('path');

// padlet.db 파일이 없으면 자동으로 만들어줍니다!
const db = new Database(path.join(__dirname, 'padlet.db'));

module.exports = {
  // 테이블 만들기 (서버 시작할 때 한 번 실행)
  initialize: async () => {
    db.exec(`
      CREATE TABLE IF NOT EXISTS cards (
        id         TEXT PRIMARY KEY,
        title      TEXT NOT NULL,
        content    TEXT DEFAULT '',
        color      TEXT DEFAULT 'yellow',
        board_id   TEXT DEFAULT 'default',
        created_at TEXT DEFAULT (datetime('now'))
      );
      CREATE TABLE IF NOT EXISTS boards (
        id   TEXT PRIMARY KEY,
        name TEXT NOT NULL
      );
      INSERT OR IGNORE INTO boards (id, name) VALUES ('default', '우리 반 보드');
    `);
    console.log('✅ SQLite DB 초기화 완료 (padlet.db)');
  },

  // 카드 목록 조회 (SELECT 사용)
  getCards: async (boardId = 'default') => {
    const stmt = boardId === 'all'
      ? db.prepare('SELECT * FROM cards ORDER BY created_at DESC')
      : db.prepare('SELECT * FROM cards WHERE board_id = ? ORDER BY created_at DESC');
    return boardId === 'all' ? stmt.all() : stmt.all(boardId);
  },

  // 카드 생성 (INSERT 사용)
  createCard: async ({ title, content = '', color = 'yellow', boardId = 'default' }) => {
    const id = Date.now().toString();
    db.prepare(
      'INSERT INTO cards (id, title, content, color, board_id) VALUES (?, ?, ?, ?, ?)'
    ).run(id, title, content, color, boardId);
    return { id, title, content, color, boardId };
  },

  // 카드 수정 (UPDATE 사용)
  updateCard: async (id, { title, content, color }) => {
    const updates = [];
    const values = [];
    if (title !== undefined)   { updates.push('title = ?');   values.push(title); }
    if (content !== undefined) { updates.push('content = ?'); values.push(content); }
    if (color !== undefined)   { updates.push('color = ?');   values.push(color); }
    values.push(id);
    db.prepare(`UPDATE cards SET ${updates.join(', ')} WHERE id = ?`).run(...values);
    return db.prepare('SELECT * FROM cards WHERE id = ?').get(id);
  },

  // 카드 삭제 (DELETE 사용)
  deleteCard: async (id) => {
    db.prepare('DELETE FROM cards WHERE id = ?').run(id);
    return { success: true };
  },

  // 보드 목록
  getBoards: async () => db.prepare('SELECT * FROM boards').all(),
  createBoard: async ({ name }) => {
    const id = name.toLowerCase().replace(/\s/g, '-');
    db.prepare('INSERT OR IGNORE INTO boards (id, name) VALUES (?, ?)').run(id, name);
    return { id, name };
  },
};
```

> ✅ **여기까지 확인 포인트:** step2-sqlite 폴더 안에 `server.js`, `db.js`, `public/` 폴더, `node_modules/` 폴더가 있으면 성공!

파일 구조가 이렇게 되어야 합니다:

## C단계: 카드 추가하고 데이터 확인하기

### C-1. 카드 추가해보기

1. **"새 카드 만들기"** 버튼을 클릭합니다
2. 제목과 내용을 입력합니다
3. 색상을 선택합니다
4. **저장**을 클릭합니다

화면에 카드가 나타나면 성공!

### C-2. padlet.db 파일 확인하기

VS Code 왼쪽 탐색기를 보세요. `padlet.db`라는 파일이 새로 생겼을 겁니다.

> 이 파일 안에 방금 추가한 카드 데이터가 들어 있습니다!
> Step 1에서는 Notion에 들어가야 데이터를 확인할 수 있었지만, Step 2에서는 **내 컴퓨터에서 바로** 확인할 수 있어요.

### C-3. VS Code SQLite Viewer 설치하기

padlet.db 파일을 열어서 데이터를 직접 보려면 확장 프로그램이 필요합니다.

1. VS Code 왼쪽 사이드바에서 **확장(Extensions)** 아이콘 클릭 (사각형 4개 모양)
2. 검색창에 `SQLite Viewer` 입력
3. **"SQLite Viewer"** (by Florian Klampfer) 선택
4. **Install** 버튼 클릭

설치가 끝나면:

1. VS Code 탐색기에서 **padlet.db** 파일을 클릭합니다
2. 테이블 형태로 데이터가 보입니다!

```
cards 테이블:
┌──────────────┬──────────┬──────────┬────────┬──────────┐
│ id           │ title    │ content  │ color  │ board_id │
├──────────────┼──────────┼──────────┼────────┼──────────┤
│ 1712345678   │ 오늘 할일 │ 수학숙제  │ yellow │ default  │
│ 1712345679   │ 점심 메뉴 │ 비빔밥   │ blue   │ default  │
└──────────────┴──────────┴──────────┴────────┴──────────┘
```

### C-4. 데이터 변화 관찰하기

이제 이런 실험을 해보세요:

1. 브라우저에서 **카드를 하나 더 추가**합니다
2. VS Code에서 padlet.db를 **다시 열면** (또는 새로고침) 새 행이 추가되어 있습니다!
3. 브라우저에서 **카드를 삭제**합니다
4. padlet.db에서 해당 행이 사라진 것을 확인합니다

> 브라우저에서 하는 동작이 padlet.db 파일에 바로 반영됩니다. 이것이 데이터베이스의 핵심입니다!

### C-5. 서버를 껐다 켜도 데이터가 남아있을까?

실험해봅시다:

1. 터미널에서 `Ctrl + C`를 눌러 **서버를 종료**합니다
2. 다시 `node server.js`로 서버를 **재시작**합니다
3. 브라우저에서 `http://localhost:3000`에 접속합니다
4. 아까 만든 카드가 **그대로 있습니다!**

> 왜 그럴까요? 데이터가 padlet.db **파일**에 저장되어 있기 때문입니다.
> 서버를 꺼도 파일은 사라지지 않으니까요!

> ✅ **여기까지 확인 포인트:** SQLite Viewer에서 데이터가 보이고, 서버 재시작 후에도 카드가 남아있으면 **완성!**

## D단계: SQL 직접 해보기 (심화)

> **🔴 심화: 직접 수정해보기**
> 이 단계는 선택 사항입니다. 여유가 있는 학생만 도전하세요!

### D-1. 터미널에서 SQL 실행하기

서버를 종료한 상태에서 (실행 중이면 `Ctrl + C`), 아래 명령어를 입력합니다:

```bash
npx better-sqlite3-cli padlet.db
```

> 이 명령어가 안 되면, Node.js REPL로도 할 수 있습니다:
> ```bash
> node -e "const db = require('better-sqlite3')('padlet.db'); console.log(db.prepare('SELECT * FROM cards').all());"
> ```

### D-2. SQL 명령어 직접 입력해보기

**모든 카드 보기:**
```sql
SELECT * FROM cards;
```

**노란색 카드만 보기:**
```sql
SELECT * FROM cards WHERE color = 'yellow';
```

**카드 직접 추가하기:**
```sql
INSERT INTO cards (id, title, content, color, board_id) 
VALUES ('test1', 'SQL로 만든 카드', '직접 SQL을 입력해서 만들었어요!', 'green', 'default');
```

**방금 추가한 카드 확인:**
```sql
SELECT * FROM cards WHERE id = 'test1';
```

**카드 삭제:**
```sql
DELETE FROM cards WHERE id = 'test1';
```

> SQL을 직접 입력해서 데이터를 조작해봤습니다!
> 우리 앱의 db.js가 하는 일이 바로 이것입니다.

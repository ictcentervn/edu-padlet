# Step 2 수업 지도안 — SQLite로 배우는 로컬 데이터베이스

> **대상:** 고등학교 1~3학년
> **시간:** 2차시 x 45분 (총 90분)
> **전제 조건:** Step 1(Notion) 수업 완료
> **준비물:** 노트북(충전 완료), VS Code 설치, Node.js 설치

---

## 준비물 목록

| 항목 | 필수 여부 | 비고 |
|------|---------|------|
| 노트북 (Windows/Mac) | 필수 | 충전 완료 상태 |
| VS Code | 필수 | Step 1에서 이미 설치됨 |
| Node.js (v18 이상) | 필수 | Step 1에서 이미 설치됨 |
| Chrome 또는 Edge 브라우저 | 필수 | |
| Step 1 프로젝트 파일 | 필수 | step1-notion 폴더 |
| 인터넷 연결 | npm install 시에만 필요 | 이후에는 불필요! |

> 💡 **교사 메모:** Step 1에서 Node.js, VS Code를 이미 설치했기 때문에 환경 설정 시간이 대폭 줄어듭니다. 인터넷은 `npm install` 할 때만 필요하고, 이후 앱 실행은 오프라인으로 가능합니다. Notion 계정도 필요 없습니다!

---

## 차시별 계획

---

### 1차시 — SQL 개념 + Step 1에서 Step 2로 전환 실습 (45분)

**학습 목표:** SQL의 기본 개념을 이해하고, Notion 기반 앱을 SQLite로 전환한다.

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~3분 | **도입:** "Step 1에서 Notion에 저장했던 거 기억나죠? 오늘은 인터넷 없이 해볼 거예요" | Step 1 복습 질문 | 생각해보기 질문 활용 |
| 3~10분 | **개념 설명:** Notion vs SQLite 비교 | 비교 표 + 다이어그램 화면 공유 | 00_개념교안.md 활용 |
| 10~18분 | **SQL 소개:** "DB에게 말하는 언어" | 도서관 사서 비유로 4가지 명령 설명 | CREATE TABLE, SELECT, INSERT, DELETE |
| 18~22분 | **구조 설명:** "server.js는 안 바뀌었어요!" | Before/After 코드 나란히 보여주기 | 01_전환가이드.md 활용 |
| 22~40분 | **실습:** Step 2 전환 따라하기 | 교사 화면 공유하며 같이 진행 | 03_학생실습지침서.md A~B단계 |
| 40~45분 | **확인:** 서버 실행 + 카드 추가 + padlet.db 생성 확인 | 돌아다니며 개별 확인 | "API 키 없이 바로 되죠!" |

> 💡 **교사 메모:**
> - Step 1에서 가장 힘들었던 API 키 발급 과정이 **없어서** 학생들이 훨씬 빠르게 진행합니다.
> - 🟡 고등학생: Before/After 코드 비교를 통해 "왜 db.js만 바꾸면 되는지" 이해시키세요.
> - `better-sqlite3` 설치 시 native module 빌드 에러가 발생할 수 있습니다. 05_트러블슈팅.md 1번 참조.
> - **핵심 순간:** 카드를 추가한 후 `padlet.db` 파일이 생기는 것을 확인시키세요. "이 파일 안에 우리 데이터가 있어요!"

### 2차시 — SQL 직접 확인 + 심화 활동 (45분)

**학습 목표:** VS Code SQLite Viewer로 데이터를 확인하고, SQL의 동작 방식을 체험한다.

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~3분 | **복습:** "padlet.db 안에 뭐가 있을까?" | 학생 답변 유도 | 지난 시간 연결 |
| 3~12분 | **실습:** VS Code SQLite Viewer 설치 + padlet.db 열기 | 화면 공유하며 같이 설치 | 03_학생실습지침서.md C단계 |
| 12~20분 | **활동:** 카드 추가/수정/삭제 후 SQLite Viewer에서 변화 확인 | 돌아다니며 지원 | 핵심 교육 포인트! |
| 20~25분 | **설명:** "서버를 껐다 켜도 데이터가 남아있어요!" | 서버 종료 → 재시작 시연 | 파일 DB의 특성 |
| 25~35분 | **심화 활동:** SQL 직접 해보기 또는 바이브코딩 | 수준별 안내 | 아래 상세 설명 |
| 35~42분 | **발표/공유:** 재미있는 카드 또는 심화 결과 소개 | 2~3명 발표 | |
| 42~45분 | **정리:** Step 1 vs Step 2 비교 정리 + 학습 목표 체크 | 체크리스트 확인 | 00_개념교안.md 목표 체크리스트 |

> 💡 **교사 메모:**
> - SQLite Viewer에서 데이터를 보는 순간이 **이 수업의 하이라이트**입니다. "여기 있는 데이터가 진짜 내가 방금 입력한 거예요!"라고 강조하세요.
> - 서버 종료 후 재시작해도 데이터가 남아있는 것을 시연하세요. "왜 남아있을까?" 질문 → "파일에 저장되니까!" 답변 유도.
> - 🟢 기본 학생: 카드 추가/삭제 + SQLite Viewer 확인만으로 충분합니다.
> - 🟡 중급 학생: 바이브코딩 프롬프트집 활용.
> - 🔴 심화 학생: 터미널에서 SQL 직접 실행 도전.

### 심화 활동 옵션 (25~35분)

학생 수준에 따라 택 1:

**🟢 기본:** "우리 반 소개 보드 만들기"
- 각자 카드 3개 이상 추가
- SQLite Viewer에서 자기 데이터 확인

**🟡 중급:** 바이브코딩으로 기능 추가
- 04_바이브코딩프롬프트집.md의 중급 프롬프트 활용
- 카드 검색, 정렬 기능 요청해보기

**🔴 심화:** SQL 직접 실행해보기
- 터미널에서 SQLite 쿼리 실행
- 직접 데이터 추가/조회/삭제
- 03_학생실습지침서.md D단계 참조

| **일부 학생만 빠른 경우** | 심화 활동 안내 (SQL 직접 실행 또는 바이브코딩) |

## Step 1과 비교한 수업 설계 차이

| 항목 | Step 1 (3차시) | Step 2 (2차시) | 이유 |
|------|--------------|--------------|------|
| 개념 설명 시간 | 1차시 전체 | 1차시 전반부 | DB, 서버, API 개념은 이미 학습 |
| 설정 시간 | 2차시 대부분 (API 키!) | 1차시 20분 | API 키 불필요! |
| 실습 시간 | 3차시 | 1차시 후반 + 2차시 | 설정이 간단해서 실습 비중 증가 |
| 심화 활동 | 시간 부족 시 생략 | 2차시에 충분히 가능 | 여유 시간 확보 |

> 💡 **교사 메모:**
> - Step 2는 Step 1보다 **진입 장벽이 훨씬 낮습니다.** API 키 발급이 없으니 학생들이 빠르게 성공 경험을 합니다.
> - 반대로 SQL이라는 새 개념이 등장하므로, SQL 설명에 충분한 시간을 투자하세요.
> - Step 1에서 막혀서 완성 못 한 학생도 Step 2에서는 성공할 가능성이 높습니다. 격려해주세요!
> - 수업 전에 교사 노트북에서 `better-sqlite3` 설치를 한 번 해보세요. native module 빌드 이슈가 있을 수 있습니다.

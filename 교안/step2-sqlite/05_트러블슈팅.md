## 1. better-sqlite3 설치 실패 (native module 빌드 에러)

### 증상
`npm install`을 실행하면 `better-sqlite3` 설치 중 빨간색 에러가 많이 나옵니다:
```
gyp ERR! build error
npm ERR! code 1
npm ERR! path .../node_modules/better-sqlite3
```

### 원인
better-sqlite3는 C++ 코드를 컴파일(빌드)해야 하는 패키지입니다. 컴파일 도구가 설치되어 있지 않으면 에러가 납니다.

### 해결법

**Mac:**
```bash
xcode-select --install
```
팝업이 나오면 "설치"를 클릭합니다. 설치 후 다시 `npm install`을 실행하세요.

**Windows:**
1. 관리자 권한으로 명령 프롬프트 또는 PowerShell을 엽니다
2. 아래 명령어를 실행합니다:
```bash
npm install -g windows-build-tools
```
3. 설치에 시간이 좀 걸립니다 (5~10분). 완료 후 다시 `npm install`을 실행하세요.

**그래도 안 되면 (대안):**
better-sqlite3 대신 `sql.js`를 사용하는 방법도 있습니다 (C++ 빌드 불필요):
```bash
npm install sql.js
```
다만 db.js 코드를 수정해야 합니다. 선생님께 도움을 요청하세요.

> 💡 **교사 메모:** 이것이 Step 2에서 가장 흔한 문제입니다. 수업 전에 교사 노트북과 학생용 노트북에서 미리 `npm install better-sqlite3`를 실행해보세요. Windows 환경에서 특히 문제가 많습니다. 사전 설치를 강력 권장합니다.

## 2. padlet.db 파일이 안 생김

### 증상
`node server.js`를 실행했는데 `padlet.db` 파일이 보이지 않습니다.

### 원인
대부분 **실행 경로**가 잘못되었습니다. padlet.db는 `node server.js`를 실행한 폴더에 생깁니다.

### 해결법

1. 터미널에서 현재 위치를 확인합니다:
   ```bash
   pwd
   ```
   결과가 `step2-sqlite` 폴더를 가리키고 있어야 합니다.

2. `ls`를 입력해서 `server.js`와 `db.js`가 보이는지 확인합니다.

3. 올바른 폴더에서 다시 실행합니다:
   ```bash
   cd step2-sqlite
   node server.js
   ```

4. 서버를 실행한 후 카드를 최소 하나 추가해야 padlet.db에 데이터가 들어갑니다. (padlet.db 파일 자체는 서버 시작 시 생성됩니다.)

> 💡 VS Code 탐색기에서 padlet.db가 안 보이면, 탐색기를 새로고침하세요 (탐색기 상단 새로고침 아이콘 클릭).

## 3. "database is locked" 에러

### 증상
카드를 추가하거나 삭제할 때:
```
SqliteError: database is locked
```

### 원인
다른 프로그램이 padlet.db 파일을 열고 있어서, 동시에 접근하려다 충돌이 발생한 것입니다.

### 해결법

1. **VS Code SQLite Viewer가 파일을 열고 있는 경우:**
   - SQLite Viewer 탭을 닫습니다
   - 다시 카드 추가/삭제를 시도합니다

2. **다른 터미널에서 SQLite를 열고 있는 경우:**
   - `npx better-sqlite3-cli padlet.db` 같은 명령으로 DB를 열어둔 다른 터미널을 종료합니다

3. **그래도 안 되면:**
   ```bash
   # 서버 종료 (Ctrl + C)
   # 서버 재시작
   node server.js
   ```

> 💡 SQLite는 한 번에 하나의 프로그램만 쓰기(write) 할 수 있습니다. 읽기(read)는 여러 프로그램이 동시에 가능합니다.

## 4. VS Code SQLite Viewer가 안 됨

### 증상
padlet.db를 클릭해도 알 수 없는 문자가 나타나거나, 아무것도 표시되지 않습니다.

### 원인
SQLite Viewer 확장 프로그램이 설치되지 않았거나, 다른 편집기로 열리고 있습니다.

### 해결법

**확장 프로그램이 설치 안 된 경우:**
1. VS Code 왼쪽 사이드바 → **확장(Extensions)** 아이콘 (사각형 4개 모양)
2. `SQLite Viewer` 검색
3. **Florian Klampfer** 개발자의 "SQLite Viewer"를 **Install**

**설치했는데 안 열리는 경우:**
1. padlet.db 파일을 **우클릭**
2. **Open With...** 선택
3. **SQLite Viewer** 선택

**대안: 다른 확장 프로그램 사용**
- `SQLite` (alexcvzz) — 명령 팔레트(Ctrl+Shift+P)에서 "SQLite: Open Database" 사용
- `vscode-sqlite3-editor` — 테이블 편집도 가능

## 5. Step 1 데이터가 없어요

### 증상
Step 2를 실행했는데 빈 보드가 나타납니다. Step 1에서 만든 카드가 없습니다.

### 원인
이것은 **정상**입니다! SQLite는 완전히 새로운 데이터베이스입니다. Step 1에서 Notion에 저장한 데이터는 Notion에 그대로 있지만, SQLite의 padlet.db에는 들어 있지 않습니다.

### 해결법
해결이 필요한 문제가 아닙니다. 새 카드를 추가하면 됩니다!

> 💡 **교육 포인트:** 이것이 바로 "데이터 마이그레이션"이 필요한 이유입니다. 실제 서비스에서 데이터베이스를 바꾸면, 기존 데이터를 새 DB로 옮기는 작업이 반드시 필요합니다.

## 6. 서버 종료 후 데이터가 남아있어요

### 증상
`Ctrl + C`로 서버를 종료하고, 다시 `node server.js`로 시작했는데 이전에 만든 카드가 그대로 있습니다.

### 원인
이것도 **정상**입니다! 이것이 바로 **파일 데이터베이스**의 특징입니다.

### 설명
padlet.db는 일반 파일입니다. 서버를 꺼도 파일은 컴퓨터에 그대로 남아 있습니다.

비유: 일기장에 글을 쓴 다음 볼펜 뚜껑을 닫아도(서버 종료), 일기장의 내용은 사라지지 않는 것과 같습니다.

> 💡 **교육 포인트:** 이것이 데이터베이스를 쓰는 핵심 이유입니다. 프로그램이 종료되어도 데이터가 유지됩니다. 만약 JavaScript 변수에만 데이터를 저장했다면, 서버를 끄는 순간 모든 데이터가 사라졌을 겁니다.

### 데이터를 처음부터 시작하고 싶다면?
padlet.db 파일을 삭제하면 됩니다:

**Mac:**
```bash
rm padlet.db
```

**Windows:**
```bash
del padlet.db
```

서버를 다시 시작하면 빈 padlet.db가 새로 만들어집니다.

## 7. node server.js 실행 경로 오류

### 증상
`node server.js`를 실행하면:
```
Error: Cannot find module './db'
```
또는
```
Error: Cannot find module 'express'
```

### 원인
`step2-sqlite` 폴더가 아닌 다른 곳에서 실행하고 있습니다.

### 해결법

1. 터미널에서 현재 위치 확인:
   ```bash
   pwd     # Mac
   cd      # Windows (현재 경로 표시)
   ```

2. `step2-sqlite` 폴더로 이동:
   ```bash
   cd step2-sqlite
   ```

3. `ls`(Mac) 또는 `dir`(Windows)로 파일 확인:
   ```
   server.js    db.js    package.json    public/    node_modules/
   ```
   이 파일들이 모두 보여야 합니다.

4. `node_modules` 폴더가 없으면:
   ```bash
   npm install
   ```

5. 다시 실행:
   ```bash
   node server.js
   ```

## 8. SQL 문법 오류 (심화 활동 시)

### 증상
터미널에서 SQL을 직접 입력했는데 에러가 납니다:
```
SqliteError: near "SELEC": syntax error
```

### 원인
SQL 문법을 잘못 입력했습니다. 오타가 가장 흔한 원인입니다.

### 해결법: 자주 하는 실수

**1. 오타:**
```sql
-- 잘못됨
SELEC * FROM cards;

-- 올바름
SELECT * FROM cards;
```

**2. 세미콜론(;) 빠뜨림:**
```sql
-- 잘못됨
SELECT * FROM cards

-- 올바름
SELECT * FROM cards;
```

**3. 따옴표 종류:**
```sql
-- 잘못됨 (큰따옴표)
SELECT * FROM cards WHERE color = "yellow";

-- 올바름 (작은따옴표)
SELECT * FROM cards WHERE color = 'yellow';
```

**4. 테이블 이름 오타:**
```sql
-- 잘못됨
SELECT * FROM card;

-- 올바름 (복수형: cards)
SELECT * FROM cards;
```

**5. 컬럼 이름 확인:**
```sql
-- 테이블 구조 확인
.schema cards

-- 결과:
-- CREATE TABLE cards (
--   id TEXT PRIMARY KEY,
--   title TEXT NOT NULL,
--   content TEXT DEFAULT '',
--   color TEXT DEFAULT 'yellow',
--   board_id TEXT DEFAULT 'default',
--   created_at TEXT DEFAULT (datetime('now'))
-- );
```

> 💡 SQL 문법이 헷갈리면 Claude.ai에 물어보세요:
> "이 SQL이 왜 에러 나는지 알려줘: [SQL 문장]"

## 10. EADDRINUSE: address already in use :::3000

### 증상
```
Error: listen EADDRINUSE: address already in use :::3000
```

### 원인
이미 다른 서버(Step 1 서버 포함!)가 3000번 포트를 사용하고 있습니다.

### 해결법

**방법 1: 이전 서버 종료**
- 다른 터미널 탭에서 Step 1 서버가 실행 중이라면 → 그 터미널에서 `Ctrl + C` 누르기

**방법 2: 강제 종료 (Mac)**
```bash
lsof -i :3000
kill -9 [PID번호]
```

**방법 2: 강제 종료 (Windows)**
```bash
netstat -ano | findstr :3000
taskkill /PID [PID번호] /F
```

그 후 다시 `node server.js` 실행

> 💡 Step 1과 Step 2 서버를 동시에 실행하려면, 한쪽의 포트 번호를 바꿔야 합니다. 하지만 보통은 하나만 실행하면 충분합니다.

## Step 1 vs Step 2 트러블슈팅 비교

Step 2는 Step 1보다 문제가 적습니다!

| Step 1에서 자주 발생한 문제 | Step 2에서는? |
|--------------------------|-------------|
| .env 파일 숨김파일 문제 | .env가 거의 필요 없어서 **문제 없음** |
| Notion API 키 발급 어려움 | API 키 불필요 → **문제 없음** |
| Integration-DB 연결 누락 | Integration 없음 → **문제 없음** |
| DB ID 복사 실수 | DB ID 없음 → **문제 없음** |
| 컬럼 대소문자 불일치 | CREATE TABLE로 직접 정의 → **내가 통제** |
| 인터넷 끊김 | 인터넷 불필요 → **문제 없음** |

> **새로 생기는 문제:** better-sqlite3 설치(빌드) 에러 → 위 1번 참조

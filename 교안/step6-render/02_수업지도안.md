**대상:** 고등학교 2~3학년 심화
**시간:** 2차시 x 45분
**준비물:** 노트북, 인터넷, GitHub 계정 (Step 4 완료), Supabase 키 (Step 3 완료)
**사전 조건:** Step 1~5 완료 (최소 Step 3~4 완료 필수)

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~5분 | 도입: "코드를 어떻게 서버에 올릴까?" 질문 | "프로그래머가 코드를 USB에 담아서 서버에 가져갈까요?" → 토론 유도 | Step 5 복습 연결 |
| 5~12분 | 개념 설명: 배포, CI/CD, 환경변수 | 00_개념교안.md의 다이어그램 활용. "원고 → 출판사" 비유 강조 | Codespaces vs Render 비교표 함께 |
| 12~15분 | Render 가입 시범 | 교사 화면 공유: render.com → GitHub 로그인 과정 시연 | 학생은 따라하지 않고 관찰만 |
| 15~25분 | 학생 실습: Render 가입 + Web Service 생성 | 순회하며 도움. 01_설정가이드.md Step 1~4 진행 | GitHub 연결이 안 되는 학생 우선 지원 |

| 30~35분 | 배포 시작 + 대기 | "Create Web Service" 클릭 → 빌드 시작 (2~3분 소요) | **대기 시간 활용:** 아래 참조 |
| 35~42분 | 배포 완료 확인 + URL 접속 | 학생 각자 URL 접속 확인. 친구 URL도 접속해보기 | 성공 경험 공유 |
| 42~45분 | 1차시 정리 | "오늘 한 일: GitHub → Render → 자동 배포 = CI/CD" | 2차시 예고: 코드 수정 → push → 자동 배포 |

#### 빌드 대기 시간(2~3분) 활용 방법

빌드가 진행되는 동안 학생들이 기다리게 됩니다. 이 시간을 활용하세요:

1. **배포 흐름 퀴즈:** "지금 Render가 하는 일을 순서대로 말해볼까요?"
   - 코드 다운로드 → npm install → node server.js → URL 공개
2. **CI/CD 비유 복습:** "출판사 비유에서 지금 어느 단계인가요?"
3. **Render 로그 읽기:** "빌드 로그에 뭐라고 써있나요? 어떤 패키지가 설치되고 있나요?"

> 💡 교사 메모: 빌드 시간은 2~5분으로 변동됩니다. 무료 플랜은 더 느릴 수 있습니다. 학생들이 답답해하지 않도록 "실제 회사에서도 빌드에 5~10분 걸리는 경우가 많다"고 안내하세요.

### 2차시 — 자동 배포 체험 + 전체 회고 + 발전 방향 토론 (45분)

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~3분 | 1차시 복습 | "Render가 하는 일을 한 문장으로?" → "GitHub 코드를 가져와서 자동 배포" | |
| 3~8분 | 시범: 코드 수정 → push → 자동 배포 | 교사 화면 공유. CSS 색상 변경 → git push → Render 자동 빌드 확인 | 실시간으로 보여주기 |
| 8~20분 | 학생 실습: 자동 배포 체험 | 각자 코드 수정 → push → 자동 배포 확인 | 03_학생실습지침서.md F단계 |
| 20~25분 | URL 공유 + 상호 방문 | 친구 URL 접속 → 카드 추가 → 서로의 앱에 흔적 남기기 | "진짜 서비스"의 느낌 체험 |
| 25~35분 | 6단계 전체 회고 | 전체 여정 다이어그램 화면에 표시. 각 단계에서 배운 핵심을 학생에게 물어보기 | 아래 회고 진행 가이드 참조 |
| 35~42분 | 발전 방향 토론 | "이 앱을 더 발전시키려면?" 자유 토론. AI에게 물어보기 활동 | 04_바이브코딩프롬프트집.md 활용 |
| 42~45분 | 마무리 | "6단계를 모두 마쳤습니다! 여러분은 실제 개발자와 같은 과정을 거쳤어요." | 수료 인증 또는 기념 |

#### 6단계 전체 회고 진행 가이드

화면에 아래 다이어그램을 보여주고, 각 단계를 짚으며 학생에게 질문합니다:

```
Step 1 (Notion)   → "데이터를 어디에 저장했나요?"
Step 2 (SQLite)   → "SQL이 뭐였죠?"
Step 3 (Supabase) → "클라우드 DB의 장점은?"
Step 4 (GitHub)   → ".env를 왜 올리면 안 되죠?"
Step 5 (Codespaces)   → "localhost와 뭐가 달랐나요?"
Step 6 (Render)   → "push만 하면 뭐가 일어나죠?"
```

핵심 메시지:

> "여러분은 데이터베이스 설계부터, 코드 관리, 배포까지 — 실제 개발자가 거치는 과정을 모두 경험했습니다. 이것이 바로 '풀스택 개발'의 기초입니다."

## 💡 교사 메모

### 수업 전 준비

- [ ] render.com에 교사 계정으로 미리 테스트 배포해보기
- [ ] 학생 GitHub 저장소에 최신 코드가 push되어 있는지 확인
- [ ] Supabase 프로젝트가 아직 살아 있는지 확인 (무료 플랜은 1주 비활동 시 일시정지 가능)
- [ ] 학교 네트워크에서 render.com 접속 가능 여부 확인

### 자주 발생하는 상황

1. **빌드 실패:** 대부분 환경변수 누락 또는 Start Command 오류. 로그를 함께 확인하세요.
2. **무료 슬립:** 시연 직전에 미리 접속하여 서버를 깨워두세요.
3. **GitHub 권한 문제:** Render에 GitHub 앱 설치 권한이 필요합니다. "Configure account" 버튼으로 해결.
4. **학생 간 속도 차이:** 빠른 학생에게는 G단계(심화: 커스텀 도메인, 로그 분석)를 안내하세요.

### 시간 부족 시 생략 가능 파트

- 2차시의 "발전 방향 토론" (10분) → 과제로 전환 가능
- 커스텀 도메인 설정 (G단계) → 관심 있는 학생만 자율 학습

### 시간 여유 시 추가 가능 파트

- GitHub Actions 소개 (자동 테스트 추가)
- 다른 배포 플랫폼 소개 (Vercel, Railway, Fly.io)
- 모니터링 / 로그 분석 실습

### 🟢 기본 프롬프트 — Render 배포 준비

**Bolt.new 또는 Claude.ai에 아래를 붙여넣으세요:**

```
Express.js로 만든 패들렛 앱을 Render에 배포하려고 해.

현재 코드 구조:
- server.js (Express 서버)
- db.js (Supabase 연동)
- public/index.html, style.css, app.js
- package.json
- .env (SUPABASE_URL, SUPABASE_ANON_KEY)

Render 배포를 위해 확인해야 할 것들을 알려줘:
1. server.js에서 PORT 설정이 올바른지
2. package.json에 start 스크립트가 있는지
3. .gitignore에 .env가 포함되어 있는지
4. 그 외 Render 배포 전 체크리스트

각 항목별로 올바른 코드와 틀린 코드를 비교해서 보여줘.
한국어 주석 포함.
```

### 🟡 중급 프롬프트 — 배포된 앱에 기능 추가 후 재배포

**기능 추가 + 자동 배포 체험:**

```
내 패들렛 앱에 "카드 색상 선택" 기능을 추가하고 싶어.

현재 상태:
- Express.js + Supabase로 만든 앱
- Render에 이미 배포되어 있음
- GitHub에 push하면 자동 배포됨

추가할 기능:
- 카드 만들 때 색상을 선택할 수 있는 드롭다운 메뉴
- 색상 옵션: 노랑, 파랑, 초록, 분홍, 보라
- 선택한 색상이 카드 배경에 적용됨
- Supabase DB에 color 컬럼이 이미 있음

수정해야 할 파일별로 변경 내용을 알려줘:
1. public/index.html (드롭다운 추가)
2. public/app.js (색상 데이터 전송)
3. public/style.css (색상별 스타일)

수정 후 git push로 자동 배포하는 명령어도 알려줘.
한국어 주석 포함.
```

### 🔴 심화 프롬프트 — GitHub Actions로 자동 테스트

```
내 패들렛 앱에 GitHub Actions를 설정해서,
push할 때마다 자동으로 테스트를 실행한 후 Render에 배포되게 하고 싶어.

현재 상태:
- Express.js + Supabase 앱
- GitHub 저장소에 코드가 있음
- Render에서 GitHub 연동 자동 배포 중

추가할 것:
1. 간단한 테스트 파일 (API 엔드포인트가 응답하는지 확인)
2. .github/workflows/test.yml 파일 (GitHub Actions 워크플로우)
3. push할 때마다: 테스트 실행 → 통과하면 Render가 자동 배포

테스트는 간단하게:
- GET /api/cards가 200을 반환하는지
- POST /api/cards가 201을 반환하는지

Jest와 supertest를 사용해줘.
한국어 주석 포함.
고등학생이 이해할 수 있는 수준으로 각 부분을 설명해줘.
```

```
Render에 배포한 Express.js 앱을 프로덕션 환경에 맞게 최적화해줘.

4. 환경변수로 개발/프로덕션 모드 구분
5. 정적 파일 캐싱 설정

각 항목이 왜 필요한지, 보안이나 성능 측면에서 설명해줘.
고등학생이 이해할 수 있는 비유를 사용해줘.
한국어 주석 포함.
```

## 6단계 전체 회고 프롬프트

**6단계를 모두 마친 후, 아래 프롬프트로 전체 과정을 정리하세요:**

```
우리는 교육용 패들렛(보드) 앱을 만들면서 6단계를 거쳤어:

Step 1: Notion API로 데이터베이스 개념 배우기
Step 2: SQLite로 SQL 쿼리 직접 작성하기
Step 3: Supabase로 클라우드 데이터베이스 사용하기
Step 4: GitHub으로 코드를 버전 관리하기
Step 5: Glitch로 브라우저에서 코딩하고 바로 배포하기
Step 6: Render로 GitHub 연동 자동 배포(CI/CD) 체험하기

이 과정에서 배운 것을 정리해주고,
이 앱을 더 발전시키려면 어떤 기능을 추가하면 좋을지 5가지 제안해줘.

각 제안에 대해:
- 어떤 기술이 필요한지
- 난이도 (쉬움/보통/어려움)
- 배울 수 있는 새로운 개념
을 함께 알려줘.

고등학생이 이해할 수 있는 수준으로 설명해줘.
```

# Step 6 — Render 배포 실습 지침서

> **대상:** 고등학교 2~3학년 심화
> **소요 시간:** 약 40분 (A~F단계)

---

## 시작 전 체크리스트

- [ ] 노트북 충전 상태 확인
- [ ] GitHub 계정 로그인 완료 (Step 4에서 만든 계정)
- [ ] GitHub에 edu-padlet 저장소가 있고, 최신 코드가 push되어 있음
- [ ] Supabase 프로젝트 URL과 anon 키를 알고 있음 (Step 3에서 사용한 값)
- [ ] VS Code + Git 사용 가능

> 🆘 Supabase 키가 기억나지 않으면? → Supabase 대시보드 → Settings → API 에서 확인

---

## Step A: Render 가입

### A-1. render.com 접속

1. 브라우저에서 **render.com** 접속
2. 우측 상단 **"Get Started for Free"** 클릭

### A-2. GitHub 계정으로 로그인

1. **"GitHub"** 버튼 클릭
2. GitHub 로그인 화면이 나오면 로그인
3. "Authorize Render" 화면에서 **허용** 클릭

> ✅ 여기까지 완료되면 Render 대시보드가 보입니다

---

## Step B: Web Service 생성 + GitHub 연결

### B-1. 새 서비스 만들기

1. 대시보드에서 **"New +"** 버튼 클릭
2. **"Web Service"** 선택

### B-2. GitHub 저장소 연결

1. "Connect a repository" 화면에서 GitHub 저장소 목록을 확인
2. **edu-padlet** 저장소를 찾아 **"Connect"** 클릭

> 🆘 저장소가 안 보이나요? → "Configure account" 클릭 → Render에 저장소 접근 권한 허용

### B-3. 서비스 설정 입력

아래 값을 정확히 입력하세요:

| 항목 | 입력값 |
|------|--------|
| Name | `내영문이름-padlet` (예: `minsoo-padlet`) |
| Region | `Singapore (Southeast Asia)` |
| Branch | `main` |
| Build Command | `npm install` |
| Start Command | `node server.js` |
| Instance Type | `Free` 선택 |

> ✅ 여기까지 완료되면 서비스 설정이 끝났습니다. 아직 "Create" 버튼을 누르지 마세요!

---

## Step C: 환경변수 설정

이 단계를 빠뜨리면 앱이 작동하지 않습니다!

### C-1. 환경변수 추가

1. 설정 페이지에서 **"Environment Variables"** 섹션 찾기
2. **"Add Environment Variable"** 클릭
3. 첫 번째 환경변수 입력:
   - Key: `SUPABASE_URL`
   - Value: (Step 3에서 사용한 Supabase 프로젝트 URL 붙여넣기)
4. 다시 **"Add Environment Variable"** 클릭
5. 두 번째 환경변수 입력:
   - Key: `SUPABASE_ANON_KEY`
   - Value: (Step 3에서 사용한 anon/public 키 붙여넣기)

### C-2. 확인

```
입력한 환경변수 2개:
✅ SUPABASE_URL       = https://xxxxx.supabase.co
✅ SUPABASE_ANON_KEY  = eyJhbG...
```

> 🆘 Supabase 키를 찾는 방법:
> 1. supabase.com 로그인
> 2. 내 프로젝트 선택
> 3. 좌측 메뉴 Settings → API
> 4. "Project URL"과 "anon public" 키 복사

> ✅ 환경변수 2개가 모두 입력되었으면 다음 단계로!

---

## Step D: 배포 완료 확인 + URL 접속

### D-1. 배포 시작

1. **"Create Web Service"** 클릭
2. 빌드 로그 화면이 나타납니다

### D-2. 빌드 진행 관찰 (2~3분 대기)

로그에 아래와 비슷한 내용이 나옵니다:

```
==> Cloning from https://github.com/...
==> Running build command 'npm install'
    added 45 packages in 12s
==> Build successful!
==> Starting service with 'node server.js'
```

**마지막에 "Your service is live!"** 메시지가 나오면 성공!

### D-3. 내 앱 접속

1. 페이지 상단에 있는 URL을 클릭 (예: `https://minsoo-padlet.onrender.com`)
2. 우리 패들렛 앱이 나타나는지 확인
3. 카드를 하나 추가해보기

> ✅ 카드가 추가되고, 페이지를 새로고침해도 남아있으면 성공!

> 🆘 "Application error" 화면이 나오나요? → 05_트러블슈팅.md의 #4 참조

---

## Step E: 친구에게 URL 공유

### E-1. URL 복사

브라우저 주소창에서 URL을 복사합니다:
```
https://내이름-padlet.onrender.com
```

### E-2. 친구에게 보내기

1. 옆자리 친구에게 내 URL을 공유합니다
2. 친구가 내 앱에 접속해서 **카드를 추가**합니다
3. 나도 친구의 URL에 접속해서 카드를 추가합니다

### E-3. 확인

```
내 앱에 친구가 추가한 카드가 보이나요?  → ✅ 진짜 서비스입니다!
```

> 💡 인터넷이 되는 곳이면 어디서든 접속할 수 있습니다. 집에서 스마트폰으로도 접속해보세요!

---

## Step F: 코드 수정 → git push → 자동 배포 확인

**이 단계가 오늘의 핵심입니다!** CI/CD를 직접 체험합니다.

### F-1. 코드 수정

VS Code에서 `public/style.css`를 열고, 원하는 부분을 수정합니다.

예시: 배경색 변경
```css
body {
  background-color: #e8f5e9;  /* 연한 초록색 */
}
```

또는 제목 색상 변경, 카드 모서리 둥글게 등 원하는 대로 수정하세요.

### F-2. git add + commit + push

VS Code 터미널에서:

```bash
git add .
git commit -m "배경색을 초록색으로 변경"
git push origin main
```

### F-3. Render에서 자동 배포 확인

1. Render 대시보드로 돌아가기
2. **"Deploys"** 탭에서 새로운 배포가 시작된 것을 확인

```
🔵 Deploy in progress...    방금 전
✅ Deploy succeeded          5분 전   ← 이전 배포
```

3. 2~3분 기다리면 **"Deploy succeeded"** 로 변경됩니다

### F-4. 결과 확인

1. 내 앱 URL을 새로고침합니다
2. 수정한 내용이 반영되어 있나요?

```
코드 수정 → git push → 2~3분 → 전 세계에 반영!
```

> ✅ 이것이 **CI/CD**입니다! 실제 개발자들도 매일 이렇게 일합니다.

## Step G: 심화 도전 (선택)

시간이 남거나 더 해보고 싶은 학생을 위한 도전 과제입니다.

### G-1. Render 로그 분석

1. Render 대시보드 → **"Logs"** 탭 클릭
2. 서버 로그를 실시간으로 확인합니다
3. 내 앱에 접속할 때마다 로그에 새로운 내용이 나타나는지 확인

```
어떤 정보가 보이나요?
- 서버 시작 메시지
- API 요청 기록
- 에러 메시지 (있다면)
```

### G-2. 기능 추가 → 자동 배포

새로운 기능을 추가하고 자동 배포를 한 번 더 체험합니다:

1. `public/index.html`에서 제목을 변경하거나 새 기능 추가
2. git add → commit → push
3. Render에서 자동 배포 확인
4. URL 접속 → 변경 확인

### G-3. 커스텀 도메인 설정 (고급)

Render는 커스텀 도메인을 연결할 수 있습니다:

1. Render 대시보드 → **"Settings"** → **"Custom Domains"**
2. 도메인을 가지고 있다면 연결 가능
3. DNS 설정이 필요하므로 관심 있는 학생만 도전

## 최종 확인 체크리스트

- [ ] Render에 앱이 배포되어 URL로 접속 가능하다
- [ ] 환경변수가 올바르게 설정되어 카드 추가/삭제가 된다
- [ ] 친구가 내 URL에 접속해서 카드를 추가했다
- [ ] 코드를 수정하고 push하면 자동으로 배포되는 것을 확인했다
- [ ] git push → Render 자동 빌드의 흐름을 설명할 수 있다

> 🎉 **모든 항목을 체크했다면, 6단계 전체 과정을 완료한 것입니다!**
> 여러분은 데이터베이스부터 실제 서비스 배포까지, 풀스택 개발의 전 과정을 경험했습니다.

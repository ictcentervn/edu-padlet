# 자주 막히는 문제 해결법 — Step 5 (Codespaces)

> **대상:** 중학교 1학년 ~ 고등학교 3학년
> 문제가 생기면 아래에서 찾아보세요. 해결이 안 되면 선생님께 질문!

---

## 1. Codespace 생성이 안 돼요

### 증상
"Create codespace on main" 버튼을 눌렀는데 에러가 나거나 반응이 없음

### 해결법

**GitHub 계정 확인:**
- github.com에 로그인되어 있나요?
- Step 4에서 만든 계정으로 접속했나요?

**무료 사용량 확인:**
- github.com → Settings → Billing → Codespaces에서 남은 시간 확인
- 월 60시간을 다 썼다면 다음 달까지 기다려야 합니다

**저장소 확인:**
- 내 Fork 저장소인지 확인 (다른 사람 저장소에서는 생성 제한이 있을 수 있음)
- 저장소에 코드가 있는지 확인 (빈 저장소에서는 Codespace가 의미 없음)

---

## 2. VS Code가 안 열리고 로딩만 계속 돼요

### 증상
"Setting up your codespace..." 화면에서 1분 이상 멈춤

### 해결법

1. 브라우저 **새로고침** (`F5` 또는 `Ctrl + R`)
2. 그래도 안 되면 **다른 브라우저**로 시도 (Chrome을 권장)
3. 여전히 안 되면:
   - github.com → Your codespaces → 해당 Codespace 삭제 → 다시 생성
4. 학교 네트워크가 차단하는 경우:
   - `*.github.dev` 도메인이 차단되지 않았는지 선생님께 확인

---

## 3. 터미널이 안 보여요

### 증상
Codespace는 열렸는데 아래쪽에 터미널 창이 없음

### 해결법

- 키보드: `Ctrl` + `` ` `` (백틱, 숫자 1 왼쪽의 키)
- 또는 상단 메뉴: **Terminal → New Terminal**
- 또는 단축키: `Ctrl + Shift + `` ` ``

> 💡 터미널 창이 너무 작으면 터미널 상단 경계선을 마우스로 잡아서 위로 끌어올리세요.

---

## 4. npm install 에러가 나요

### 증상
`npm install`을 했는데 빨간 에러 메시지가 나옴

### 해결법

**"npm: command not found"**
→ 거의 발생하지 않음 (Codespace에 이미 설치됨). Codespace를 삭제하고 다시 생성하세요.

**"ENOENT: no such file or directory, open 'package.json'"**
→ 현재 폴더에 package.json이 없음. 파일 목록에서 package.json이 보이는지 확인하세요.
→ 터미널에서 `ls`를 입력해 파일 목록을 확인하세요.

**네트워크 에러 (ETIMEDOUT, ECONNREFUSED 등)**
→ 인터넷 연결 확인. Codespace는 클라우드이므로 보통은 문제 없지만, 학교 네트워크가 불안정할 수 있습니다.
→ 잠시 후 다시 시도: `npm install`

## 5. "Open in Browser" 팝업이 안 떠요

### 증상
`node server.js`를 실행했는데 팝업 알림이 나타나지 않음

### 해결법

팝업을 놓쳤거나 브라우저가 차단했을 수 있습니다.

**수동으로 열기:**
1. 하단 **"PORTS"** 탭 클릭
2. 3000번 포트 행 확인
3. 🌐 **지구본 아이콘** 클릭 → 새 탭에서 앱이 열림

**PORTS 탭이 안 보이면:**
- 하단 패널에서 "PROBLEMS", "OUTPUT", "TERMINAL", "PORTS" 중 "PORTS"를 찾아 클릭
- 또는 `Ctrl + Shift + P` → "Focus on Ports View" 검색

---

## 6. 친구가 내 앱에 접속 못 해요

### 증상
URL을 공유했는데 친구가 열면 에러 페이지가 나옴

### 해결법

**포트가 Private 상태일 가능성이 높습니다:**
1. 하단 "PORTS" 탭 클릭
2. 3000번 포트의 "Visibility" 열 확인
3. **"Private"** 이면 → 우클릭 → "Port Visibility" → **"Public"** 으로 변경
4. URL을 다시 복사해서 친구에게 보내기

**서버가 꺼져 있을 수 있습니다:**
- Codespace 터미널에서 서버가 실행 중인지 확인
- 꺼져 있으면: `node server.js` 다시 실행

**Codespace가 중지(Stop)되어 있을 수 있습니다:**
- Codespace 브라우저 탭이 닫혀 있으면 서버도 꺼짐
- Codespace를 다시 열고 서버를 실행하세요

---

## 7. Codespace를 닫았더니 앱도 꺼졌어요

### 증상
Codespace 탭을 닫았더니 친구가 앱에 접속 못 함

### 해결법

**이것은 정상입니다!**
Codespace를 닫으면 서버도 종료됩니다.
친구에게 보여줄 때는 Codespace를 열어둔 상태여야 합니다.

> 💡 "24시간 365일 앱을 공개하고 싶어요!"
> → 그건 **Step 6 (Render)** 에서 배웁니다.
> Render에 배포하면 Codespace를 닫아도 앱이 계속 실행됩니다.

## 8. 무료 시간을 다 썼다는 알림이 나와요

### 증상
"You've used all of your included usage" 메시지

### 해결법

**원인:**
Codespace를 사용 후 중지(Stop)하지 않아서 시간이 계속 소모된 것일 가능성이 높습니다.

**즉시 조치:**
1. 모든 Codespace를 중지(Stop)하세요
2. github.com → Your codespaces → "..." → "Stop codespace"

**다음 달까지 기다리기:**
- 매월 1일에 무료 시간이 리셋됩니다
- 그때까지는 내 컴퓨터에서 Node.js로 실행하세요 (Step 1~4 방식)

**예방법:**
- 수업 끝나면 반드시 Codespace 중지!
- github.com → Settings → Billing → Codespaces에서 사용량 확인 습관

## 9. .env 파일을 실수로 GitHub에 올려버렸어요

### 증상
git push 후 GitHub에서 .env 파일이 보임 (API 키가 노출됨!)

### 해결법

**Step 1: .env를 Git 추적에서 제거**
```
git rm --cached .env
```

**Step 2: .gitignore 확인**
`.gitignore` 파일을 열어서 `.env`가 포함되어 있는지 확인:
```
.env
node_modules/
```

없으면 추가하세요.

**Step 3: 커밋 + 푸시**
```
git add .gitignore
git commit -m ".env 파일 제거 및 .gitignore 수정"
git push
```

**Step 4: API 키 변경 (중요!)**
- 노출된 Supabase Key를 Supabase 대시보드에서 재발급하세요
- 새 키를 .env 파일에 입력

> ⚠️ 한 번이라도 push된 비밀 키는 이미 노출된 것입니다.
> 반드시 키를 재발급하세요!

## 10. Live Share가 작동 안 해요

### 증상
Live Share 아이콘을 클릭했는데 반응이 없거나 에러

### 해결법

**확장이 설치되지 않았을 수 있습니다:**
1. 왼쪽 사이드바에서 확장(Extensions) 아이콘 클릭 (네모 4개 모양)
2. 검색창에 "Live Share" 입력
3. "Live Share" 확장을 **Install** 클릭
4. 설치 완료 후 다시 시도

**로그인이 필요할 수 있습니다:**
- Live Share 시작 시 GitHub 계정 로그인을 요청하면 승인하세요

**상대방도 브라우저에서 열어야 합니다:**
- 초대 링크를 받은 친구도 브라우저에서 링크를 열면 됩니다

## 11. Codespace에서 서버가 갑자기 꺼졌어요

### 증상
잘 실행되던 서버가 에러 없이 종료됨

### 해결법

**비활성 시간 초과:**
- Codespace를 오래 방치하면 자동으로 중지될 수 있습니다
- 터미널에서 다시 `node server.js` 실행

**터미널에서 실수로 Ctrl+C를 누른 경우:**
- `Ctrl + C`는 서버를 종료하는 명령입니다
- 다시 `node server.js`를 입력하세요

---

## 12. 포트 번호가 3000이 아닌 다른 번호로 열려요

### 증상
PORTS 탭에 3000이 아닌 다른 포트 번호가 보임

### 해결법

- server.js에서 포트 번호를 확인하세요:
```javascript
const PORT = process.env.PORT || 3000;
```
- .env 파일에 `PORT=다른번호`가 설정되어 있다면 제거하거나 3000으로 변경
- 또는 PORTS 탭에 표시된 포트 번호 그대로 사용해도 됩니다

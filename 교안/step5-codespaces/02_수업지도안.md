# Step 5 수업 지도안 — GitHub Codespaces

**대상:** 중학교 1학년 ~ 고등학교 3학년
**시간:** 2차시 x 45분 (총 90분)
**준비물:** 노트북 또는 태블릿 (브라우저만 있으면 됨), 인터넷 연결
**선수 학습:** Step 4 (GitHub 계정 + 저장소 push 완료)

---

## 이번 단계에서 바뀐 것

| 항목 | Step 4까지 | Step 5 |
|------|-----------|--------|
| 코드 편집 | 내 컴퓨터 VS Code | 브라우저 VS Code (Codespaces) |
| 서버 실행 | 내 컴퓨터 터미널 | Codespaces 터미널 |
| 앱 접속 | localhost:3000 (나만) | 공개 URL (누구나) |
| 설치 필요 | Node.js + VS Code | 없음 (브라우저만!) |

---

## 차시별 계획

### 1차시 — Codespaces 시작하기 (45분)

| 시간 | 활동 | 교사 행동 | 학생 활동 | 비고 |
|------|------|---------|---------|------|
| 0~5분 | 도입 | "학교 컴퓨터에 Node.js 설치 못 하면 어떡하죠?" 질문 | 자유 답변 | 동기부여 |
| 5~10분 | 개념 설명 | Codespaces = 클라우드 PC방 비유, localhost vs 공개 URL 비교 | 개념교안 읽기 | 00_개념교안.md |
| 10~20분 | 시범 시연 | 교사 화면으로 Codespace 생성 → 서버 실행 → 앱 확인 과정 보여주기 | 따라보기 | 교사가 먼저 시연 |
| 20~35분 | 실습 | 순회 지도 | Codespace 만들기 → npm install → .env 설정 → 서버 실행 | 01_설정가이드 + 03_실습지침서 |
| 35~42분 | 포트 공개 | "PORTS 탭에서 Public 설정" 안내 | 포트 Public으로 변경 → URL 복사 → 옆 친구에게 공유 | 핵심 활동! |
| 42~45분 | 정리 | "오늘 배운 것 3줄로 정리" 안내 | 정리 및 Codespace 중지(Stop) | ⚠️ 중지 필수! |

### 2차시 — 코드 수정 + 협업 (45분)

| 시간 | 활동 | 교사 행동 | 학생 활동 | 비고 |
|------|------|---------|---------|------|
| 0~5분 | 복습 | "지난 시간에 만든 Codespace를 다시 열어볼까요?" | 기존 Codespace 열기 (또는 새로 생성) | github.com → Your codespaces |
| 5~10분 | 서버 재시작 | "터미널에서 다시 node server.js" | 서버 실행 + 앱 확인 | 포트도 다시 Public 설정 |
| 10~25분 | 코드 수정 | "style.css에서 배경색을 바꿔봅시다" | CSS 수정 → 새로고침 → 변경 확인 | 바이브코딩 프롬프트 활용 |
| 25~35분 | Git 연동 | "수정한 코드를 GitHub에 올려봅시다" | Codespace 터미널에서 add → commit → push | Codespace 안에서 Git 사용 |
| 35~42분 | 심화: Live Share | "친구와 같은 코드를 동시에 편집해봅시다" (선택) | Live Share 링크 생성 → 친구 초대 → 동시 편집 | 시간 부족 시 생략 가능 |
| 42~45분 | 정리 + 중지 | Codespace 중지 안내, Step 6 예고 | Codespace Stop + 소감 | ⚠️ 중지 필수! |

## 💡 교사 메모

> **환경 준비가 거의 필요 없습니다:**
> - 학교 컴퓨터에 아무것도 설치 안 해도 됩니다.
> - 크롬북, iPad 태블릿에서도 가능합니다.
> - VS Code와 동일한 UI이므로 Step 1~3에서 VS Code를 사용한 학생은 바로 적응합니다.
>
> **Codespace 중지 루틴을 만드세요:**
> - 수업 종료 3분 전에 "Codespace 중지했나요?" 확인하세요.
> - 중지 방법: github.com → 상단 프로필 → Your codespaces → "..." → Stop codespace
> - 비활성 30분 후 자동 중지되지만, 수동 중지를 습관화하면 안전합니다.
> - 무료 월 60시간이므로 매일 수업해도 충분하지만, 켜놓고 잊으면 빨리 소모됩니다.
>
> **학생이 자주 막히는 부분:**
> - Codespace 생성 시 "Code" 버튼의 "Local" 탭이 아닌 "Codespaces" 탭을 선택해야 합니다.
> - .env 파일에 Supabase 키를 잘못 입력하는 경우가 많습니다. 복사-붙여넣기를 안내하세요.
> - 포트를 Public으로 변경하지 않으면 친구가 접속할 수 없습니다.
> - "Open in Browser" 팝업을 놓칠 수 있습니다. PORTS 탭에서 수동으로 열 수 있다고 안내하세요.
>
> **시간 부족 시 생략 가능:**
> - 2차시의 Live Share 활동은 심화 활동입니다. 시간이 부족하면 생략해도 됩니다.
> - 코드 수정 + commit/push까지만 해도 충분한 학습 목표를 달성합니다.
>
> **Step 6 예고:**
> - "Codespace를 닫으면 앱도 꺼지죠? 24시간 켜놓으려면 어떻게 할까요?"
> - Step 6 (Render)에서 무료 배포를 배웁니다.
>
> **평가 기준 예시:**
> - 🟢 Codespace에서 서버를 실행하고 앱을 확인할 수 있다
> - 🟡 포트를 Public으로 변경하고 친구에게 URL을 공유할 수 있다
> - 🔴 코드를 수정하고 commit/push 또는 Live Share를 할 수 있다

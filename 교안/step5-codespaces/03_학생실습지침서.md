# 따라하면 완성! Codespaces에서 내 앱 실행하기

> **대상:** 중학교 1학년 ~ 고등학교 3학년
> **소요 시간:** 약 30~40분
> **준비물:** 브라우저 하나 (Node.js 설치 불필요, VS Code 설치 불필요)

---

## 시작 전 체크리스트

- [ ] GitHub 계정에 로그인했다 (Step 4에서 만든 계정)
- [ ] edu-padlet 저장소가 GitHub에 있다
- [ ] Supabase URL과 Key를 알고 있다 (Step 3에서 사용한 것)
- [ ] 브라우저가 열려 있다

---

## A단계: GitHub에서 Codespace 만들기

### A-1. 내 저장소 열기

1. 브라우저에서 [github.com](https://github.com) 접속
2. 로그인
3. 내 `edu-padlet` 저장소로 이동

### A-2. Codespace 생성

1. 초록색 **"Code"** 버튼 클릭
2. **"Codespaces"** 탭 선택 (⚠️ "Local"이 아닙니다!)
3. **"Create codespace on main"** 클릭
4. 30초~1분 기다리기...

> ✅ **확인 포인트:** 브라우저에 VS Code 화면이 나타났나요?
> 왼쪽에 파일 목록, 가운데에 편집기, 아래에 터미널이 보이면 성공!

---

## B단계: 서버 실행하기

### B-1. 터미널 확인

화면 아래에 터미널이 보이나요?
안 보이면: `Ctrl` + `` ` `` 또는 상단 **Terminal → New Terminal**

### B-2. 패키지 설치

터미널에 입력:
```
npm install
```
1~2분 기다려주세요. "added XX packages" 메시지가 나오면 완료!

### B-3. 환경 변수 설정

터미널에 입력:
```
cp .env.example .env
```

### B-4. Supabase 키 입력

1. 왼쪽 파일 목록에서 `.env` 파일 클릭
2. 아래 내용을 수정:

```
SUPABASE_URL=여기에_본인의_Supabase_URL_붙여넣기
SUPABASE_KEY=여기에_본인의_Supabase_Key_붙여넣기
```

> 💡 Step 3에서 사용했던 URL과 Key입니다.
> 기억나지 않으면: [supabase.com](https://supabase.com) → 프로젝트 → Settings → API

### B-5. 서버 실행!

터미널에 입력:
```
node server.js
```

### B-6. 앱 열기

- **팝업이 뜨면:** "Open in Browser" 클릭
- **팝업이 안 뜨면:** 하단 "PORTS" 탭 → 3000번 포트 → 🌐 아이콘 클릭

> ✅ **확인 포인트:** 새 탭에서 패들렛 앱이 보이나요?
> 카드가 표시되는 보드 화면이 나오면 성공!

> 🆘 **막혔나요?** → [05_트러블슈팅.md](./05_트러블슈팅.md)를 확인하세요

## C단계: 앱 동작 확인

### C-1. 카드 추가

1. 앱에서 **카드 추가** 버튼 클릭
2. 제목과 내용 입력
3. 색상 선택
4. 저장!

### C-2. 카드 삭제

1. 카드의 **삭제** 버튼 클릭
2. 카드가 사라지는지 확인

### C-3. 새로고침 후 확인

1. 브라우저에서 `F5` 또는 `Ctrl+R`
2. 카드가 여전히 남아있으면 성공! (Supabase에 저장되니까요)

> ✅ **확인 포인트:** 카드 추가/삭제가 되고, 새로고침해도 데이터가 유지되나요?
> Step 1에서 Notion에서 데이터가 보였듯, 이번엔 Supabase에서 데이터가 저장되고 있어요!

## D단계: 포트 공개 + 친구에게 URL 공유

이것이 이번 Step의 가장 중요한 활동입니다!

### D-1. 포트를 Public으로 변경

1. 하단 **"PORTS"** 탭 클릭
2. 3000번 포트를 **우클릭**
3. **"Port Visibility"** → **"Public"** 선택

### D-2. URL 복사

1. PORTS 탭에서 3000번 포트의 주소를 확인
2. 주소를 우클릭 → **"Copy Local Address"**
   또는 주소 옆 복사 아이콘 클릭

### D-3. 친구에게 공유!

1. 옆 친구에게 URL을 보내세요 (카카오톡, 메모장 등)
2. 친구가 링크를 열어서 앱이 보이는지 확인
3. **친구가 카드를 추가해보세요!**
4. 내 화면에서 새로고침 → 친구가 추가한 카드가 보이나요?

> ✅ **확인 포인트:** 친구가 내 앱에 접속해서 카드를 추가/삭제할 수 있나요?
> "localhost에서는 나만 볼 수 있었는데, 이제는 누구나 볼 수 있다!"

## E단계: 코드 수정하고 즉시 확인하기

Codespaces의 강점은 **수정하면 바로 확인**할 수 있다는 것!

### E-1. CSS 배경색 변경

1. 왼쪽 파일 목록에서 `public/style.css` 클릭
2. `background-color` 값을 찾아서 다른 색으로 변경

예시:
```css
/* 변경 전 */
background-color: #f0f0f0;

/* 변경 후 — 원하는 색으로! */
background-color: #ffe4e1;  /* 연분홍 */
```

3. 파일 저장: `Ctrl + S`
4. 앱이 열린 브라우저 탭으로 이동
5. 새로고침 (`F5`)
6. 배경색이 바뀌었나요?

> ✅ **확인 포인트:** CSS를 수정하고 새로고침했을 때 변경이 바로 반영되나요?

### E-2. 수정 내용을 GitHub에 올리기

Codespace 안에서 바로 commit + push 할 수 있습니다!

터미널에 입력:
```
git add .
git commit -m "배경색 변경"
git push
```

> ✅ **확인 포인트:** github.com에서 내 저장소를 열어보세요. 커밋 기록에 "배경색 변경"이 보이나요?

## F단계 (심화): Live Share로 실시간 협업

> 🔴 이 단계는 심화 활동입니다. 시간이 남으면 도전해보세요!

Live Share를 사용하면 친구와 **같은 코드를 동시에 편집**할 수 있습니다.
Google Docs에서 같이 문서를 편집하는 것처럼요!

### F-1. Live Share 시작

1. 왼쪽 사이드바에서 **Live Share** 아이콘 클릭 (사람 모양 + 화살표)
2. "Share" 버튼 클릭
3. 초대 링크가 클립보드에 복사됨

> 💡 Live Share 아이콘이 안 보이면, 왼쪽 사이드바 하단 확장 아이콘에서 "Live Share"를 검색해 설치하세요.

### F-2. 친구 초대

1. 복사된 링크를 친구에게 전송
2. 친구가 링크를 클릭하면 같은 코드가 열림
3. 서로 다른 파일을 편집하거나, 같은 파일을 동시에 수정해보세요!

> ✅ **확인 포인트:** 내가 코드를 수정하면 친구 화면에도 바로 반영되나요?

## G단계: Codespace 중지 (필수!)

**수업이 끝나면 반드시 Codespace를 중지하세요!**
안 끄면 무료 시간(월 60시간)이 계속 소모됩니다.

### 방법 1: GitHub 웹사이트에서

1. [github.com](https://github.com) 접속
2. 상단 프로필 아이콘 클릭
3. **"Your codespaces"** 선택
4. 내 Codespace 옆 **"..."** 클릭
5. **"Stop codespace"** 클릭

### 방법 2: Codespace 안에서

1. 좌하단 **"Codespaces: ..."** 텍스트 클릭
2. 상단에 나타나는 명령 팔레트에서 **"Stop Current Codespace"** 선택

> ✅ **확인 포인트:** Codespace 상태가 "Stopped"로 바뀌었나요?

> ⚠️ **다시 열고 싶을 때:** github.com → Your codespaces → 내 Codespace 클릭 → 바로 다시 열립니다!
> 이전에 작업한 내용이 그대로 남아 있어요.

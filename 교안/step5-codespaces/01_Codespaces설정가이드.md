# Codespace 만들기 — 버튼 하나로 시작

> **대상:** 중학교 1학년 ~ 고등학교 3학년
> **선수 학습:** Step 4 (GitHub 계정 + 저장소) 완료 필수
> **소요 시간:** 10~15분

---

## 시작 전 체크리스트

- [ ] GitHub 계정이 있다 (Step 4에서 만들었어요)
- [ ] edu-padlet 저장소가 GitHub에 있다 (내 Fork 또는 직접 push한 저장소)
- [ ] 브라우저가 열려 있다 (Chrome, Edge, Safari 등)

> 이것만 있으면 됩니다. Node.js 설치 불필요. VS Code 설치 불필요.

---

## Step 1: GitHub에서 내 저장소 열기

1. 브라우저에서 [github.com](https://github.com) 접속
2. 로그인 후 내 프로필 → "Your repositories" 클릭
3. `edu-padlet` 저장소 클릭

> 💡 선생님이 별도 저장소를 알려줬다면 그 주소로 이동하세요.

---

## Step 2: Codespace 만들기

1. 저장소 페이지에서 초록색 **"Code"** 버튼 클릭
2. 상단 탭에서 **"Codespaces"** 탭 선택 (기본은 "Local" 탭)
3. **"Create codespace on main"** 클릭

```
  ┌────────────────────────────┐
  │  Code ▼                    │
  ├────────────────────────────┤
  │  Local  │ ★Codespaces★    │
  ├────────────────────────────┤
  │                            │
  │  Codespaces               │
  │                            │
  │  ┌──────────────────────┐  │
  │  │ + Create codespace   │  │
  │  │   on main            │  │
  │  └──────────────────────┘  │
  │                            │
  └────────────────────────────┘
```

4. **3~5분 기다리기** → 브라우저에서 VS Code가 열립니다!

> ⏳ 처음 만들 때는 **3~5분** 걸릴 수 있습니다. 로딩 화면이 나오면 기다려주세요!
> 두 번째부터는 1분 이내로 빨라집니다.

> ✅ 화면에 VS Code가 나타나고, 아래쪽에 **터미널(검은 창)**이 보이면 성공입니다!

---

## ⚠️ 주의: `.github.dev`는 Codespace가 아닙니다!

GitHub에서 `.`(마침표) 키를 누르거나 URL을 `.github.dev`로 바꾸면 열리는 편집기는 **Codespace가 아닙니다.**

| | `.github.dev` (웹 편집기) | Codespace |
|--|:---:|:---:|
| 파일 편집 | ✅ | ✅ |
| **터미널** | **❌ 없음** | **✅ 있음** |
| **node server.js 실행** | **❌ 불가** | **✅ 가능** |
| 서버 실행 + 포트 공개 | ❌ | ✅ |

**터미널이 없으면 `.github.dev`로 열린 것**입니다. 반드시 위의 Step 2 방법(Code 버튼 → Codespaces 탭)으로 열어야 합니다!

### 이미 만든 Codespace를 다시 열려면

1. **https://github.com/codespaces** 접속
2. 목록에서 내 Codespace 클릭
3. 3~4분 대기 → 터미널이 포함된 VS Code 열림

---

## Step 3: 터미널 확인

Codespace가 열리면 하단에 터미널이 이미 열려 있을 수 있습니다.

만약 터미널이 보이지 않으면:
- 키보드: `Ctrl` + `` ` `` (백틱, 숫자 1 왼쪽 키)
- 또는 상단 메뉴: **Terminal → New Terminal**

터미널에 커서가 깜빡이면 준비 완료!

---

## Step 4: 서버 실행 준비

터미널에 아래 명령어를 **한 줄씩** 입력하세요.

### 4-1. 패키지 설치

```
npm install
```

여러 줄의 설치 로그가 나옵니다. 1~2분 정도 기다려주세요.
"added XX packages" 메시지가 나오면 완료!

### 4-2. 환경 변수 파일 만들기

```
cp .env.example .env
```

> 이 명령은 `.env.example` 파일을 복사해서 `.env` 파일을 만듭니다.

### 4-3. .env 파일에 Supabase 키 입력

왼쪽 파일 목록에서 `.env` 파일을 클릭합니다.

```
SUPABASE_URL=여기에_Supabase_URL_입력
SUPABASE_KEY=여기에_Supabase_Key_입력
```

Step 3에서 사용했던 Supabase URL과 Key를 붙여넣으세요.

> 💡 Supabase URL과 Key가 기억나지 않는다면:
> [supabase.com](https://supabase.com) → 내 프로젝트 → Settings → API에서 확인

---

## Step 5: 서버 실행!

터미널에 입력:

```
node server.js
```

```
  터미널 출력:
  ┌──────────────────────────────────────────┐
  │  $ node server.js                        │
  │  ✅ 서버가 3000번 포트에서 시작됨          │
  └──────────────────────────────────────────┘
```

---

## Step 6: 앱 열기

서버가 시작되면 **팝업 알림**이 나타납니다.

```
  ┌──────────────────────────────────────────────────┐
  │  Your application running on port 3000 is         │
  │  available.                                       │
  │                                                   │
  │  [ Open in Browser ]   [ Make Public ]            │
  └──────────────────────────────────────────────────┘
```

**"Open in Browser"** 를 클릭하세요!
새 탭에서 우리의 패들렛 앱이 열립니다.

### 팝업이 안 뜨는 경우

1. 하단 **"PORTS"** 탭을 클릭합니다
2. 3000번 포트 행에서 **지구본 아이콘** (🌐)을 클릭합니다
3. 새 탭에서 앱이 열립니다

```
  ┌─────────────────────────────────────────────────────┐
  │  PORTS                                               │
  ├────────┬──────────┬───────────┬─────────────────────┤
  │  Port  │ Address  │ Visibility│ Action               │
  ├────────┼──────────┼───────────┼─────────────────────┤
  │  3000  │ https:// │ Private   │ 🌐 (브라우저로 열기)  │
  └────────┴──────────┴───────────┴─────────────────────┘
```

> ✅ 브라우저에서 앱이 보이면 성공입니다!

---

## Step 7: 포트 공개 — 친구에게 URL 공유하기

지금은 포트가 **Private** 상태입니다. 나만 볼 수 있어요.
친구에게 보여주려면 **Public**으로 바꿔야 합니다.

### 방법:

1. 하단 **"PORTS"** 탭 클릭
2. 3000번 포트를 **우클릭**
3. **"Port Visibility"** → **"Public"** 선택
4. URL을 **복사** (포트 행의 주소를 우클릭 → "Copy Local Address")
5. 친구에게 URL을 보내세요!

```
  친구에게 보내는 메시지:
  ┌───────────────────────────────────────────────────┐
  │  "내가 만든 앱 볼래? 이 링크 눌러봐!"               │
  │  https://fluffy-space-abc123-3000.app.github.dev  │
  └───────────────────────────────────────────────────┘
```

> ✅ 친구가 링크를 눌러서 앱이 보이면 대성공!

## 중요한 주의사항

### ⚠️ Codespace를 닫으면 서버도 꺼집니다

Codespace 브라우저 탭을 닫으면 서버가 종료됩니다.
친구에게 보여줄 때는 **Codespace를 열어둔 상태**여야 합니다.

> 💡 "24시간 공개하고 싶어요!" → Step 6 (Render)에서 배웁니다.

### ⚠️ 사용 후 반드시 Codespace 중지(Stop)

1. [github.com](https://github.com) 접속
2. 상단 프로필 아이콘 클릭 → **"Your codespaces"**
3. 내 Codespace 옆 **"..."** 클릭 → **"Stop codespace"**

```
  ┌──────────────────────────────────────────┐
  │  Your codespaces                          │
  │                                          │
  │  edu-padlet (main)         ...           │
  │  Active                    ┌──────────┐  │
  │                            │ Stop      │  │
  │                            │ Delete    │  │
  │                            │ Open in...│  │
  │                            └──────────┘  │
  └──────────────────────────────────────────┘
```

**안 끄면 무료 시간(월 60시간)이 계속 소모됩니다!**

## 💡 교사 메모

> **사전 준비:**
> - Codespace 생성에는 GitHub 계정이 필요합니다 (Step 4에서 이미 완료).
> - 교사가 미리 Codespace를 한 번 만들어보면 수업이 원활합니다.
> - .env.example 파일이 저장소에 있는지 확인하세요.
>
> **무료 시간 관리:**
> - 무료 60시간/월은 수업용으로 충분하지만, 학생들이 사용 후 Codespace를 중지(Stop)하도록 반드시 안내하세요.
> - 수업 끝날 때 "Codespace 껐나요?" 확인 루틴을 만들면 좋습니다.
> - 비활성 상태 30분 후 자동 중지되지만, 확실하게 수동 중지를 안내하세요.
>
> **환경 제약 없음:**
> - 학교 컴퓨터에 아무것도 설치할 필요 없습니다.
> - 크롬북, 태블릿(iPad 등)에서도 브라우저만 있으면 됩니다.
> - 학생 개인 기기 사용 가능 (BYOD 환경에 적합).
>
> **Supabase 키 관리:**
> - .env 파일은 Codespace 안에서만 존재합니다. GitHub 저장소에는 올라가지 않습니다.
> - .gitignore에 .env가 포함되어 있는지 확인하세요.
> - 학생마다 다른 Supabase 프로젝트를 사용하면 데이터가 섞이지 않습니다.

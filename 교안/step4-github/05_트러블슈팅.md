# 자주 막히는 문제 해결법 — Step 4: GitHub

> Git/GitHub 실습 중 자주 발생하는 문제와 해결법입니다.
> 에러 메시지를 Ctrl+F (Cmd+F)로 검색하면 빠르게 찾을 수 있습니다.

---

## 1. `git: command not found`

### 증상
터미널에 `git --version`을 입력했는데 아래 메시지가 나옴:
```
zsh: command not found: git
```

### 원인
Git이 설치되어 있지 않습니다.

### 해결법

**Mac:**
```
xcode-select --install
```
팝업이 뜨면 "설치" 클릭 → 몇 분 후 완료 → 터미널을 닫았다가 다시 열기

**Windows:**
1. https://git-scm.com 에서 다운로드
2. 설치 (모든 옵션 기본값 Next)
3. VS Code를 완전히 껐다가 다시 열기

## 2. Permission denied (publickey)

### 증상
`git push`를 했는데:
```
Permission denied (publickey).
fatal: Could not read from remote repository.
```

### 원인
SSH 방식으로 연결을 시도했습니다. 초보자는 HTTPS 방식이 편합니다.

### 해결법

저장소 주소를 HTTPS로 변경:
```
git remote set-url origin https://github.com/내이름/edu-padlet.git
```

확인:
```
git remote -v
```
`https://` 로 시작하면 OK!

## 3. `fatal: not a git repository`

### 증상
`git add .` 또는 `git commit`을 했는데:
```
fatal: not a git repository (or any of the parent directories): .git
```

### 원인
`git init`을 실행하지 않았거나, 다른 폴더에서 명령어를 실행하고 있습니다.

### 해결법

1. 현재 위치 확인:
```
pwd
```
`~/edu-padlet/code/step3-supabase` 가 맞는지 확인

2. 맞다면 `git init` 실행:
```
git init
```

3. 폴더가 다르다면 이동:
```
cd ~/edu-padlet/code/step3-supabase
```

## 4. .env 파일이 GitHub에 올라갔다!

### 증상
GitHub 저장소에 `.env` 파일이 보입니다. API 키가 노출되었습니다!

### 긴급 대응법

**1단계: .gitignore에 .env 추가 (아직 안 했다면)**
`.gitignore` 파일에 `.env`가 있는지 확인하세요.

**2단계: Git에서 .env 파일 제거 (로컬 파일은 유지)**
```
git rm --cached .env
git commit -m ".env 파일을 GitHub에서 제거"
git push
```

**3단계: API 키 재발급 (매우 중요!)**
한 번이라도 GitHub에 올라간 키는 **이미 노출된 것**입니다.
- Supabase 대시보드 → Settings → API → 키를 재생성
- 새 키를 `.env` 파일에 입력

> ⚠️ `git rm --cached .env`는 GitHub에서만 삭제하고, 내 컴퓨터의 `.env` 파일은 그대로 남깁니다.

> 💡 **교사 메모:** 이 상황이 발생하면 당황하지 말고 위 단계를 따라가세요. 학생들에게 "비밀번호를 인터넷에 올린 것과 같다"고 설명하면 이해가 빠릅니다. 키 재발급은 반드시 해야 합니다.

## 5. node_modules가 GitHub에 올라갔다

### 증상
GitHub 저장소에 `node_modules` 폴더가 보입니다. 파일이 수천 개!

### 원인
`.gitignore` 파일을 만들기 전에 `git add .`를 했습니다.

### 해결법

```
git rm -r --cached node_modules
git commit -m "node_modules 제거"
git push
```

`.gitignore`에 `node_modules/`가 있는지 확인하세요.

## 6. `rejected - non-fast-forward`

### 증상
`git push`를 했는데:
```
! [rejected]        main -> main (non-fast-forward)
error: failed to push some refs to '...'
```

### 원인
GitHub에 있는 코드가 내 컴퓨터보다 앞서 있습니다. (다른 컴퓨터에서 올렸거나 GitHub에서 직접 수정한 경우)

### 해결법

먼저 GitHub의 변경사항을 받아온 후 다시 push:
```
git pull --rebase origin main
git push
```

## 7. GitHub 비밀번호 입력이 안 됨 / 인증 실패

### 증상
`git push`를 했는데 비밀번호를 입력해도 인증에 실패함:
```
remote: Support for password authentication was removed on August 13, 2021.
```

### 원인
2021년 8월부터 GitHub는 비밀번호 대신 **Personal Access Token(PAT)** 을 사용합니다.

### 해결법

1. GitHub 접속 → 오른쪽 위 프로필 아이콘 → **Settings**
2. 왼쪽 맨 아래 **Developer settings** 클릭
3. **Personal access tokens** → **Tokens (classic)** → **Generate new token (classic)**
4. Note: `edu-padlet` 입력
5. Expiration: 90 days
6. Select scopes: `repo` 체크
7. **Generate token** 클릭
8. 생성된 토큰을 **복사** (한 번만 보임! 메모장에 붙여넣기)
9. 터미널에서 비밀번호 대신 이 토큰을 입력

> ⚠️ 토큰은 한 번만 보여줍니다! 꼭 안전한 곳에 저장하세요.

> 💡 **교사 메모:** Mac은 보통 `git push` 시 브라우저 인증 창이 떠서 토큰이 필요 없는 경우가 많습니다. Windows에서 Git Credential Manager가 설치되어 있으면 마찬가지입니다. 토큰이 필요한 경우에만 이 방법을 사용하세요.

## 8. 사용자명/이메일 설정 안 했다는 에러

### 증상
`git commit`을 했는데:
```
Author identity unknown
*** Please tell me who you are.
```

### 원인
`git config`로 이름과 이메일을 설정하지 않았습니다.

### 해결법

```
git config --global user.name "내 이름"
git config --global user.email "내@이메일.com"
```

설정 후 다시 commit:
```
git commit -m "첫 번째 커밋: 우리 반 패들렛"
```

## 9. `fatal: remote origin already exists`

### 증상
`git remote add origin ...`을 했는데:
```
fatal: remote origin already exists.
```

### 원인
이미 원격 저장소가 등록되어 있습니다.

### 해결법

이미 설정되어 있으니 **그냥 넘어가도 됩니다!**

주소를 바꾸고 싶다면:
```
git remote set-url origin https://github.com/내이름/edu-padlet.git
```

현재 설정 확인:
```
git remote -v
```

## 10. 한글 파일명이 깨져서 보임

### 증상
`git status`를 했는데 한글 파일명이 숫자+기호로 깨져서 나옴:
```
"\355\225\234\352\270\200\355\214\214\354\235\274.js"
```

### 원인
Git이 한글을 인코딩 처리하고 있습니다.

### 해결법

```
git config --global core.quotepath false
```

다시 `git status`를 하면 한글이 정상적으로 보입니다.

## 11. 커밋 메시지를 잘못 적었다

### 증상
`git commit -m "오타가 있는 메시지"`를 이미 실행해버렸습니다.

### 해결법

아직 push 하기 전이라면:
```
git commit --amend -m "수정된 올바른 메시지"
```

이미 push 했다면 그냥 두세요. 다음 커밋에 잘 적으면 됩니다!

## 12. `git add .` 했는데 아무 반응이 없다

### 증상
`git add .`을 했는데 아무 메시지도 안 나옴.

### 원인
정상입니다! `git add`는 성공하면 아무 말도 안 합니다.

### 확인법

```
git status
```

초록색으로 파일 목록이 나오면 정상적으로 추가된 것입니다.

## 문제 해결 순서 요약

뭔가 안 될 때 이 순서로 확인하세요:

1. **현재 폴더가 맞나?** → `pwd`로 확인
2. **Git이 시작되어 있나?** → `git status`로 확인
3. **인터넷이 연결되어 있나?** → `ping github.com`으로 확인
4. **에러 메시지를 읽자!** → 에러 메시지에 답이 있습니다
5. **AI에게 물어보자!** → 에러 메시지를 복사해서 Claude.ai에 붙여넣기

```
이 에러가 났습니다. 원인과 해결법을 알려주세요:
[에러 메시지 붙여넣기]
```

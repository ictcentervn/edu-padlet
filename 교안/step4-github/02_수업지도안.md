# Step 4: GitHub 수업 지도안

**대상:** 중학교 1학년 ~ 고등학교 3학년
**시간:** 2차시 x 45분
**준비물:** 노트북, 인터넷, 이메일 계정, Step 3 완성 코드

---

## 사전 준비 (교사)

- [ ] 학교 네트워크에서 github.com 접속 가능 확인
- [ ] Git 설치 여부 확인 (학생 노트북에서 `git --version`)
- [ ] Windows 학생용 Git 설치 파일 USB에 준비
- [ ] 교사 본인의 GitHub 저장소 미리 만들어서 시범용으로 준비
- [ ] 학생들이 이메일 주소를 알고 있는지 확인

---

## 1차시 — Git 개념 + GitHub 가입 + 첫 Push (45분)

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~5분 | 도입: "USB 잃어버린 이야기" | 질문 던지기 — "USB 잃어버린 적 있나요?" "파일 날아간 적?" | 학생 경험 끌어내기 |
| 5~15분 | 개념 설명: Git과 GitHub | 00_개념교안.md 다이어그램 보여주기. Git = 세이브 포인트, GitHub = 구글 드라이브 비유 설명 | 화면 공유 또는 프로젝터 |
| 15~25분 | GitHub 가입 실습 | 01_가이드를 따라 학생들과 함께 가입. 사용자명 정하는 팁 안내 | 이메일 인증 시간 필요 |
| 25~30분 | Git 설치 확인 + 초기 설정 | `git --version`, `git config` 함께 실행 | Windows 학생 주의 |
| 30~40분 | 첫 Push 시범 + 실습 | 교사가 먼저 시범 → 학생들 따라하기. `git init` → `add` → `commit` → `push` | 03_실습지침서 A~E단계 |
| 40~45분 | 정리 + 확인 | GitHub에서 내 코드 보이는지 확인. 못 올린 학생 파악 | 다음 시간에 이어서 |

### 1차시 핵심 포인트
- **외울 필요 없다.** 명령어 3개만 기억: `add`, `commit`, `push`
- `.env` 파일 올리면 안 된다는 것만 꼭 강조
- 가입에 시간이 걸리므로 여유 있게 진행

> 💡 **교사 메모:** GitHub 가입 시 이메일 인증이 필요합니다. 학교 이메일이 안 되는 경우가 있으니, 개인 이메일을 쓰게 하거나 미리 가입을 과제로 내주는 것도 방법입니다. 수업 전에 과제로 "GitHub 가입해오기"를 내면 시간을 절약할 수 있습니다.

## 2차시 — 코드 수정 + 재업로드 + 친구 코드 Clone (45분)

| 시간 | 활동 | 교사 행동 | 비고 |
|------|------|---------|------|
| 0~5분 | 복습: "지난 시간에 한 것" | "add-commit-push 가 뭐였죠?" 질문 | 간단 복습 |
| 5~15분 | 코드 수정 → 재업로드 실습 | 코드에 작은 변경 → `git add .` → `git commit -m "메시지"` → `git push` | 변경-저장-업로드 사이클 체험 |
| 15~25분 | 친구 코드 Clone 실습 | 옆자리 친구의 GitHub 주소를 받아 `git clone` 실행 → 코드 실행해보기 | 짝 활동 |
| 25~35분 | 포트폴리오 설명 + README 작성 | GitHub 프로필이 포트폴리오가 되는 것 설명. README.md 간단히 작성해보기 | 🟡 기본 이상 |
| 35~40분 | 커밋 히스토리 살펴보기 | GitHub에서 커밋 목록 클릭 → 변경 내역 확인 | "타임머신" 개념 체감 |
| 40~45분 | 정리 + 소감 나누기 | "내 GitHub 주소를 친구에게 자랑해보세요!" | 성취감 |

### 2차시 핵심 포인트
- 수정 → commit → push 사이클을 **최소 2번** 반복
- clone 후 `npm install` → `node server.js`로 실행까지 해보기
- README.md는 시간이 되면 작성, 부족하면 과제

> 💡 **교사 메모:** Clone 실습 시 친구의 `.env` 파일은 없으므로 Supabase가 연결되지 않습니다. "코드 구조를 보는 것"이 목적이라고 안내하세요. 실행까지 하려면 각자의 `.env`를 넣어야 한다는 점을 설명하면 `.gitignore`의 중요성을 자연스럽게 이해합니다.

## 평가 기준

| 수준 | 달성 기준 |
|------|---------|
| 🟢 입문 | GitHub에 가입하고, 코드를 push했다 |
| 🟡 기본 | 코드를 수정하고 다시 push했다. 친구 코드를 clone했다 |
| 🔴 심화 | README.md를 작성하고, 커밋 히스토리를 설명할 수 있다 |

---

## 💡 교사 메모 모음

> - **Git 명령어는 외울 필요 없습니다.** 학생들에게 "add, commit, push 세 단어만 기억하세요"라고 반복하세요.
> - **시간이 부족하면** 2차시의 README 작성과 커밋 히스토리 탐색은 과제로 전환 가능합니다.
> - **Windows 학생**은 Git 설치에 시간이 걸릴 수 있습니다. 가능하면 수업 전에 설치를 안내하세요.
> - **네트워크 문제**가 가장 큰 변수입니다. 학교 Wi-Fi가 불안정하면 핫스팟 준비를 권장합니다.
> - **실수해도 괜찮다는 분위기**를 만들어주세요. Git은 되돌리기가 되니까요!

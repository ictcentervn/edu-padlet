## 🟢 기본 프롬프트 — Git 명령어 물어보기

### 프롬프트 1: Git 기초 명령어

**Claude.ai에 아래를 붙여넣으세요:**

```
Git을 처음 배우는 중학생입니다.

아래 상황에서 터미널에 입력할 명령어를 알려주세요.
각 명령어가 하는 일도 한 줄로 설명해주세요.

1. 프로젝트 폴더에서 Git을 시작하고 싶어요
2. 모든 파일을 저장 대상으로 선택하고 싶어요
3. "첫 번째 커밋"이라는 메모로 세이브 포인트를 만들고 싶어요
4. GitHub에 업로드하고 싶어요
```

### 프롬프트 2: .gitignore 만들기

```
Node.js Express 프로젝트입니다.
GitHub에 올리면 안 되는 파일 목록을 .gitignore로 만들어주세요.

프로젝트에 있는 파일들:
- server.js
- db.js
- package.json
- node_modules/ (npm 패키지 폴더)
- .env (API 키가 들어있음)
- padlet.db (SQLite 데이터베이스)
- public/ (HTML, CSS, JS 파일)
```

## 🟡 중급 프롬프트 — README.md 만들기

### 프롬프트 3: 포트폴리오용 README 생성

**Claude.ai에 아래를 붙여넣으세요:**

```
GitHub 저장소에 올릴 README.md를 만들어주세요.
마크다운 문법으로 멋지게 만들어주세요.

프로젝트 정보:
- 이름: 우리 반 패들렛
- 설명: 학교에서 사용하는 온라인 보드 앱. 카드를 붙이고 떼는 패들렛 클론
- 기능: 카드 추가, 수정, 삭제, 색상 선택
- 기술 스택: Node.js, Express, HTML/CSS/JavaScript, Supabase
- 실행 방법: npm install → .env 설정 → node server.js
- 만든 사람: [내 이름]

포함해주세요:
- 프로젝트 설명
- 스크린샷 넣을 자리 (이미지 placeholder)
- 기능 목록
- 사용 기술
- 실행 방법
- 배운 점 섹션
```

## 코드 수정 후 다시 올리기

### 프롬프트 7: 변경사항 업로드

**코드를 수정한 후 Claude.ai에 물어보세요:**

```
Git으로 코드를 관리하고 있습니다.
파일을 수정했는데, 이 변경사항을 커밋하고 GitHub에 올리는 명령어를 알려주세요.

상황:
- 이미 git init과 첫 push는 완료했습니다
- server.js 파일에서 포트 번호를 3000에서 8080으로 바꿨습니다
- 커밋 메시지는 "포트 번호 변경"으로 하고 싶어요
```

### 프롬프트 8: 되돌리기

```
Git으로 관리하는 코드를 실수로 잘못 수정했어요.
마지막 커밋 상태로 되돌리고 싶습니다.

아직 커밋은 안 했고, 파일만 수정한 상태입니다.
되돌리는 명령어를 알려주세요.
주의사항도 함께 알려주세요.
```

# 따라하면 완성! GitHub에 내 코드 올리기

> **대상:** 중학교 1학년 ~ 고등학교 3학년
> **소요 시간:** 약 40~50분

---

## 시작 전 체크리스트

- [ ] 노트북 충전 상태 확인
- [ ] 인터넷 연결 확인
- [ ] VS Code 설치 완료
- [ ] Step 3 코드가 `~/edu-padlet/code/step3-supabase` 폴더에 있다

---

## Step A: GitHub 가입 + Git 설치 확인

### A-1. GitHub 가입

1. 브라우저에서 **https://github.com** 접속
2. **Sign up** 클릭
3. 이메일, 비밀번호, 사용자명 입력
4. 이메일 인증 코드 입력
5. 가입 완료!

> 사용자명은 영문+숫자로 깔끔하게! 예: `minsoo-kim`, `suyeon-dev`

### A-2. Git 설치 확인

VS Code에서 터미널을 열고 (상단 메뉴 → Terminal → New Terminal) 입력:

```
git --version
```

`git version 2.xx.x` 라고 나오면 OK!

> 🆘 `git: command not found` 가 나왔나요? → [05_트러블슈팅.md] 1번 참조

### A-3. Git 초기 설정

```
git config --global user.name "내 이름"
git config --global user.email "내@이메일.com"
```

> ⚠️ 따옴표("") 안에 자기 이름과 이메일을 넣으세요!

> ✅ **확인 포인트:** `git config --global user.name` 입력했을 때 내 이름이 나오면 성공!

---

## Step B: 프로젝트 폴더에서 git init

터미널에서 프로젝트 폴더로 이동합니다:

```
cd ~/edu-padlet/code/step3-supabase
```

Git을 시작합니다:

```
git init
```

`Initialized empty Git repository` 라는 메시지가 나오면 성공!

> ✅ **확인 포인트:** `Initialized empty Git repository...` 메시지가 보인다

---

## Step C: .gitignore 만들기

올리면 안 되는 파일을 제외하는 설정 파일을 만듭니다.

VS Code에서 `~/edu-padlet/code/step3-supabase` 폴더 안에 새 파일을 만드세요.

- 파일 이름: `.gitignore` (점으로 시작합니다!)
- 내용:

```
node_modules/
.env
padlet.db
.DS_Store
```

저장합니다 (Ctrl+S 또는 Cmd+S).

> ⚠️ `.env` 파일에는 API 키가 들어 있습니다. GitHub에 올리면 전 세계에 공개됩니다!

> ✅ **확인 포인트:** `.gitignore` 파일이 프로젝트 폴더에 있다

---

## Step D: git add → git commit → git push

### D-1. GitHub에서 저장소 만들기

1. GitHub에 로그인
2. 오른쪽 위 **+** → **New repository**
3. Repository name: `edu-padlet`
4. **Public** 선택
5. 나머지는 체크하지 않기
6. **Create repository** 클릭

### D-2. 터미널에서 명령어 입력

한 줄씩 입력하고 Enter를 누르세요:

```
git add .
```
파일을 세이브 대상으로 선택합니다.

```
git commit -m "첫 번째 커밋: 우리 반 패들렛"
```
세이브 포인트를 만듭니다.

```
git remote add origin https://github.com/내이름/edu-padlet.git
```
GitHub 저장소 주소를 등록합니다.

> ⚠️ `내이름`을 자기 GitHub 사용자명으로 바꾸세요!

```
git branch -M main
```

```
git push -u origin main
```
GitHub에 업로드합니다!

> 로그인 창이 뜨면 GitHub 계정으로 로그인하세요.

> ✅ **확인 포인트:** 에러 없이 `push` 가 완료되었다

---

## Step E: GitHub에서 내 코드 확인

1. 브라우저에서 `https://github.com/내이름/edu-padlet` 접속
2. 파일 목록 확인:

| 보여야 하는 파일 | 보이면 안 되는 파일 |
|----------------|-------------------|
| server.js | .env |
| db.js | node_modules/ |
| package.json | padlet.db |
| public/ 폴더 | |
| .gitignore | |

> ✅ **확인 포인트:** GitHub에 내 코드가 보인다. `.env`는 보이지 않는다.

---

## Step F: 친구 코드 clone해서 실행해보기

### F-1. 친구의 GitHub 주소 받기

옆자리 친구에게 GitHub 저장소 주소를 받으세요:
`https://github.com/친구이름/edu-padlet`

### F-2. Clone하기

터미널에서:

```
cd ~/Desktop
git clone https://github.com/친구이름/edu-padlet.git friend-padlet
```

`friend-padlet` 폴더가 바탕화면에 생깁니다.

### F-3. 실행해보기

```
cd ~/Desktop/friend-padlet
npm install
```

> 💡 `.env` 파일이 없기 때문에 Supabase 연결은 안 됩니다. 코드 구조를 확인하는 것이 목적입니다!

코드를 실행하고 싶다면 자기의 `.env` 파일을 복사해 넣으세요:
```
cp ~/edu-padlet/code/step3-supabase/.env ~/Desktop/friend-padlet/.env
node server.js
```

> ✅ **확인 포인트:** 친구의 코드를 내 컴퓨터에 다운받았다

---

## Step G (심화): README.md 꾸미기 🔴

README.md는 프로젝트를 소개하는 파일입니다. GitHub에서 저장소에 들어가면 가장 먼저 보이는 글입니다.

### G-1. README.md 파일 만들기

프로젝트 폴더에 `README.md` 파일을 만들고 아래 내용을 참고해서 작성하세요:

```markdown
# 우리 반 패들렛

학교에서 사용하는 온라인 보드 앱입니다.

## 기능
- 카드 추가, 수정, 삭제
- 카드 색상 선택 (노랑, 파랑, 초록, 분홍)
- Supabase 클라우드 데이터베이스 연동

## 사용 기술
- Node.js + Express
- HTML / CSS / JavaScript
- Supabase

## 실행 방법
1. `npm install`
2. `.env` 파일에 Supabase 키 입력
3. `node server.js`
4. 브라우저에서 http://localhost:3000 접속

## 만든 사람
- [내 이름] (GitHub: @내이름)
```

### G-2. README를 GitHub에 올리기

```
git add README.md
git commit -m "README.md 추가: 프로젝트 소개"
git push
```

> ✅ **확인 포인트:** GitHub 저장소 메인 페이지에 README 내용이 보인다

## 수정 후 다시 올리기 (앞으로 계속 쓸 방법)

코드를 수정할 때마다 이 3줄만 기억하세요:

```
git add .
git commit -m "무엇을 바꿨는지 메모"
git push
```

예시:
```
git add .
git commit -m "카드 색상에 보라색 추가"
git push
```

> 이게 끝입니다! **add → commit → push**, 세 단어만 기억하세요!

# GitHub — 코드를 안전하게 저장하고 공유하는 방법

> **대상:** 중학교 1학년 ~ 고등학교 3학년 (수준별 표시 포함)

---

## 오늘 배울 내용

> 우리가 만든 패들렛 앱 코드를 **GitHub**에 올려서 안전하게 보관하고, 친구와 공유하고, 포트폴리오로 만듭니다.

---

## 생각해보기

레포트를 USB에 담아서 학교에 가져간 적 있나요?

- USB를 잃어버린 적은요?
- 컴퓨터가 갑자기 꺼져서 파일이 날아간 적은요?
- 친구와 같은 파일을 수정하다가 "최종", "진짜최종", "진짜진짜최종" 파일이 생긴 적은요?

코드도 마찬가지입니다. 열심히 만든 앱이 컴퓨터 고장 하나로 사라질 수 있어요.

오늘 배울 **GitHub**은 이 모든 문제를 해결해 줍니다!

---

## 핵심 개념 1: Git — 코드의 타임머신

```
   세이브 1         세이브 2         세이브 3
  (첫 버전)   →   (기능 추가)   →   (버그 수정)
     │                │                │
     ▼                ▼                ▼
  ┌────────┐    ┌────────┐    ┌────────┐
  │ v1.0   │    │ v2.0   │    │ v3.0   │
  │ 카드추가│    │ 수정기능│    │ 색상수정│
  └────────┘    └────────┘    └────────┘
                                   │
                          "아, v2.0이 더 나았는데..."
                                   │
                              ◀────┘  되돌리기 가능!
```

### 비유로 이해하기: 게임 세이브 포인트

게임을 하다가 중요한 장면 전에 세이브하죠?

- 보스전에서 지면 → 세이브 포인트로 돌아가기
- 코드를 망치면 → **이전 커밋(세이브 포인트)** 으로 돌아가기

**Git**은 코드에 세이브 포인트를 만들어주는 프로그램입니다.
세이브할 때마다 "무엇을 바꿨는지" 메모도 함께 남깁니다.

## 핵심 개념 2: GitHub — 코드의 구글 드라이브

```
  ┌─────────────────────────────────────────────┐
  │              GitHub (인터넷)                  │
  │                                              │
  │   ┌──────────────────────────────────────┐   │
  │   │  edu-padlet 저장소                     │   │
  │   │                                      │   │
  │   │  📁 public/                           │   │
  │   │  📄 server.js                         │   │
  │   │  📄 db.js                             │   │
  │   │  📄 package.json                      │   │
  │   │                                      │   │
  │   │  세이브 기록:                           │   │
  │   │  ✅ "첫 번째 커밋: 우리 반 패들렛"      │   │
  │   │  ✅ "카드 색상 기능 추가"               │   │
  │   └──────────────────────────────────────┘   │
  └─────────────────────────────────────────────┘
```

### 비유로 이해하기: 구글 드라이브에 폴더 올리기

구글 드라이브에 과제 폴더를 올려본 적 있나요?

- 어디서든 인터넷만 있으면 다운받을 수 있고
- 친구에게 링크를 보내면 공유할 수 있죠

**GitHub**도 같습니다. 단, **코드 전용**이라 더 강력합니다.

- 누가 언제 무엇을 바꿨는지 기록됩니다.
- 이전 버전으로 되돌릴 수 있습니다.
- 전 세계 개발자가 여기에 코드를 공유합니다.

---

## 핵심 개념 정리

| 용어 | 한 줄 설명 | 비유 |
|------|-----------|------|
| **Git** | 코드 변경 기록을 저장하는 프로그램 | 게임 세이브 시스템 |
| **GitHub** | Git 기록을 인터넷에 저장하는 서비스 | 코드 전용 구글 드라이브 |
| **Repository (저장소)** | 하나의 프로젝트 폴더 | 구글 드라이브의 폴더 하나 |
| **Commit (커밋)** | 세이브 포인트 만들기 | 게임 세이브 버튼 누르기 |
| **Push (푸시)** | 내 컴퓨터 → GitHub 업로드 | 구글 드라이브에 파일 올리기 |
| **Clone (클론)** | GitHub → 내 컴퓨터 다운로드 | 구글 드라이브에서 폴더 받기 |

---

## 왜 GitHub을 쓰는가?

### 1. 코드를 잃어버리지 않는다
컴퓨터가 고장나도 GitHub에 코드가 남아 있습니다.

### 2. 실수해도 되돌릴 수 있다
코드를 잘못 고쳤다면, 이전 커밋으로 돌아가면 됩니다.

### 3. 친구와 함께 작업할 수 있다
같은 저장소에 여러 사람이 코드를 올리고 받을 수 있습니다.

### 4. 포트폴리오가 된다
GitHub 프로필은 **개발자의 이력서**입니다.
대입 자기소개서에 "제 GitHub을 보세요"라고 쓸 수 있어요.
IT 회사 취업할 때도 GitHub 주소를 꼭 물어봅니다.

## 전체 흐름 다이어그램

```
  [내 컴퓨터]                    [GitHub]                    [친구 컴퓨터]
  ┌──────────┐               ┌──────────┐               ┌──────────┐
  │          │    push       │          │    clone       │          │
  │  코드    │ ────────────▶ │  저장소   │ ◀──────────── │  코드    │
  │  작성    │               │  (원본)   │               │  복사본  │
  │          │    clone      │          │    push        │          │
  │          │ ◀──────────── │          │ ◀──────────── │          │
  └──────────┘               └──────────┘               └──────────┘

  ① 내가 코드를 작성한다
  ② commit으로 세이브 포인트를 만든다
  ③ push로 GitHub에 업로드한다
  ④ 친구가 clone으로 내 코드를 다운받는다
  ⑤ 친구가 수정하고 push로 다시 올린다
```

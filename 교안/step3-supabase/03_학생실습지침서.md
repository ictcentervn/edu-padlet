
> **주의:** `=` 앞뒤에 공백 넣지 마세요! 따옴표로 감싸지 마세요!

### B-5. db.js 파일 만들기

VS Code에서 **db.js** 파일을 새로 만들고, 아래 코드를 **그대로** 붙여넣으세요:

```javascript
// Supabase db.js — 데이터를 클라우드(Supabase)에 저장합니다
const { createClient } = require('@supabase/supabase-js');
require('dotenv').config();

// Supabase 클라이언트 생성 (.env에서 URL과 Key를 읽어옵니다)
const supabase = createClient(
  process.env.SUPABASE_URL,
  process.env.SUPABASE_ANON_KEY
);

module.exports = {
  // 연결 테스트 (서버 시작할 때 한 번 실행)
  initialize: async () => {
    const { error } = await supabase.from('cards').select('id').limit(1);
    if (error) throw new Error(`Supabase 연결 실패: ${error.message}`);
    console.log('✅ Supabase 연결 성공');
  },

  // 카드 목록 조회
  getCards: async (boardId = 'default') => {
    const query = supabase
      .from('cards')
      .select('*')
      .order('created_at', { ascending: false });
    if (boardId !== 'all') query.eq('board_id', boardId);
    const { data, error } = await query;
    if (error) throw error;
    return data;
  },

  // 카드 생성
  createCard: async ({ title, content = '', color = 'yellow', boardId = 'default' }) => {
    const { data, error } = await supabase
      .from('cards')
      .insert({ title, content, color, board_id: boardId })
      .select()
      .single();
    if (error) throw error;
    return data;
  },

  // 카드 수정
  updateCard: async (id, updates) => {
    const { data, error } = await supabase
      .from('cards')
      .update(updates)
      .eq('id', id)
      .select()
      .single();
    if (error) throw error;
    return data;
  },

  // 카드 삭제
  deleteCard: async (id) => {
    const { error } = await supabase.from('cards').delete().eq('id', id);
    if (error) throw error;
    return { success: true };
  },

  // 보드 목록
  getBoards: async () => {
    const { data, error } = await supabase.from('boards').select('*');
    if (error) throw error;
    return data;
  },
  createBoard: async ({ name }) => {
    const id = name.toLowerCase().replace(/\s/g, '-');
    const { data, error } = await supabase
      .from('boards').insert({ id, name }).select().single();
    if (error) throw error;
    return data;
  },
};
```

> ✅ **여기까지 확인 포인트:** step3-supabase 폴더 안에 아래 파일들이 있으면 성공!

파일 구조가 이렇게 되어야 합니다:

```
step3-supabase/
  ├── server.js          ← Step 2에서 복사 (수정 없음!)
  ├── db.js              ← 새로 만든 Supabase 버전!
  ├── .env               ← Supabase URL + Key
  ├── package.json       ← npm init으로 생성
  ├── node_modules/      ← npm install로 생성
  └── public/            ← Step 2에서 복사 (수정 없음!)
       ├── index.html
       ├── style.css
       └── app.js
```

> **Step 2와 비교:** padlet.db 파일이 없어졌고, .env 파일이 생겼습니다!
> 데이터가 내 컴퓨터가 아니라 클라우드에 저장되니까요.

## D단계: Supabase 대시보드에서 데이터 확인하기

> 이 단계가 **핵심 교육 포인트**입니다!

### D-1. Supabase 대시보드 열기

1. 브라우저에서 **https://supabase.com** 에 접속합니다 (새 탭 사용)
2. 프로젝트(edu-padlet)를 클릭합니다
3. 좌측 메뉴에서 **"Table Editor"** 를 클릭합니다
4. **cards** 테이블을 클릭합니다

### D-2. 데이터 확인하기

방금 브라우저에서 추가한 카드가 **Supabase Table Editor에 보입니다!**

```
cards 테이블:
┌──────────────────────────────┬──────────┬──────────┬────────┬──────────┐
│ id (UUID)                    │ title    │ content  │ color  │ board_id │
├──────────────────────────────┼──────────┼──────────┼────────┼──────────┤
│ a1b2c3d4-e5f6-7890-...       │ 오늘 할일 │ 수학숙제  │ yellow │ default  │
│ f9e8d7c6-b5a4-3210-...       │ 점심 메뉴 │ 비빔밥   │ blue   │ default  │
└──────────────────────────────┴──────────┴──────────┴────────┴──────────┘
```

> **기억나나요?**
> - Step 1에서는 **Notion**에서 데이터를 확인했죠
> - Step 2에서는 **VS Code SQLite Viewer**에서 확인했죠
> - Step 3에서는 **Supabase Table Editor**에서 확인합니다!
>
> 도구는 달라도, "내가 입력한 데이터가 DB에 저장되었는지 확인한다"는 행동은 같습니다.

### D-3. 데이터 변화 관찰하기

1. 브라우저(localhost:3000)에서 **카드를 하나 더 추가**합니다
2. Supabase Table Editor에서 **새로고침**(F5 또는 테이블 다시 클릭)
3. 새 행이 추가되어 있습니다!
4. 브라우저에서 **카드를 삭제**합니다
5. Supabase Table Editor에서 해당 행이 사라진 것을 확인합니다

### D-4. 서버를 껐다 켜도 데이터가 남아있을까?

실험해봅시다 (Step 2에서도 해본 실험!):

1. 터미널에서 `Ctrl + C`를 눌러 **서버를 종료**합니다
2. 다시 `node server.js`로 서버를 **재시작**합니다
3. 브라우저에서 `http://localhost:3000`에 접속합니다
4. 아까 만든 카드가 **그대로 있습니다!**

> 왜 그럴까요? 데이터가 **Supabase 클라우드**에 저장되어 있기 때문입니다.
> Step 2에서는 "파일에 저장되니까 남아있다"였고,
> Step 3에서는 "클라우드에 저장되니까 남아있다"입니다.
> 서버를 꺼도, 심지어 **내 컴퓨터를 꺼도** 데이터는 Supabase에 그대로 있습니다!

> ✅ **여기까지 확인 포인트:** Supabase Table Editor에서 데이터가 보이고, 서버 재시작 후에도 카드가 남아있으면 **Step 3 기본 과정 완성!**

## 축하합니다!

여기까지 완료했으면 **3단계 전체 여정이 끝났습니다!**

**3단계에 걸쳐 만든 것:**

| | Step 1 | Step 2 | Step 3 |
|--|--------|--------|--------|
| 저장소 | Notion | SQLite | Supabase |
| 바꾼 파일 | - | db.js | db.js |
| 핵심 경험 | API로 남의 서비스 사용 | 내 컴퓨터에 직접 저장 | 클라우드에 저장, 배포 |

**불변의 진실:**
- **server.js** → 3번 모두 안 바뀜!
- **public/** → 3번 모두 안 바뀜!
- **db.js** → 이것만 3번 바뀜!

**이것이 소프트웨어 설계의 핵심입니다.** 부품(모듈)을 잘 나눠두면, 한 부분만 교체해도 전체가 동작합니다.

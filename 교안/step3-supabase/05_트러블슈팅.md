## 1. Supabase 프로젝트 생성이 안 됨 (GitHub 인증 문제)

### 증상
Supabase에 로그인하려는데 GitHub 인증이 실패하거나, 프로젝트 생성 버튼이 비활성화됩니다.

### 원인
GitHub 계정 문제이거나, GitHub에서 Supabase 앱 접근 권한을 허용하지 않은 경우입니다.

### 해결법

**GitHub 계정이 없는 경우:**
1. https://github.com 에서 가입
2. 이메일 인증 완료 (메일함 확인!)
3. 다시 Supabase 로그인 시도

**GitHub 인증이 실패하는 경우:**
1. GitHub에 먼저 로그인되어 있는지 확인
2. 브라우저 캐시/쿠키 삭제 후 재시도
3. 또는 **이메일로 직접 가입**을 선택 (GitHub 대신)

**프로젝트 생성 버튼이 비활성화:**
- 무료 프로젝트 2개 제한에 걸렸을 수 있음 → 8번 참조

> 💡 **교사 메모:** GitHub 계정 만들기를 **수업 전에 숙제**로 내주세요. 가입 시 이메일 인증에 시간이 걸려서 수업 시간에 하면 5~10분 낭비됩니다.

## 2. SQL 실행 에러 (테이블 이미 존재 등)

### 증상
SQL Editor에서 Run을 눌렀을 때 빨간색 에러:
```
ERROR: relation "cards" already exists
```

### 원인
이미 SQL을 한 번 실행해서 테이블이 만들어진 상태에서 또 실행한 것입니다.

### 해결법

**방법 1: 무시하고 진행**
- 테이블이 이미 있다는 뜻이니, 그냥 다음 단계로 넘어가면 됩니다!
- Table Editor에서 cards, boards 테이블이 보이는지 확인하세요.

**방법 2: 테이블 삭제 후 다시 만들기**
```sql
DROP TABLE IF EXISTS cards;
DROP TABLE IF EXISTS boards;
```
위 SQL을 먼저 실행한 다음, 원래 CREATE TABLE SQL을 다시 실행하세요.

> **주의:** DROP TABLE은 데이터를 모두 삭제합니다! 처음 설정 중이라면 상관없지만, 이미 카드를 추가한 후에는 조심하세요.

## 3. "relation does not exist" 에러

### 증상
서버를 실행하면:
```
Error: Supabase 연결 실패: relation "public.cards" does not exist
```

### 원인
Supabase에 테이블을 아직 만들지 않았습니다. SQL Editor에서 CREATE TABLE을 실행하지 않은 것입니다.

### 해결법
1. Supabase 대시보드로 이동
2. 좌측 메뉴 → **SQL Editor**
3. 아래 SQL을 붙여넣고 **Run** 클릭:

```sql
CREATE TABLE cards (
  id         UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title      TEXT NOT NULL,
  content    TEXT DEFAULT '',
  color      TEXT DEFAULT 'yellow',
  board_id   TEXT DEFAULT 'default',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE boards (
  id   TEXT PRIMARY KEY,
  name TEXT NOT NULL
);

INSERT INTO boards (id, name) VALUES ('default', '우리 반 보드');
```

4. "Success" 확인 후 서버 재시작

## 4. SUPABASE_URL 또는 SUPABASE_ANON_KEY 오류

### 증상
서버를 실행하면:
```
Error: supabaseUrl is required
```
또는
```
Error: supabaseKey is required
```
또는
```
FetchError: request to https://undefined.supabase.co/rest/v1/cards failed
```

### 원인
.env 파일에 URL이나 Key가 비어있거나, 잘못 입력되었습니다.

### 해결법

**1. .env 파일이 있는지 확인:**
```bash
ls -la .env     # Mac
dir .env        # Windows
```

**2. .env 파일 내용 확인:**
```
SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**3. 흔한 실수 체크리스트:**
- [ ] `=` 앞뒤에 **공백이 없는지** 확인
  - 틀림: `SUPABASE_URL = https://...`
  - 맞음: `SUPABASE_URL=https://...`
- [ ] 값을 **따옴표로 감싸지 않았는지** 확인
  - 틀림: `SUPABASE_URL="https://..."`
  - 맞음: `SUPABASE_URL=https://...`
- [ ] URL이 `https://`로 시작하는지 확인
- [ ] Key가 `eyJ`로 시작하는지 확인
- [ ] 줄바꿈이 정확한지 확인 (URL과 Key가 각각 한 줄)

**4. Supabase 대시보드에서 다시 복사:**
- Settings → Data API → Project URL의 **복사 아이콘** 클릭
- Settings → Data API → anon public의 **복사 아이콘** 클릭

> 💡 **교사 메모:** .env 관련 문제는 Step 1에서도 가장 많이 발생했던 문제입니다. "복사 아이콘"을 사용하도록 강조하세요. 드래그로 선택하면 앞뒤 공백이나 줄바꿈이 포함될 수 있습니다.

## 5. .env 형식 오류

### 증상
.env 파일을 만들었는데 서버가 URL이나 Key를 읽지 못합니다.

### 원인
.env 파일 형식이 잘못되었습니다. Step 1에서도 겪었던 문제입니다!

### 해결법

> **Step 1 교훈 리마인드:**
> .env 파일은 매우 까다로운 형식을 요구합니다.

**올바른 형식:**
```
SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**자주 하는 실수:**

1. **빈 줄이 값 중간에 들어감:**
   ```
   SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6
   IkpXVCJ9...
   ```
   → Key는 **반드시 한 줄**에 다 들어가야 합니다!

2. **한글 주석 형식 오류:**
   ```
   # 여기에 URL을 넣으세요
   SUPABASE_URL=https://...
   ```
   → `#`으로 시작하는 줄은 주석입니다. 이건 괜찮아요!

3. **파일 이름이 .env가 아님:**
   - `.env.txt` → 안 됨!
   - `env` → 안 됨!
   - `.env` → 맞음!
   - Windows에서 메모장으로 저장하면 `.env.txt`가 될 수 있어요. VS Code에서 만드세요.

4. **BOM(Byte Order Mark) 포함:**
   - Windows에서 메모장으로 만들면 파일 앞에 보이지 않는 문자가 추가될 수 있음
   - **VS Code에서 만드는 것이 안전**합니다

## 6. 데이터가 Supabase 대시보드에 안 보임

### 증상
카드를 추가했는데 Supabase Table Editor에 데이터가 보이지 않습니다.

### 원인
대부분 **새로고침을 안 해서** 그렇습니다.

### 해결법

**1. Table Editor 새로고침:**
- 브라우저에서 **F5** 또는 **Ctrl+R**로 페이지 새로고침
- 또는 좌측 메뉴에서 cards 테이블을 **다시 클릭**

**2. 올바른 테이블을 보고 있는지 확인:**
- 좌측에 `cards` 테이블을 클릭했는지 확인
- `boards` 테이블과 혼동하지 마세요

**3. 데이터가 정말 저장 안 된 경우:**
- 브라우저 개발자 도구(F12) → **Console** 탭 → 빨간색 에러 확인
- 터미널에서 서버 로그 확인 (에러 메시지가 있는지)

**4. 필터가 걸려 있는 경우:**
- Table Editor 상단에 필터가 설정되어 있으면 일부 데이터만 보일 수 있음
- 필터를 모두 해제하세요

## 7. "permission denied" 에러

### 증상
카드를 추가하려 하면:
```
Error: permission denied for table cards
```

### 원인
anon key가 아닌 다른 키를 넣었거나, RLS(Row Level Security) 설정 문제입니다.

### 해결법

**1. anon key를 확인:**
- Settings → Data API에서 **"anon" "public"** 이라고 표시된 키를 복사했는지 확인
- **"service_role" "secret"** 키를 넣으면 다른 문제가 생길 수 있음
- anon key는 `eyJ`로 시작하는 긴 문자열입니다

**2. RLS 비활성화 (교육용 설정):**
- Supabase 대시보드 → Authentication → Policies
- cards 테이블의 RLS가 활성화되어 있으면, 아래 SQL을 실행:

```sql
-- 교육용: 모든 접근 허용 (실제 서비스에서는 이렇게 하면 안 됩니다!)
ALTER TABLE cards ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow all access to cards" ON cards
  FOR ALL USING (true) WITH CHECK (true);

CREATE POLICY "Allow all access to boards" ON boards
  FOR ALL USING (true) WITH CHECK (true);
```

> 💡 **교사 메모:** 교육 환경에서는 RLS를 비활성화하거나 모든 접근을 허용하는 정책을 사용하는 것이 편합니다. 실제 서비스에서는 반드시 적절한 보안 정책을 설정해야 한다는 점을 학생들에게 알려주세요.

## 8. 무료 프로젝트 2개 초과

### 증상
"New project" 버튼이 비활성화되거나 "You have reached the maximum number of free projects" 메시지가 나옵니다.

### 원인
Supabase 무료 플랜은 **프로젝트 2개까지만** 허용합니다.

### 해결법

**방법 1: 안 쓰는 프로젝트 일시정지(Pause)**
1. Supabase 대시보드에서 안 쓰는 프로젝트 선택
2. Settings → General → 맨 아래 **"Pause project"** 클릭
3. 일시정지된 프로젝트는 무료 개수에서 제외됩니다
4. 나중에 다시 활성화(Resume) 가능

**방법 2: 안 쓰는 프로젝트 삭제**
1. Settings → General → 맨 아래 **"Delete project"** 클릭
2. 프로젝트 이름 입력하여 확인
3. **주의: 삭제하면 데이터도 사라집니다!**

**방법 3: 다른 이메일로 새 계정 만들기**
- 권장하지 않지만, 급할 때 사용 가능

> 💡 **교사 메모:** 수업 시작 전에 학생들에게 "Supabase 계정에 기존 프로젝트가 있는지 확인하고, 필요 없는 것은 Pause해두세요"라고 안내하세요.

## 9. Vercel 배포 에러 (선택 심화)

### 증상
`vercel` 명령어 실행 시 에러, 또는 배포 후 사이트가 안 열림

### 원인별 해결법

**"vercel: command not found"**
```bash
npm install -g vercel
```

**배포 후 500 에러 (Internal Server Error)**
- 환경변수가 설정되지 않았을 가능성이 높음
```bash
vercel env add SUPABASE_URL
vercel env add SUPABASE_ANON_KEY
vercel --prod    # 환경변수 설정 후 재배포
```

**"Cannot find module 'express'"**
- package.json에 express가 dependencies에 있는지 확인
- `npm install`을 다시 실행한 후 재배포

**배포 후 API 호출이 안 됨**
- Vercel은 기본적으로 **서버리스(Serverless)** 환경입니다
- Express.js를 서버리스로 감싸는 `vercel.json` 설정이 필요할 수 있습니다:
```json
{
  "version": 2,
  "builds": [
    { "src": "server.js", "use": "@vercel/node" }
  ],
  "routes": [
    { "src": "/(.*)", "dest": "server.js" }
  ]
}
```

## 10. Realtime 구독이 안 됨 (선택 심화)

### 증상
한쪽 브라우저에서 카드를 추가해도 다른 쪽에서 자동으로 반영되지 않습니다.

### 원인별 해결법

**1. Realtime이 활성화되지 않음:**
- Supabase 대시보드 → Database → Replication
- cards 테이블의 Realtime이 **활성화(ON)** 되어 있는지 확인
- 꺼져 있으면 토글을 ON으로 바꾸세요

**2. CDN 스크립트가 없음:**
- `index.html`의 `<head>`에 아래가 있는지 확인:
```html
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
```

**3. 구독 코드 오류:**
- `app.js`에서 Supabase URL과 Key를 직접 넣어야 합니다 (프론트엔드에서는 .env를 못 읽음!)
- 콘솔(F12)에서 에러 메시지 확인

**4. 새로고침으로 대체:**
- Realtime이 안 되면, 일단 **수동 새로고침(F5)** 으로 확인하세요
- Realtime은 심화 기능이니, 기본 동작이 되는 것만으로도 충분합니다!

## 3단계 전체 트러블슈팅 비교표

각 Step에서 가장 흔한 에러 Top 3:

| 순위 | Step 1 (Notion) | Step 2 (SQLite) | Step 3 (Supabase) |
|------|----------------|----------------|-----------------|
| 1위 | Notion API 키 발급 어려움 | better-sqlite3 설치 실패 | .env URL/Key 오류 |
| 2위 | DB ID 복사 실수 | 실행 경로 오류 (다른 폴더에서 실행) | SQL 미실행 ("relation does not exist") |
| 3위 | Integration-DB 연결 누락 | padlet.db 안 생김 | GitHub 계정/인증 문제 |

### Step별 난이도 비교

| 항목 | Step 1 | Step 2 | Step 3 |
|------|--------|--------|--------|
| 설정 난이도 | 높음 (API 키 발급 복잡) | 낮음 (npm install이면 끝) | 중간 (Supabase 설정 필요) |
| 트러블슈팅 난이도 | 높음 (Notion API 에러 해독 어려움) | 중간 (빌드 에러 가능) | 중간 (.env 실수 주의) |
| 자주 막히는 지점 | API 키 + DB 연결 | 패키지 설치 | URL/Key 복사 |
| 해결 시간 (평균) | 10~20분 | 5~10분 | 5~10분 |

### 3단계 공통 문제

| 문제 | 해결법 |
|------|-------|
| `node server.js` 실행 위치가 다름 | `pwd`로 현재 폴더 확인 → 올바른 폴더에서 실행 |
| `Cannot find module 'express'` | `npm install` 실행 |
| `EADDRINUSE: address already in use :::3000` | 다른 터미널에서 실행 중인 서버 종료 (Ctrl+C) |
| .env 파일이 Finder에서 안 보임 | Cmd+Shift+. (Mac) 또는 VS Code에서 편집 |
| 브라우저에서 화면이 안 나옴 | `http://localhost:3000` 정확히 입력 + 서버 실행 중 확인 |
